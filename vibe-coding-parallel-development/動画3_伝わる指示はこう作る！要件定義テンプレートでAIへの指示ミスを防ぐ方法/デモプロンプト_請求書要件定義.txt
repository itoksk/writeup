# 🎯 要件定義テンプレート・デモプロンプト集：AIへの指示を一発で伝える

## ❌ ダメな例（曖昧すぎる指示）

```
Todoアプリ作って
```

### AIの反応：
「どのような機能が必要ですか？対象ユーザーは？技術スタックは？」
→ 質問攻めで進まない、モチベーション低下

---

## ⭕ 良い例（テンプレート使用）

```
【概要】個人用のシンプルなTodoアプリ
【機能要件】
1. タスクの追加・削除・編集
2. 完了/未完了の切り替え  
3. 期限設定
【技術要件】GAS + HTML + スプレッドシート
【制約】30分で動くMVP
【AI活用方針】コード生成はChatGPT、UI設計はGemini
```

### AIの出力：
```javascript
// 完全な実装コードが即座に生成！
function addTodo(title, dueDate) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('Todos');
  sheet.appendRow([new Date(), title, dueDate, 'pending']);
}
// ... 続く
```

## 🚀 魔法の要件定義テンプレート

### 基本テンプレート（5項目）
```
【概要】○○を解決する（1行）
【機能要件】最重要3つ
【技術要件】使う技術スタック
【制約】時間・スキル・環境
【AI活用方針】どのAIで何を
```

### 実例：請求書管理システム
```
【概要】請求書作成を5分以内で完了するシステム
【機能要件】
1. 請求書の作成・保存
2. 顧客情報の管理
3. 請求書一覧の表示
【技術要件】GAS + スプレッドシート + HTMLサービス
【制約】開発時間2時間、社内10人利用、既存Googleアカウント
【AI活用方針】ChatGPT:ロジック、Gemini:UI、Claude:テスト
```

## 📝 段階的な要件定義作成

### Step 1：最小要件から始める
```
請求書管理システムの要件を整理したい。
まず最小限の機能から始めたいです。
何が必要でしょうか？
```

### Step 2：AIの提案を受けて具体化
```
提案された機能のうち、
1. 請求書作成
2. データ保存
3. 一覧表示
この3つで始めます。技術スタックはGASで。
```

### Step 3：テンプレートに落とし込む
```
以下のテンプレートで要件を整理してください：
[先ほどの内容を入れて]
【概要】
【機能要件】
【技術要件】
【制約】
【AI活用方針】
```

## 🎮 用途別テンプレート

### 新規プロジェクト用
```
【概要】[解決したい課題を1行で]
【MVP機能】[今すぐ必要な機能3つ]
【技術】[使い慣れた技術]
【期限】[○時間以内]
【スコープ外】[今回やらないこと]
```

### 機能追加用
```
【現状】[既存システムの概要]
【追加機能】[新しく加えたい機能]
【影響範囲】[変更が及ぶ箇所]
【工数】[見積時間]
【リスク】[考慮すべき点]
```

### バグ修正用
```
【症状】[エラーの内容]
【再現手順】[どうすると起きるか]
【期待動作】[本来の動き]
【環境】[技術スタック]
【緊急度】[高/中/低]
```

## 💡 要件定義を明確にする質問集

### AIに聞くべき質問
```
1. この機能で最も重要な部分は何ですか？
2. 最小限で動かすなら何が必要ですか？
3. 30分で作るならどこまでできますか？
4. どの技術を使うのが最速ですか？
5. 省略しても問題ない機能は何ですか？
```

### 自分で答えるべき質問
```
1. 誰が使うのか？（対象ユーザー）
2. いつまでに必要か？（期限）
3. どんな環境で動くか？（制約）
4. 何ができればOKか？（成功基準）
5. 何はしなくていいか？（スコープ外）
```

## 🔥 スピード重視の要件定義

### 5分で決める要件
```
1. 作るもの：○○システム
2. 必須機能：3つまで
3. 使う技術：慣れているもの
4. 完成期限：○時間
5. こだわらない：デザイン、最適化
```

### 実例：タスク管理
```
1. 作るもの：タスク管理
2. 必須機能：追加・完了・一覧
3. 使う技術：GAS
4. 完成期限：30分
5. こだわらない：UI、認証
```

## ✅ 良い要件定義の特徴

```
✅ 具体的な数値がある
  「5分以内に完了」「月100件処理」

✅ 制約が明確
  「既存システムと連携」「社内10人で使用」

✅ 優先順位がはっきり
  「まず○○、次に△△」

✅ スコープが限定的
  「今回は○○だけ」

✅ 技術が決まっている
  「GAS + スプレッドシート」
```

---

## ❌ 悪い要件定義の特徴

```
❌ 曖昧な表現
  「使いやすく」「かっこよく」

❌ 欲張りすぎ
  「全機能搭載」「完璧なシステム」

❌ 技術未定
  「最適な技術で」「ベストプラクティスで」

❌ 期限なし
  「いつか完成」「時間があるとき」

❌ 対象不明
  「みんなが使える」「汎用的に」
```

---

## 📊 要件の優先度マトリクス

```
        緊急度高              緊急度低
重要度高  【今すぐ】          【計画的に】
        ログイン機能         レポート機能
        データ保存          グラフ表示

重要度低  【クイック対応】     【後回し】
        エラー表示          アニメーション
        入力確認           テーマ変更
```

## 💬 覚えておくべきこと

**完璧な要件定義は不要**
60%の精度で十分。作りながら改善。

状況に応じて柔軟に変更OK。

**AIは要件定義のパートナー**
