# 第3回：伝わる指示はこう作る！要件定義テンプレートでAIへの指示ミスを防ぐ方法（10分）

## 導入フレーズ（1分）

「AIにお願いしても、微妙な結果ばかり…その原因、指示が曖昧だからかもしれません。

- 「いい感じで作って」と言っても伝わらない
- 何度も修正指示の繰り返し
- 結局自分で書き直すことに

でも実は、この問題には簡単な解決策があるんです。

それが『要件定義テンプレート』。今日は、AIへの指示を一発で伝える方法をマスターしましょう！」

## Before/After（1.5分）

「曖昧な指示と、テンプレートを使った指示の違いを見てみましょう。

**Before：曖昧な指示**
- 「請求書システム作って」
- AIが質問攻めで返してくる
- 何度もやり取りが必要
- 時間がかかる上に期待外れ
- 結果：フラストレーション

**After：テンプレート使用**
- 構造化された要件を提示
- AIが的確に理解
- 一発で期待通りの結果
- 修正も最小限
- 結果：スムーズな開発

この違いが、開発効率を3倍に変えます！」

## デモ（2分）

「例：同じTodoアプリの要件を曖昧verとテンプレverで比べてみましょう。

【画面共有：2画面比較】

**曖昧な指示：**
```
Todoアプリ作って。
使いやすい感じで。
```

AIの反応：
```
どのような機能が必要ですか？
対象ユーザーは？
技術スタックは？
デザインの方向性は？
```
→ 質問の嵐で進まない

**テンプレート使用：**
```
【概要】個人用のシンプルなTodoアプリ
【機能要件】
- タスクの追加・削除・編集
- 完了/未完了の切り替え
- 期限設定
【技術要件】GAS + HTML + スプレッドシート
【制約】30分で動くMVP
【AI活用方針】コード生成はChatGPT、UI設計はGemini
```

AIの反応：
```
了解しました。以下のコードで実装できます：
[完全なコード生成]
```
→ 即座に実装可能なコード！」

## 構築手順（1分）

「テンプレの構成は①概要②機能要件③技術要件④制約⑤AI活用方針の5項目です。

**要件定義テンプレート：**

1. **概要** - 何を作るか（1行）
2. **機能要件** - 必須機能3つ
3. **技術要件** - 使う技術
4. **制約** - 時間・予算・スキル
5. **AI活用方針** - どのAIで何を

これだけで、AIへの指示が劇的に改善します！」

## 設定実演（3.5分）

「ChatGPTにテンプレを渡して、『これを埋めてください』と聞く実演を行います。

【画面共有：ChatGPT実演】

**Step 1：テンプレートを提示**
```
以下のテンプレートで請求書管理システムの要件を整理してください：

【概要】
【機能要件】（最重要3つ）
【技術要件】
【制約】
【AI活用方針】

私の状況：
- GASは使える
- 開発時間は2時間

```

**ChatGPTの出力：**

```
【機能要件】
1. 請求書の作成・保存
2. 顧客情報の管理
```

**Step 3：他のAIツールへも展開**


→ 統一された理解で各ツールが協力！」

## 注意点・活用ポイント（1分）

「"欲しい機能"ではなく"何を解決したいか"を書くことが重要です。

**✅ 良い要件定義：**
- 「請求書作成を5分以内に」（解決したい課題）
- 「月100件を処理」（具体的な数値）
- 「既存システムと連携」（制約条件）

**❌ 悪い要件定義：**
- 「かっこいいUI」（曖昧）
- 「最新技術を使う」（目的不明）
- 「全機能搭載」（スコープ不明）

**成功の秘訣：**
完璧を求めず、60%の精度でスタート！」
