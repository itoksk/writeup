## ⭕ 良い例（適材適所で使い分け）

### 【ChatGPT】要件整理とチケット化担当

```
Todoアプリを作りたいです。
必要な機能をチケット形式で整理してください。

各チケットには以下を含めて：
- 目的（1行）
- 完了条件（3つ）
- 見積時間
- 担当ツール（ChatGPT/Gemini/AI Studio）
```

**ChatGPTの出力：**
```
チケット #001：データ構造設計
目的：Todoデータの保存構造を決める
完了条件：
- スプレッドシートの列定義完了
- データ型決定
- サンプルデータ作成
見積時間：30分
担当ツール：Google AI Studio

チケット #002：入力フォームUI
目的：Todo入力画面を作る
完了条件：
- HTMLフォーム完成
- レスポンシブ対応
- Material Design適用
見積時間：45分
担当ツール：Gemini
```

### 【Gemini】UIデザイン担当

```
TodoアプリのUIを作って。
Material Design風で、モバイルファースト。

必要な要素：
- タスク入力欄
- 追加ボタン
- タスクリスト（チェックボックス付き）
- 完了/未完了の切り替えタブ

HTMLとCSSを全部含めて、コピペで動くやつ。
```

**Geminiの特殊能力（画像からUI生成）：**
```
[手書きスケッチの画像をアップロード]
このTodoアプリのワイヤーフレームを
HTMLとCSSで再現して。
Material Designのコンポーネントで。
```

### 【Google AI Studio】バックエンド担当

```
TodoアプリのバックエンドAPI設計。
GASで実装、スプレッドシート連携。

必要な関数：
- addTodo(title, dueDate)
- getTodos()
- updateTodo(id, status)
- deleteTodo(id)

各関数の完全なコードを。
```

```
各シートの列名とデータ型を。
```

**→ 15分で全パーツが揃う！**

## 📝 段階的な実装例

### Phase 1：要件整理（ChatGPT）
```
シンプルなTodoアプリの要件を整理して。
MVP（最小限の機能）で。
技術スタックはGAS + HTML。
```

### Phase 2：UI作成（Gemini）
```
Phase 1の要件に基づいて、
Todoアプリの入力フォームHTML作って。
シンプルでいいから、すぐ使えるやつ。
```

### Phase 3：バックエンド（Google AI Studio）
```
Phase 1の要件で、
GASのバックエンド関数群を作って。
スプレッドシートにデータ保存。
```

### Phase 4：統合（ChatGPT）
```
GeminiのHTMLとAI StudioのGASを
統合するコードを書いて。
HTMLからGAS関数を呼び出す方法で。
```

## 🏆 効率化のコツ

1. **3つのタブを常に開く** - 切り替えロスを最小に
2. **5分ルール** - 1つのツールで5分以上悩まない
3. **動いたら次へ** - 完璧を求めない
4. **エラーは別ツール** - 視点を変える
5. **15分で統合** - 早めに繋げて確認

**Remember: ツールの使い分けが開発速度を2倍にする！**
