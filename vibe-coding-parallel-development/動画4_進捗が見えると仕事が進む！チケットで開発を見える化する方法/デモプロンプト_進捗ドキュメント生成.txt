## 🚀 スプレッドシートでチケット管理

### 基本テンプレート作成

```
| チケット# | タイトル | 優先度 | 見積時間 | 実績時間 | ステータス | 完了日 |
|----------|---------|--------|----------|----------|-----------|--------|
| #001 | ログイン画面 | 高 | 3h | 2.5h | 完了 | 11/20 |
| #002 | DB接続 | 高 | 2h | - | 作業中 | - |
| #003 | レポート機能 | 中 | 4h | - | 未着手 | - |
```

### 条件付き書式設定

```
優先度「高」→ 赤色背景
ステータス「完了」→ 緑色文字
期限超過 → 黄色警告
```

### 進捗計算式

```
完了数：=COUNTIF(F:F,"完了")
完了率：=COUNTIF(F:F,"完了")/COUNTA(F:F)*100
今週完了：=COUNTIFS(F:F,"完了",G:G,">="&TODAY()-7)
```

## 📊 グラフで可視化

### 円グラフ：ステータス別割合
```
=QUERY(A:F,"SELECT F, COUNT(A) GROUP BY F")
→ 円グラフ作成
```

### 棒グラフ：日別完了数
```
=QUERY(A:G,"SELECT G, COUNT(A) WHERE F='完了' GROUP BY G")
→ 棒グラフ作成
```

### 折れ線：累計進捗率
```
=SPARKLINE(進捗率の範囲)
→ ミニグラフで推移表示
```

## 📈 進捗ダッシュボード作成

### スプレッドシートの関数

```
【今日の進捗】
=COUNTIFS(F:F,"完了",G:G,TODAY())

【今週の進捗】
=COUNTIFS(F:F,"完了",G:G,">="&TODAY()-7)

【残タスク数】
=COUNTIF(F:F,"未着手")+COUNTIF(F:F,"作業中")

【平均処理時間】
=AVERAGE(E:E)

【見積精度】
=AVERAGE(E:E/D:D)*100
```

### 視覚化のコツ

```
1. 色分けルール
   完了 = 緑（#4CAF50）
   作業中 = 黄（#FFC107）
   未着手 = 灰（#9E9E9E）
   
2. 進捗バー
   =REPT("█",完了数)&REPT("░",未完了数)
   
3. 達成率表示
   =TEXT(完了率,"0%")&" 完了"
```

## 💡 便利な進捗管理テクニック

### 1. ステータス管理の3段階

```
未着手 → 作業中 → 完了

必ず1つずつ進める
同時に複数を「作業中」にしない
```

### 2. 優先度の明確化

```
高：今日中に必要
中：今週中に必要
低：余裕があれば
```

### 3. 見積と実績の記録

```
見積：作業前に時間を予想
実績：実際にかかった時間
精度：実績÷見積×100%

→ 徐々に見積精度が向上
```

## 📝 チケット作成のコツ

### 良いチケット例

```
タイトル：ログイン画面のバリデーション追加
優先度：高
見積：2h
詳細：
- メールアドレス形式チェック
- パスワード8文字以上
- エラーメッセージ表示
完了条件：
- バリデーション動作確認
- エラー表示確認
- テスト3ケース通過
```

### 悪いチケット例

```
タイトル：機能追加
優先度：？
見積：？
詳細：いろいろ
```

## 🔄 週次振り返りテンプレート

### AIへの振り返り依頼

```
今週のチケット実績から振り返りをしてください：

【データ】
完了：8チケット（24h）
未完了：4チケット
見積精度：85%
優先度達成率：高100%、中50%、低0%

【質問】
1. 良かった点は？
2. 改善点は？
3. 来週の注力ポイントは？
```

### 改善アクション

```
1. 見積精度向上
   → 類似タスクの実績を参考に
   
2. 優先度の見直し
   → 毎朝5分で優先度確認
   
3. 作業中タスクの削減
   → 1つ完了してから次へ
```

## 🏆 成功のポイント

### DO ✅
- 毎日5分の更新時間確保
- 完了チケットは残す（記録として）
- グラフで視覚化
- 週1回の振り返り
- 小さく分割（2時間以内）

### DON'T ❌
- 細かすぎる管理（5分単位）
- 更新を後回し
- 完璧主義
- 複数同時進行
- 見積なしスタート
