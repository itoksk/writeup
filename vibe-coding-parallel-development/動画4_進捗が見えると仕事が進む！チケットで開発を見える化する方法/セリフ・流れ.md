# 第4回：進捗が見えると仕事が進む！チケットで開発を見える化する方法（10分）

## 導入フレーズ（1分）

「"進んでる感じがしない"って思ったことありませんか？

- 朝から晩までコード書いているのに、成果が見えない
- 1週間前と今で、何が変わったか分からない
- モチベーションが続かない

それ、実は"見える化不足"が原因なんです。

今日は、進捗を可視化して、達成感とモチベーションを維持する方法をお伝えします！」

## Before/After（1.5分）

「直感だけで進めた場合と、チケット管理を導入した場合の違いを見てみましょう。

**Before：見える化なし**
- 月曜：「今週中に完成させよう」
- 火曜：「どこまでやったっけ？」
- 水曜：「進んでない気がする...」
- 木曜：「もういいや、来週にしよう」
- 結果：完成率20%、挫折

**After：チケット管理導入**
- 月曜：「今日3チケット完了！」
- 火曜：「進捗40%、順調！」
- 水曜：「あと5チケット」
- 木曜：「残り2つ、明日完成！」
- 結果：完成率85%、達成感

数字で見えると、やる気が全然違います！」

## 構築手順（1分）

「優先度設定 → ステータス列追加 → 進捗ボード作成 の流れで管理画面を作ります。

**3つのステップ：**

1. **ステータス管理**
   - 未着手 / 作業中 / 完了
   - 必ず1つずつ進める

2. **優先度設定**
   - 高 / 中 / 低で色分け
   - 高から順に着手

3. **進捗の可視化**
   - 完了率をグラフ化
   - 週次サマリー作成」

## 設定実演（3.5分）

「Googleスプレッドシートでチケット一覧を作成し、フィルタや条件付き書式で"見える化"していきます。

【画面共有：スプレッドシート実演】

**Step 1：基本テーブル作成**

| チケット# | タイトル | 優先度 | 見積時間 | 実績時間 | ステータス | 完了日 |
|----------|---------|--------|----------|----------|-----------|--------|
| #001 | ログイン画面 | 高 | 3h | 2.5h | 完了 | 11/20 |
| #002 | DB接続 | 高 | 2h | - | 作業中 | - |
| #003 | レポート機能 | 中 | 4h | - | 未着手 | - |

**Step 2：条件付き書式設定**
```
優先度「高」→ 赤色背景
ステータス「完了」→ 緑色文字
期限超過 → 黄色警告
```

**Step 3：進捗ダッシュボード作成**
```
完了数：=COUNTIF(F:F,"完了")
完了率：=COUNTIF(F:F,"完了")/COUNTA(F:F)*100
今週完了：=COUNTIFS(F:F,"完了",G:G,">="&TODAY()-7)
```

**Step 4：グラフで可視化**
- 円グラフ：ステータス別割合
- 棒グラフ：日別完了数
- 折れ線：累計進捗率

**Step 5：AIで週次レポート生成**
```
[ChatGPTへ]
このチケット一覧から今週の成果をまとめて：
[データをコピペ]
```

→ 自動で振り返りレポートが生成！」

## 注意点・活用ポイント（1分）

「"完璧な管理"を目指すより、"自分が迷わない仕組み"を作ることが大切です。

**よくある失敗：**
- 細かすぎる管理（5分単位）→ 管理が目的化
- 更新を忘れる → 形骸化

**成功のコツ：**
- 最小単位は30分〜2時間
- 1日の終わりに5分更新
- 完了チケットは残す（達成感の記録）
- 週1回の振り返り（改善点発見）

**Remember：**
見えないものは管理できない。
見えるものは改善できる！」
