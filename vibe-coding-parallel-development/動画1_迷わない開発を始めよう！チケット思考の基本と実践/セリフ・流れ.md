# 第1回：迷わない開発を始めよう！チケット思考の基本と実践（10分）

## 導入フレーズ（1分）

「AIでアプリを作ろうと思っても、途中で"何を作っているのか分からなくなる"って経験ありませんか？

- 開始時はモチベーション高い
- 数日後には『これ何だっけ？』
- 未完成プロジェクトが増えていく

そんな状態を防ぐのが、今回紹介する『チケット思考』です。

開発の迷いを無くし、着実に前進する方法を学びましょう！」

## Before/After（1.5分）

「従来の開発とチケット思考の違いを見てみましょう。

**Before：思いつきで試作 → 挫折の繰り返し**
- 「とりあえず作ってみる」でスタート
- 何を作っていたか忘れる
- どこまでできたか分からない
- 同じミスを繰り返す
- 結果：未完成プロジェクトの山

**After：チケット思考で見える化**
- 目的・進捗・完了基準が見える化
- やることが明確
- 進捗が一目瞭然
- 完了基準がはっきり
- 結果：着実に完成へ

この違いが、開発の成功率を大きく変えます！」

## デモ（2分）

「実際に、1つの開発アイデアをチケットに落とし込んだ例をご覧ください。

【画面共有：チケット一覧】

**例：請求書管理システム開発**

```
チケット #001：基本データ構造の作成
【目的】請求書データを保存する仕組みを作る
【完了条件】
- スプレッドシートに請求書シート作成
- 必要な列（顧客名、金額、日付）を定義
- サンプルデータ3件を登録
【見積時間】30分
【ステータス】完了✅
【実績】25分

チケット #002：入力フォームの作成
【目的】請求書データを入力する画面を作る
【完了条件】
- HTMLフォームが表示される
- 入力値をGASで受け取れる
- バリデーションが動作する
【見積時間】2時間
【ステータス】作業中🔧
```

このように、やることを明確に定義することで、迷いがなくなります。
何より『完了した！』という達成感が、次のモチベーションになるんです。」

## 構築手順（1分）

「チケット化の基本は3ステップです。

**①目的を明確に**
- 「何のために作るか」を1文で
- 例：『請求書の入力を簡単にする』

**②分解して並べる**
- 大きな機能を小さなタスクに
- 例：データ構造 → 入力 → 保存 → 表示

**③完了条件を決める**
- 「どうなったら完了か」を具体的に
- 例：『3件のデータが保存できる』

これだけで、開発の見通しが格段に良くなります！」

## 設定実演（3.5分）

「ChatGPTで '開発チケットを作成してください' と指示し、どのように指示を出すと精度が上がるか見ていきましょう。

【画面共有：ChatGPT】

**基本プロンプト：**
```
請求書管理システムを作りたいです。
必要な機能をチケット形式で分解してください。
```

**改善版プロンプト（精度UP）：**
```
請求書管理システムを作りたいです。
必要な機能をチケット形式で分解してください。

各チケットには以下を含めてください：
- 目的（1行）
- 完了条件（箇条書き3つ）
- 見積時間（30分〜3時間）
- 優先度（高/中/低）

まず基本機能5つをお願いします。
```

**ChatGPTの出力例：**
```
チケット #001：データ構造の設計【優先度：高】
目的：請求書データの保存先を準備する
完了条件：
- スプレッドシートに必要な列が作成されている
- データ型が定義されている
- テストデータが1件入っている
見積時間：30分
```

**さらに精度を上げるコツ：**
- 「チケット形式で」と明示
- フォーマットを具体的に指定
- 時間の上限を設定（半日以内）
- 「基本機能から」と順番を指示

こうすることで、AIが実用的なチケットを生成してくれます！」

## 注意点・活用ポイント（1分）

「"1チケット＝半日以内に完了できる"サイズ感がコツです。大きすぎると管理できなくなります。

**✅ 良いチケットの例：**
- 「ログインフォームを作成」（2-3時間）
- 「バリデーション追加」（1時間）
- 「エラーメッセージ表示」（30分）

**❌ 悪いチケットの例：**
- 「認証システム全体を構築」（数日〜数週間）
- 「アプリを完成させる」（曖昧すぎる）
- 「いい感じにする」（完了条件が不明）

**成功のポイント：**
- 小さく分解する勇気
- 完了の喜びを味わう
- 週次で振り返る習慣」

## まとめ（1分）

「チケット化で『今やること』を明確にする。これがVibe Codingを成功させる第一歩です。

**今日のポイント：**
- チケット思考 = 開発の羅針盤
- 小さく分解 = 達成感の積み重ね  
- 見える化 = モチベーション維持
- 1チケット = 半日以内が鉄則

チケット思考は、開発を『作業』から『達成』の連続に変えます。

次回は『複数のAIツールをどう使い分けるか』を解説します。
ChatGPT、Gemini、Google AI Studio...それぞれの得意分野を活かす方法をお伝えします！

まずは今日、やりたいことを3つのチケットに分解してみてください。
きっと、開発が楽しくなるはずです！」
