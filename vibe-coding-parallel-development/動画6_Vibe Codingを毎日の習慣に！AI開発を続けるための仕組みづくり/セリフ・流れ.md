# 第6回：作って終わりにしない！改善を続ける仕組みづくり（10分）

## 導入フレーズ（1分）

「せっかく作ったアプリ、1回使って終わっていませんか？

- 作った時は便利だったけど、今は使ってない
- 不具合があるけど、そのまま放置
- もっと改善できるのに、時間がなくて手つかず

実は、作ったアプリを『継続的に改善する仕組み』を作れば、どんどん便利になっていくんです。

今日は最終回。業務改善を継続させる、改善サイクルの作り方をマスターしましょう！」

## Before/After（1.5分）

「従来は"やってみて終わり"でしたが、習慣化のステップを踏むことで、業務改善が継続するようになります。

**Before：作りっぱなし（改善なし）**
- 初版で満足 → そのまま放置
- 不具合発見 → 修正せず回避
- 新しいニーズ → 対応できない
- 結果：徐々に使われなくなる

**After：改善サイクル（成長し続ける）**
- 週1振り返り → 問題点を発見
- 月1改善 → 機能追加・修正
- ユーザーフィードバック → 反映
- 結果：1年後には完璧なツールに！

この差が、本当に使えるツールになるかの分かれ道です！」

// 使用ログを自動記録
function logUsage(action, time, status) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('使用ログ');
  
  sheet.appendRow([
    new Date(),
    action,
    time,
    status,
    Session.getActiveUser().getEmail()
  ]);
  
  // エラーの場合は別途記録
  if (status === 'error') {
    logError(action);
  }
}

// 請求書作成時に自動実行
function createInvoice() {
  const start = new Date();

## 注意点・活用ポイント（1分）

「最初から完璧を目指さず、"使って直す"のマインドが大事です。

**改善を続けるコツ：**
- 小さな改善を積み重ねる
- ユーザーの声を聞く
- データで判断する

**よくある失敗：**
- 大規模改修を狙う → 挫折
- 感覚で改善 → 効果不明
- フィードバック無視 → ズレていく

**成功の秘訣：**
『完璧な初版より、改善し続ける仕組み』

**Remember：**
ソフトウェアは生き物。育てることで価値が生まれる！」
