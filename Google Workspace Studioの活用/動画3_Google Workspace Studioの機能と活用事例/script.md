## 2. スターターの種類と使い分け（1:00-3:00）

### 講師セリフ

「まず、スターター（エージェントを起動するきっかけ）の種類を整理しましょう。

**（画面：スターター一覧を表示）**

### イベントベースのスターター

特定のイベントが発生したときにエージェントを起動します。

**① Gmail：When I receive an email**
- 特定の人からのメール受信時
- 特定の件名のメール受信時
- ラベル付きメール受信時

**② Google Forms：When a form response is submitted**
- フォーム回答送信時
- 特定のフォームへの回答時

**③ Google Drive：When a file is uploaded**
- 特定フォルダへのファイルアップロード時
- 特定の種類のファイル（PDF、Docなど）

**④ Google Chat：When I'm mentioned**
- Chatスペースでメンションされたとき
- 特定のキーワードを含むメッセージ

**⑤ Google Calendar：Before a meeting starts**
- 会議開始の指定時間前（30分前など）

### スケジュールベースのスターター

定期的にエージェントを実行します。

**① 時間指定**
- 毎日午前8時
- 毎週月曜日午前9時

**② 頻度指定**
- 1時間ごと
- 毎日

**（画面：使い分けの表を表示）**

| ユースケース | 推奨スターター |
|-------------|---------------|
| メール通知 | イベント（受信時） |
| 定期レポート | スケジュール（毎日/毎週） |
| フォーム回答処理 | イベント（回答時） |
| ファイル処理 | イベント（アップロード時） |
| 会議準備 | イベント（会議前） |」

## 4. サードパーティ連携（5:00-6:30）

### 講師セリフ

「Studioは、Google Workspace内だけでなく、外部サービスとも連携できます。

**（画面：サードパーティ連携の図）**

### 連携可能なサービス

**プロジェクト管理**
- Asana
- Jira

**マーケティング**
- Mailchimp
- Salesforce

**その他**
- Slack（Webhook経由）
- Microsoft Teams（Webhook経由）

### 連携の仕組み

**（画面：連携設定画面）**

1. **事前構築されたステップ**：Asana、Jira等は、専用のステップが用意されています
2. **Webhook**：外部サービスにHTTPリクエストを送信
3. **Apps Script**：高度なカスタム統合

### 連携例

**Asana連携**：
- Gmailでタスク依頼メールを受信 → Geminiがタスクを抽出 → Asanaにタスク作成

**Jira連携**：
- フォーム回答（バグ報告）→ Geminiが内容を整理 → Jiraにチケット作成」

## 5. 実践的な活用事例（6:30-9:00）

### 講師セリフ

「それでは、実際のビジネスシーンで使える活用事例を4つ紹介します。

**（画面：4つの活用事例カード）**

### 事例1：フォーム回答の自動通知とスプレッドシート記録

**（画面：フロー図を表示）**

**シナリオ**：問い合わせフォームの回答を即座にチームに通知し、スプレッドシートに記録

**フロー**：
1. **スターター**：Formに回答が送信されたとき
2. **ステップ1**：Geminiで回答内容を要約・カテゴリ分類
3. **ステップ2**：Chatスペースに通知
4. **ステップ3**：Sheetsに記録

**効果**：手動通知が不要になり、対応漏れを防止

### 事例2：Driveファイルの自動要約と共有

**（画面：フロー図を表示）**

**シナリオ**：共有フォルダにアップロードされた資料を自動で要約してチームに通知

**フロー**：
1. **スターター**：Driveの特定フォルダにファイルがアップロードされたとき
2. **ステップ1**：Geminiでファイル内容を要約
3. **ステップ2**：Chatスペースに要約と共にファイルリンクを通知

**効果**：資料確認の時間を大幅削減、重要資料の見落とし防止

### 事例3：会議前の自動ブリーフィング

**（画面：フロー図を表示）**

**シナリオ**：会議30分前に、参加者情報と関連資料をGeminiが整理して通知

**フロー**：
1. **スターター**：会議開始30分前
2. **ステップ1**：Geminiが参加者リスト、議題、過去のメモを収集
3. **ステップ2**：要約を作成
4. **ステップ3**：Chatで通知

**効果**：会議準備時間をゼロに、生産的な会議を実現

### 事例4：サポート問い合わせのトリアージ

**（画面：フロー図を表示）**

**シナリオ**：サポートメールをGeminiが分析し、優先度と担当者を自動判定

**フロー**：
1. **スターター**：サポート用メールアドレスにメール受信
2. **ステップ1**：Geminiで内容分析、優先度判定、カテゴリ分類
3. **ステップ2**：担当者のChatスペースに通知
4. **ステップ3**：Sheetsにログ記録

**効果**：対応時間短縮、優先順位に基づく効率的な対応

**（画面：Kärcher社の事例）**

### 実績例：Kärcher社

清掃機器メーカーのKärcherは、Studioを使って製品企画ワークフローを自動化。複数のエージェントが：
- ブレインストーミング
- フィージビリティ分析
- ユーザー体験検証
- ドキュメント作成

を担当し、従来**数時間かかっていた作業を2分に短縮**、**作業時間を90%削減**しました。」
