## スターターの種類と使い分け

### イベントベース

| サービス | トリガー内容 |
|---------|-------------|
| Gmail | 特定の人からのメール受信時、特定の件名のメール受信時 |
| Google Forms | フォーム回答送信時、特定のフォームへの回答時 |
| Google Drive | 特定フォルダへのファイルアップロード時 |
| Google Chat | Chatスペースでメンションされたとき |
| Google Calendar | 会議開始の指定時間前（30分前など） |

### スケジュールベース

| タイプ | 例 |
|--------|-----|
| 時間指定 | 毎日午前8時、毎週月曜日午前9時など |
| 頻度指定 | 1時間ごと、毎日など |

### ユースケース別の推奨スターター

| ユースケース | 推奨スターター | 理由 |
|-------------|---------------|------|
| 緊急メール通知 | イベント（メール受信時） | 即座に対応が必要なため |
| 定期レポート | スケジュール（毎日/毎週） | 定期的な集計・報告のため |
| フォーム回答処理 | イベント（回答時） | リアルタイム対応が求められるため |
| ファイル要約 | イベント（アップロード時） | 新規ファイルを即座に処理するため |
| 会議準備 | イベント（会議前） | 会議に間に合うタイミングで通知 |

## AIスキルとツール：Studioの真の強み

### Studioの差別化ポイント
Chatスペース投稿やWebhookが削除されましたが、Studioの真の価値は**AIスキル**と**ツール**にあります。これらを組み合わせることで、他のノーコードツールでは実現できない高度な自動化が可能です。

---

## AIスキル（AI Skills）

特定のタスクに最適化されたAI機能。「Geminiに聞く」よりも高精度・高速に処理できます。

### 1. 未読メールを要約する（Recap unread emails）
指定期間の未読メールを自動で要約。送信者、件名、重要ポイントを抽出してリスト化。毎朝の受信トレイチェックを数秒に短縮。

**活用例**：毎朝9時に「過去24時間の未読メール」を要約→DM通知

### 2. 抽出する（Extract）
テキストから特定の情報を抽出。日付、金額、名前、アクションアイテムなど、構造化データとして取り出せます。

**活用例**：請求書メールから「金額」「支払期日」「取引先名」を抽出→Sheetsに自動記録

### 3. 決める（Decide）
AI判断による自動分岐。「緊急/通常/低優先」「承認/要確認/却下」など、定義した基準でコンテンツを分類・判定します。

**活用例**：問い合わせメールを「クレーム/質問/依頼」に分類→それぞれ異なるラベルを付与

### 4. 要約（Summarize）
ドキュメント、メールスレッド、長文テキストを簡潔に要約。箇条書き、段落、指定文字数など、出力形式を柔軟に設定可能。

**活用例**：Driveにアップされた報告書を3行で要約→DM通知

---

## ツール（Tools）

データの絞り込みや条件分岐を行うための機能。AIスキルと組み合わせて使います。

### A. チェックする（Check）
条件に基づいてYES/NO判定。「金額が10万円以上か」「期限が今週中か」など、明確な条件でフローを分岐させます。

**組み合わせ例**：「抽出する」で金額取得→「チェックする」で10万円以上か判定→高額のみラベル付与

### B. フィルター（Filter）
複数のアイテムから条件に合うものだけを絞り込み。「未完了のタスクのみ」「特定ラベルのメールのみ」など、処理対象を限定。

**組み合わせ例**：「未読メールを要約」→「フィルター」で重要顧客からのみ抽出→要約をDM通知

## AIエージェント：Geminiに聞く

上記のAIスキルで対応できない複雑なタスクには「Geminiに聞く」を使用。自然言語でプロンプトを書いて、柔軟な処理が可能です。

### 効果的なプロンプトのコツ

- **具体的に指示**：「要約して」→「3つの箇条書きで要約して」
- **出力形式を指定**：「以下の形式で出力して」
- **言語を指定**：「日本語で回答して」または「Please respond in Japanese.」
- **条件を明確に**：「緊急のものは先頭に」

### AIスキル vs Geminiに聞く：使い分け

- **AIスキルを優先**：「未読メール要約」「抽出」「決める」「要約」で対応できるならAIスキルを使用（高速・高精度）
- **Geminiに聞くを使う場面**：複雑な判断、カスタムフォーマット出力、複数タスクの組み合わせが必要な場合

## 実践的な活用事例4選

### 事例1：フォーム回答の自動通知とスプレッドシート記録

**シナリオ**：問い合わせフォームの回答を即座にチームに通知し、スプレッドシートに記録

```
Formに回答送信 → Geminiで要約・分類 → Chatに通知 → Sheetsに記録
```

**効果**：手動通知が不要になり、対応漏れを防止

---

### 事例2：Driveファイルの自動要約と共有

**シナリオ**：共有フォルダにアップロードされた資料を自動で要約してチームに通知

```
Driveにファイルアップロード → Geminiで要約 → Chatに要約+リンク通知
```

**効果**：資料確認の時間を大幅削減、重要資料の見落とし防止

---

### 事例3：会議前の自動ブリーフィング

**シナリオ**：会議30分前に、参加者情報と関連資料をGeminiが整理して通知

```
会議30分前 → 参加者・議題を収集 → Geminiで要約 → Chatに通知
```

**効果**：会議準備時間をゼロに、生産的な会議を実現

---

### 事例4：サポート問い合わせのトリアージ

**シナリオ**：サポートメールをGeminiが分析し、優先度と担当者を自動判定

```
サポートメール受信 → Geminiで分析・優先度判定 → 担当者Chatに通知 → Sheetsにログ
```

**効果**：対応時間短縮、優先順位に基づく効率的な対応

---

**90%** - Karcher社の作業時間削減実績（従来数時間の作業が2分に短縮）
