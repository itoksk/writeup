# AIフロー - AI研修教材作成システム

## 概要

このリポジトリは、AI活用研修の教材を効率的に作成・管理するためのシステムです。原案から完成した研修資料まで、一貫した品質とデザインで教材を制作できます。

## 目的

- AI研修の講座資料を統一されたフォーマットで作成
- 原案（script.md）から視覚的な教材（materials.html）への変換
- 実践的なサンプルとガイドの一元管理
- 講師が使いやすい台本と受講者が理解しやすい教材の両立

## デザインシステム

### literary-editorial-template

すべてのHTML教材は`literary-editorial-template`のデザインシステムに基づいて作成されます：

- **カラーパレット**
  - Primary (Ink): #0A0E27
  - Secondary (Paper): #FEFEF8  
  - Accent (Electric): #00F0FF
  - Warning (Coral): #FF6B6B

- **特徴**
  - モノクロームベースの洗練されたデザイン
  - 視認性の高いタイポグラフィ
  - スムーズなアニメーション
  - レスポンシブ対応
  - ダークモード切り替え機能

## ワークフロー

### 原案作成の進め方

1. **基本フロー**: プロンプト2.md からスタート
2. **品質改善が必要な場合**: プロンプト1.md から開始
3. **オプション**: 
   - プロンプト3.md（追加の構成調整が必要な場合）
   - スライド作成プロンプト.md（スライド形式の資料が必要な場合）

### 教材作成フロー

```
1. 原案作成（上記プロンプトを使用）
   ↓
2. script.md 作成（講師用台本）
   ↓
3. materials.html 生成（視覚教材）
   ↓
4. sample_prompts.md 作成（実践例）
   ↓
5. setup_guide.md 作成（環境構築）
```

## ファイル構成

### 各動画ディレクトリの標準構成

```
動画X_タイトル/
├── script.md           # 講師用台本（10分構成）
├── materials.html      # インタラクティブ教材
├── sample_prompts.md   # 実践用プロンプト集
├── setup_guide.md      # 環境構築・設定ガイド
└── demo_prompts/       # デモ用ファイル（必要に応じて）
```

### ファイルの役割

| ファイル | 用途 | 対象者 | 内容 |
|---------|------|--------|------|
| script.md | 講義進行 | 講師 | タイムライン、セリフ、画面操作指示 |
| materials.html | 視覚教材 | 受講者・講師 | スライド形式のインタラクティブ教材 |
| sample_prompts.md | 実践演習 | 受講者 | コピペで使えるプロンプト例 |
| setup_guide.md | 環境構築 | 受講者 | ツールの設定手順、API取得方法 |

## 教材作成ガイドライン
1. このカリキュラムで伝えたい本質を理解して、各動画での整合性を確認してください。\
2. 改善が必要な場合は、改善してください。\
3. 各ディレクトリ内に、これを受講者に説明できるWebサイト形式の資料を作成してください。\
　- [ ] 資料は、コンポーネントを分けてなくても良い。html形式のファイル一つで、それぞれの動画ごとで作っておいてください


1. このカリキュラムに沿って、動画の準備として、動画ごとのディレクトリを作成、
2. 動画ごとのセリフと流れの作成、
3, 受講者に説明しながら見せるためのwebサイト資料をhtml1つのファイルで。
4. デモで使うコードやプロンプトを、各動画ごとのディレクトリに作成してください.
5. 不可能な実装や過大な実装、10分動画での解説に不向きな内容は排除します。
6. 次の動画構成を守ってください。
7. ノーコードやAIの使い方を学んでもらう動画を作ります。動画の解説時にデモで実践するためのコード、プロンプトを作成してください。
8. 動画で説明する流れは、ナレッジの構成を参考に、セリフも同時に作ってください。
9. これを見て学人は、初級から中級者です。
10. 私の得意とするのは、Gemini, ChatGPT,Claude,ノーコードツール（AppSheet,Dify,Make）、Google Apps Script です。
11. 必ずしも、Google 完結型にする必要はありません。

### script.md の構成

```markdown
# 動画X：タイトル

## 講師用台本（全10分）

### 【0:00-1:00】導入
- フック（問題提起）
- 今日学ぶことの概要

### 【1:00-X:00】デモ・実演
- 画面共有での実演
- ステップバイステップの説明

### 【X:00-9:00】構築・設定
- 実際の手順説明
- 注意点の解説

### 【9:00-10:00】まとめ・次回予告
- 重要ポイントの振り返り
- 次回への期待感醸成
```

### materials.html の構成
literary-editorial-template/design-system.yaml
```html
<!-- literary-editorial-templateベース -->
<section class="content-section">
    <h2 class="section-title">セクションタイトル</h2>
    <div class="section-content">
        <!-- コンテンツ -->
    </div>
</section>
```

#### 必須要素
- ナビゲーション（右側の縦並びドット）
- セクション単位のスクロール
- レスポンシブ対応
- code-blockでのプロンプト表示（white-space: pre-wrap必須）

### 品質チェックリスト

- [ ] script.mdは10分で完結する構成か
- [ ] materials.htmlはダークモード対応か
- [ ] code-block内の改行は正しく表示されるか
- [ ] code-blockのコードは、コピーしやすいボタンがついているか（Google サイトに埋め込み対策済み）
- [ ] APIキー関連の説明は含まれているか（該当する場合）
- [ ] sample_prompts.mdはコピペですぐ使えるか
- [ ] setup_guide.mdは初心者でも理解できるか

## 技術仕様

### HTML/CSS
- CSS Variables使用によるテーマ管理
- Flexbox/Gridによるレイアウト

## メンテナンス

### 更新時の注意点

   - 料金体系の変更を確認

3. **新規教材追加時**

## トラブルシューティング

### よくある問題

| 問題 | 原因 | 解決方法 |
|------|------|----------|
| プロンプトの改行が表示されない | CSSの設定不足 | `white-space: pre-wrap`を追加 |
| ダークモードが効かない | CSS変数の未定義 | CSS Variablesを確認 |
| APIキーエラー | 設定ミス | setup_guide.mdを参照 |
| スクロールがスムーズでない | JS未読み込み | スクリプトの読み込み順を確認 |
