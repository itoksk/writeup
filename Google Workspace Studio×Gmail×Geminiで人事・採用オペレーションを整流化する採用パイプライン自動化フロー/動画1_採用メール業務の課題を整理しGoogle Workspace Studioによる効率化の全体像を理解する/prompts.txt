## 2. 事前準備：選考ステージ定義

### ステージ一覧
```
【応募受付】
- 応募メール受信
- 書類確認中

【書類選考】
- 一次スクリーニング完了
- 詳細レビュー中
- 書類選考通過/不通過

【一次面接】
- 日程調整中
- 面接予定
- 面接完了
- 一次通過/不通過

【二次面接】
- 日程調整中
- 面接予定
- 面接完了
- 二次通過/不通過

【最終面接】
- 日程調整中
- 面接予定
- 面接完了

【内定】
- 内定通知済み
- 内定承諾
- 内定辞退

【不採用】
- 書類不採用
- 面接不採用
```

## 3. 次回以降で使用するプロンプト（プレビュー）

### 第2回：応募者スクリーニングプロンプト
```
Screen this job application and evaluate fit:

Application content:
{email_content}

Job requirements:
{job_requirements}

Provide evaluation in Japanese:
【適合度】HIGH/MEDIUM/LOW
【マッチするスキル】
【不足スキル】
【推奨アクション】
```

### 第3回：面接日程調整プロンプト
```
Generate interview scheduling email:

Candidate: {candidate_name}
Available slots: {available_times}
Interview type: {interview_type}

Provide email draft in Japanese.
```

### 第4回：履歴書サマリ作成プロンプト
```
Summarize this resume:

{resume_content}

Provide summary in Japanese:
【氏名】
【連絡先】
【経験年数】
【主なスキル】
【直近の職歴】
【強み】
```

### 第5回：採用KPIレポートプロンプト
```
Create weekly recruitment report:

{selection_data}

Format in Japanese:
【今週の応募数】
【選考中】
【内定】
【辞退/不採用】
【平均選考日数】
```

## 4. Studio アクセス確認

### アクセスURL
```
studio.workspace.google.com
```

### 確認手順
1. 上記URLにアクセス
2. Google Workspaceアカウントでサインイン
3. 「Discover」ページが表示されればOK
4. アクセスできない場合は管理者に問い合わせ

## 5. 講座全体の学習ポイント

### 第1回（本講座）
- 採用メール業務を5つの要素に分解
- 各要素の自動化可能性を理解
- 事前準備の重要性を認識

### 第2回
- 応募メールの自動スクリーニング
- 募集要件との適合度評価
- 優先候補のアラート

### 第3回
- Calendar連携
- 面接日程の自動調整
- 候補者への日程提案

### 第4回
- Drive連携
- 履歴書・職務経歴書の自動保存
- サマリ自動作成

### 第5回
- Sheets連携
- 選考ステータス自動更新
- 週次KPIレポート
