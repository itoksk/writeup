# 第4回：履歴書・職務経歴書を自動整理し、Drive保存＆サマリ作成で選考を高速化

**動画時間**: 10分
**学習目標**: 応募メールの添付ファイルをGoogle Driveに自動保存し、Geminiでサマリを作成するエージェントを構築する

---

## 1. 導入（0:00-1:00）

### 講師セリフ

「皆さん、こんにちは。採用パイプライン自動化フロー、第4回をお届けします。

前回は、Calendar連携で面接日程を自動調整するエージェントを作成しました。

今回は、採用担当者のもう一つの悩み、**採用書類の管理**を自動化します。

**（画面：散らばったファイルのイメージ）**

応募メールには履歴書や職務経歴書が添付されています。

- メールを開いて添付ファイルをダウンロード
- Driveの適切なフォルダに保存
- ファイル名を候補者名に変更
- 書類の内容を確認して要約...

1件あたり5〜10分。月に50人の応募があれば、それだけで4〜8時間。

**（画面：自動化後のイメージ）**

このエージェントは、添付ファイルを自動でDriveに保存し、候補者ごとのフォルダに整理。さらにGeminiが書類の要約を自動作成してくれます。」

---

## 2. エージェントの全体像（1:00-2:00）

### 講師セリフ

「今回作るエージェントの構成を確認しましょう。

**（画面：エージェントフロー図）**

### フロー

```
[応募メール受信] → [添付ファイル取得] → [Drive保存] → [Geminiサマリ作成] → [スプレッドシート記録]
```

### 機能

1. **トリガー**：応募メール受信時に自動起動
2. **Drive連携**：添付ファイルを候補者フォルダに自動保存
3. **Gemini生成**：履歴書・職務経歴書のサマリを自動作成
4. **スプレッドシート**：候補者情報とサマリを一覧に追記」

---

## 3. Drive連携の設定（2:00-4:00）

### 講師セリフ

「新しいエージェントを作成します。

**（画面：Create → 新規エージェント）**

エージェント名：『採用書類自動整理エージェント』

### スターター設定

**（画面：Starter設定）**

『Gmail』→『When labeled email arrives』を選択します。

- **Label**: 「応募_新規」
- **Has attachment**: ON

応募メールに「応募_新規」ラベルが付き、添付ファイルがある場合に起動します。

### Drive保存設定

**（画面：Add Step → Google Drive）**

『Google Drive』→『Upload file』を選択します。

**（画面：Drive設定）**

- **Source**: Email attachment
- **Folder**: 採用/候補者/{candidate_name}
- **File name**: {candidate_name}_{date}_{original_filename}

これで、候補者ごとのフォルダに自動保存されます。

フォルダが存在しない場合は自動作成されるので、事前準備は不要です。」

---

## 4. Geminiでサマリ作成（4:00-6:00）

### 講師セリフ

「次に、保存した書類からサマリを作成します。

**（画面：Add Step → Gemini）**

### プロンプト

```
Analyze the attached resume and create a summary in Japanese:

Document: {uploaded_file}
Position applied: {position}

Create a structured summary including:
1. Basic Info (name, contact, education)
2. Work Experience (companies, years, roles)
3. Key Skills (technical skills, certifications)
4. Relevance to Position (match level: High/Medium/Low)
5. Notable Points (strengths and concerns)

Keep the summary concise (within 300 characters).
```

**（画面：出力例）**

```
【候補者サマリ】
■ 基本情報
氏名：山田太郎
学歴：〇〇大学 工学部 2018年卒

■ 職務経歴
株式会社ABC（2018-2022）：Webエンジニア
株式会社DEF（2022-現在）：シニアエンジニア

■ スキル
言語：JavaScript, Python, Go
資格：AWS SAA, 基本情報

■ 適合度：High
募集要件のフロントエンド経験3年以上を満たし、
バックエンド経験もあり即戦力として期待。

■ 特記事項
・チームリーダー経験あり
・転職回数：1回（標準的）
```」

---

## 5. スプレッドシートへの記録（6:00-8:00）

### 講師セリフ

「サマリをスプレッドシートに記録して、候補者を一覧管理できるようにします。

**（画面：Add Step → Google Sheets）**

### Sheets設定

- **Action**: Append row（行を追加）
- **Spreadsheet**: 採用管理シート
- **Sheet**: 候補者一覧

### 記録する列

| 列 | 内容 |
|---|---|
| A | 応募日 |
| B | 候補者名 |
| C | 応募職種 |
| D | メールアドレス |
| E | 書類リンク（Drive） |
| F | サマリ |
| G | 適合度 |
| H | ステータス |

**（画面：スプレッドシートのイメージ）**

これで、全候補者の情報がスプレッドシートで一覧できます。

書類を開かなくても、サマリと適合度で優先度を判断できますね。」

---

## 6. テストと有効化（8:00-9:00）

### 講師セリフ

「エージェントをテストしましょう。

**（画面：テスト実行）**

1. テスト用の応募メールを用意（履歴書添付）
2. 「応募_新規」ラベルを付与
3. エージェントが起動するか確認
4. Driveに正しく保存されるか
5. サマリが正しく生成されるか
6. スプレッドシートに追記されるか

**（画面：テスト結果確認）**

問題なければ『Enable』で有効化します。」
