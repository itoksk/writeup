# 第2回：応募者スクリーニングを自動化し、一次評価と優先対応候補を即座に抽出する

**動画時間**: 10分
**学習目標**: 応募メールを自動でスクリーニングし、募集要件との適合度を評価、優先候補をアラート通知するエージェントを作成する

---

## 1. 導入（0:00-1:00）

### 講師セリフ

「皆さん、こんにちは。採用パイプライン自動化フロー、第2回をお届けします。

前回は、採用メール業務の課題と、Studioによる自動化の全体像を学びました。

今回から実際にStudioにアクセスして、最初のエージェントを作成します。

**（画面：Studio画面を表示）**

今日作るのは『応募者スクリーニングエージェント』です。

応募メールをGeminiが自動で分析し、募集要件との適合度を評価。優秀な候補者は即座にアラート通知してくれます。

これで、1日1時間かけていたスクリーニングが数分に短縮され、優秀人材への対応スピードが大幅に向上します。」

---

## 2. Studioにアクセスする（1:00-2:00）

### 講師セリフ

「まず、Studioにアクセスしましょう。

**（画面：ブラウザでURLを入力）**

ブラウザで `studio.workspace.google.com` にアクセスします。

**（画面：Discoverページ）**

右上の『Create』をクリックして、新しいエージェントを作成します。

エージェント名を入力します。『応募者スクリーニングエージェント』としましょう。」

---

## 3. スターター設定（2:00-3:00）

### 講師セリフ

「エージェントの最初の設定は『スターター』です。

**（画面：Starterメニュー）**

応募メールを受信したら即座に起動したいので、『Gmail - Email received』を選択します。

**（画面：フィルター設定）**

フィルターで採用関連メールに絞り込みます：

```
subject:(応募 OR エントリー OR 採用) OR from:(recruit OR career)
```

または、採用専用メールアドレス宛のメールのみを対象にすることもできます。」

---

## 4. Geminiでスクリーニング（3:00-5:30）

### 講師セリフ

「次に、応募メールをGeminiで分析します。

**（画面：Add Step → Gemini）**

『Add Step』から『Gemini』を選択します。

**（画面：プロンプト入力）**

### プロンプト

```
Screen this job application and evaluate fit against requirements:

Application:
{email_content}

Job Requirements:
- Required: 3+ years software development, Python or Java
- Preferred: Cloud experience (AWS/GCP), Team leadership
- NG: No work authorization

Provide evaluation in Japanese:

【候補者名】Name from email
【適合度】HIGH/MEDIUM/LOW
【評価スコア】1-10
【マッチする要件】
【不足している要件】
【強み】
【懸念点】
【推奨アクション】書類通過/追加確認/不採用
【優先度】1-5（1が最優先）
【サマリ】2-3文の要約
```

**（画面：変数確認）**

`{email_content}` には応募メールの全文が入ります。Job Requirementsの部分は、実際の募集要件に書き換えてください。」

---

## 5. 条件分岐とアラート（5:30-7:30）

### 講師セリフ

「Geminiの評価結果に基づいて、条件分岐を設定します。

**（画面：Add Step → Conditional）**

### 条件1：適合度HIGH

```
{gemini_evaluation} contains "HIGH"
```

True → Chat通知（優先アラート）：

```
🎯 優秀候補者を検知しました！

{gemini_evaluation}

📧 応募メール: {email_link}

⚡ 優先的に対応してください！
```

**（画面：ラベル付与）**

同時にGmailラベル「採用_優先候補」を付与します。

### 条件2：適合度MEDIUM

通常のChat通知と「採用_検討中」ラベルを付与。

### 条件3：適合度LOW

「採用_不採用候補」ラベルを付与（通知なし）。」

---

## 6. 返信テンプレート生成（7:30-8:30）

### 講師セリフ

「さらに便利な機能として、返信テンプレートを自動生成させましょう。

**（画面：追加ステップ → Gemini）**

適合度HIGHの場合に、面接案内の返信テンプレートを生成：

```
Generate a professional interview invitation email in Japanese:

Candidate: {candidate_name}
Position: {position}
Evaluation: {gemini_evaluation}

Include:
- Thank for application
- Positive evaluation mention
- Request for interview availability
- Next steps
```

**（画面：下書き保存）**

生成された返信は、Gmailの下書きとして保存することもできます。」

---

## 7. テストと有効化（8:30-9:30）

### 講師セリフ

「エージェントが完成しました。テストしましょう。

**（画面：テスト実行）**

サンプルの応募メールでテストします。

**（画面：結果確認）**

- Geminiが正しくスクリーニングしているか
- 適合度が正しく判定されているか
- Chatに通知が届いているか
- ラベルが付与されているか

問題なければ『Enable』で有効化します。」

---

## 8. まとめ・次回予告（9:30-10:00）

### 講師セリフ

「今日は、応募者スクリーニングエージェントを作成しました。

### 今日のポイント

1. **Gmail受信トリガー**で応募メールを即座に検知
2. **Geminiでスクリーニング**：募集要件との適合度を自動評価
3. **条件分岐**：適合度に応じてラベル・通知を分岐
4. **返信テンプレート**：面接案内を自動生成

### Before / After

| 項目 | Before | After |
|------|--------|-------|
| スクリーニング時間 | 1時間/日 | 数分 |
| 優秀候補への対応 | 2-3日後 | 即座にアラート |
| 評価の一貫性 | 担当者により異なる | Geminiで標準化 |

### 次回予告

次回：「面接日程調整の手間をゼロにし、Calendar連携で候補者対応を自動化する」

それでは、次回もお楽しみに！」

---

## 講師への補足指示

- **募集要件のカスタマイズ**：実際の募集要件に書き換える方法を強調
- **プライバシー配慮**：候補者情報の取り扱いに関する注意を補足
- **評価基準の調整**：適合度判定の基準はプロンプトで調整可能と説明
