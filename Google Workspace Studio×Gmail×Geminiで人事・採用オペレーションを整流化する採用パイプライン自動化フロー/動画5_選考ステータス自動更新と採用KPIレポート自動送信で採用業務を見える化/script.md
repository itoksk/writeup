# 第5回：選考ステータス自動更新と採用KPIレポート自動送信で採用業務を見える化

**動画時間**: 10分
**学習目標**: 選考ステータスの自動更新と週次KPIレポートを自動送信するエージェントを作成し、採用業務全体を可視化する

---

## 1. 導入（0:00-1:00）

### 講師セリフ

「皆さん、こんにちは。採用パイプライン自動化フロー、最終回の第5回をお届けします。

前回は、履歴書・職務経歴書を自動整理し、Driveに保存、サマリを作成するエージェントを構築しました。

最終回の今回は、**採用業務全体を見える化**するエージェントを作ります。

**（画面：ステータス管理の課題イメージ）**

採用業務で困るのは、『今、どの候補者がどのステージにいるのか』がわからなくなること。

- スプレッドシートを更新し忘れる
- 面接結果の反映が遅れる
- 採用KPIが把握できない
- 上司への報告資料を毎週手作り...

**（画面：自動化後のイメージ）**

このエージェントは、選考ステータスを自動更新し、週次でKPIレポートをChatに自動送信。採用業務全体がリアルタイムで見える化されます。」

---

## 2. エージェントの全体像（1:00-2:00）

### 講師セリフ

「今回は2つのエージェントを作成します。

**（画面：エージェントフロー図）**

### エージェント1：ステータス自動更新

```
[面接完了メール] → [ステータス抽出] → [Sheets更新]
```

### エージェント2：週次KPIレポート

```
[週次トリガー] → [Sheets集計] → [Geminiレポート生成] → [Chat送信]
```

この2つで、採用プロセスの入口から出口まで自動管理できます。」

---

## 3. ステータス自動更新エージェント（2:00-4:30）

### 講師セリフ

「まず、ステータス自動更新エージェントを作成します。

**（画面：Create → 新規エージェント）**

エージェント名：『選考ステータス自動更新エージェント』

### スターター設定

**（画面：Starter設定）**

『Gmail』→『When email arrives matching search』

検索クエリ：`subject:(面接結果) OR subject:(選考結果)`

面接結果や選考結果のメールを検知します。

### Geminiでステータス抽出

**（画面：Add Step → Gemini）**

```
Extract status update from this email:

Email content: {email_body}

Extract:
1. Candidate name
2. New status (one of: 書類選考中/一次面接済/二次面接済/最終面接済/内定/不採用/辞退)
3. Date of status change
4. Notes (if any)

Output as JSON format.
```

### Sheets更新

**（画面：Add Step → Google Sheets）**

- **Action**: Update row（行を更新）
- **Spreadsheet**: 採用管理シート
- **Match column**: 候補者名
- **Update column**: ステータス、更新日

これで、メールが来るたびにスプレッドシートが自動更新されます。」

---

## 4. 週次KPIレポートエージェント（4:30-7:00）

### 講師セリフ

「次に、週次KPIレポートエージェントを作成します。

**（画面：Create → 新規エージェント）**

エージェント名：『採用KPIレポート自動送信エージェント』

### スターター設定

**（画面：Starter設定）**

『Schedule』→ 毎週月曜 9:00

週次でスケジュール実行します。

### Sheetsからデータ取得

**（画面：Add Step → Google Sheets）**

- **Action**: Get data
- **Spreadsheet**: 採用管理シート
- **Range**: 全データ

### GeminiでKPI集計・レポート生成

**（画面：Add Step → Gemini）**

```
Analyze recruitment data and create a KPI report in Japanese:

Data: {sheet_data}
Report period: Past week

Calculate and report:
1. New applications this week
2. Pipeline by status (count per stage)
3. Conversion rates (application→interview, interview→offer)
4. Average time per stage
5. Top concerns or bottlenecks

Format as a professional weekly report.
Include specific numbers and percentages.
```

**（画面：出力例）**

```
【採用KPIレポート】2025年1月第3週

■ 今週の応募
新規応募：12件（前週比 +3件）
応募経路：Indeed 5件、LinkedIn 4件、直接応募 3件

■ パイプライン状況
├ 書類選考中：8名
├ 一次面接済：5名
├ 二次面接済：3名
├ 最終面接済：2名
├ 内定：1名
└ 合計アクティブ：19名

■ 転換率
書類通過率：75%（9/12）
一次通過率：60%（3/5）
内定承諾率：100%（1/1）

■ 平均所要日数
書類→一次：3.2日
一次→二次：5.1日

■ 注目ポイント
・エンジニア職の応募が増加傾向
・一次→二次の移行に時間がかかっている
```

### Google Chatに送信

**（画面：Add Step → Google Chat）**

- **Space**: 採用チームスペース
- **Message**: {kpi_report}」

---

## 5. テストと有効化（7:00-8:30）

### 講師セリフ

「両方のエージェントをテストしましょう。

**（画面：テスト実行）**

### ステータス更新エージェント

1. テスト用の面接結果メールを送信
2. ステータスが正しく抽出されるか
3. スプレッドシートが更新されるか

### KPIレポートエージェント

1. 手動でテスト実行
2. データが正しく集計されるか
3. レポートが適切な形式か
4. Chatに正しく送信されるか

**（画面：テスト結果確認）**

問題なければ両方を『Enable』で有効化します。」

---

## 6. 講座全体のまとめ（8:30-10:00）

### 講師セリフ

「お疲れさまでした！これで採用パイプライン自動化フローの全5回が完了です。

### 作成した4つのエージェント

| 回 | エージェント | 効果 |
|---|---|---|
| 2 | 応募者スクリーニング | 評価時間80%削減 |
| 3 | 面接日程自動調整 | 調整時間90%削減 |
| 4 | 採用書類自動整理 | 探す時間ゼロ |
| 5 | KPIレポート自動送信 | レポート作成自動化 |

### 採用業務全体の変化

**Before**：
- 応募確認：1時間/日
- 書類整理：30分/件
- 日程調整：20分/件
- レポート作成：2時間/週

**After**：
- 応募確認：10分/日（サマリ確認のみ）
- 書類整理：0分（自動）
- 日程調整：2分/件（確認のみ）
- レポート作成：0分（自動）

**月間削減時間**：約30時間

### 次のステップ

1. 各エージェントを自社の採用フローに合わせてカスタマイズ
2. ラベルやフォルダ名を実運用に合わせて調整
3. 他の採用業務（オファー作成、入社手続き等）への展開

Google Workspace Studioを使えば、採用業務のほぼすべてを自動化できます。

ぜひ実際に作って、採用業務を効率化してください。

ありがとうございました！」
