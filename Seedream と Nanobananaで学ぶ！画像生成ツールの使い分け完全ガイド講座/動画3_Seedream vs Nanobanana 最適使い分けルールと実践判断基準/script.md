## 構築手順（全体像）（4:30 - 5:30）

**【画面】**: 使い分けチャートを表示

**【講師セリフ】**:
```
使い分けのルールを体系化するために、
「判断マトリクス」を作成しました。

【判断の基準は4つ】

1. **目的**: 何のために使うか？
   - 商品紹介、プレゼン、SNS、ブランディング など

2. **求める要素**: 何を重視するか？
   - リアルさ、個性、印象、スタイル性 など

3. **対象物**: 何を描くか？
   - 商品、人物、風景、キャラクター など

4. **用途**: どこで使うか？
   - ECサイト、プレゼン資料、SNS、印刷物 など

この4つの基準を組み合わせて、
「Seedream」か「Nanobanana」かを判断します。

【判断フロー】

ステップ1：「リアルさ」が必要か？
→ YES → Seedream
→ NO → ステップ2へ

ステップ2：「個性・スタイル性」が必要か？
→ YES → Nanobanana
→ NO → Seedream（汎用）

ステップ3：迷った場合は？
→ 両方で試して比較（時間に余裕があれば）
→ または、デフォルトでSeedreamから試す

この3ステップで、ほとんどの場面で適切な選択ができます。

では、具体的な事例を見ていきましょう。
```

## 設定実演（詳細操作）（5:30 - 7:30）

**【画面】**: 使い分けチャート（Excel）を画面共有

**【講師セリフ】**:
```
それでは、実際の「使い分けチャート」を使って、
判断方法を学んでいきましょう。

【画面操作：Excelファイルを開く】

このExcelファイルは、今回の教材に含まれています。
ダウンロードして、ご自身でも使ってみてください。

【シート1：基本的な使い分け表】

この表では、用途別に推奨ツールが一覧になっています。

例えば：
- 商品写真 → Seedream（リアルさが必要）
- キャラクターイラスト → Nanobanana（表現力が必要）
- 風景写真 → Seedream（写実的）
- コンセプトアート → Nanobanana（スタイル性）

このように、用途ごとに「第一選択」が明記されています。

【シート2：詳細判断フロー】

こちらは、より詳細な判断フローです。

質問形式になっていて、YESかNOで答えていくと、
最適なツールが決まる仕組みです。

例えば：
Q1: リアルな質感が必要ですか？
→ YES → Seedream
→ NO → Q2へ

Q2: アーティスティックな表現が必要ですか？
→ YES → Nanobanana
→ NO → Seedream（汎用）

このフローに沿って判断すれば、
迷うことなく適切なツールを選べます。

【シート3：場面別プロンプト例】

こちらは、場面別のプロンプト例が載っています。

各用途ごとに：
- Seedream用プロンプト
- Nanobanana用プロンプト

が記載されているので、参考にしてください。

【実践：実際に判断してみる】

では、いくつかの例で、実際に判断してみましょう。

【例1：企業サイトのヘッダー画像】
→ 目的：企業イメージ、プロフェッショナル
→ リアルさ：必要
→ 判断：Seedream

【例2：SNS投稿用のキャラクターイラスト】
→ 目的：目を引く、個性的
→ スタイル性：必要
→ 判断：Nanobanana

【例3：商品カタログの写真】
→ 目的：商品の正確な紹介
→ リアルさ：必要
→ 判断：Seedream

【例4：ブランドイメージのビジュアル】
→ 目的：ブランドの世界観表現
→ 個性・印象：必要
→ 判断：Nanobanana

このように、チャートに沿って判断すれば、
誰でも適切なツールを選択できます。
```

## 注意点・活用ポイント（7:30 - 8:30）

**【画面】**: 注意点のスライド表示

**【講師セリフ】**:
```
ここで、使い分けの際の注意点と応用テクニックをお伝えします。

【注意点1：境界が曖昧な場面】
すべての場面で明確に分けられるわけではありません。

例えば、「プレゼン資料用のイラスト」は：
- リアルさも欲しいが、印象も大切
- 中間的な表現が求められる

こういう場合は：
- 両方で試してみる（時間に余裕があれば）
- チーム内で相談して決める
- デフォルトでSeedreamから試す

【注意点2：ハイブリッド利用】
実は、2つのツールを組み合わせることもできます。

例：
- Seedreamでベースを作成
- Nanobananaで雰囲気を追加
- または、逆のパターン

ただし、これは上級者向けなので、
まずは基本の使い分けをマスターしてからにしましょう。

【活用ポイント1：チーム内での運用ルール化】
この使い分けチャートを、チーム内で共有し、
運用ルールとして定着させることが重要です。

例えば：
- 新規プロジェクトの際、必ずチャートで確認
- 定期的に見直し、改善点を議論
- 成功事例・失敗事例を蓄積

【活用ポイント2：継続的な見直し】
使い分けルールは、固定的なものではありません。

定期的に見直し、改善しましょう：
- 3ヶ月に1回程度、チーム内で振り返り
- 新しい用途が出てきたら、チャートに追加
- ツールのアップデートに応じて調整

【活用ポイント3：社内ガイドライン化】
最終的には、社内の公式ガイドラインとして
整備することをおすすめします。

含めるべき内容：
- 使い分け判断フロー
- 用途別プロンプト例
- 失敗事例と対処法
- よくある質問（FAQ）

これにより、誰が使っても一定水準の成果が出せる
運用基盤が整います。
```
