## トラブルシューティング

### 問題1：出力が単調すぎる
```
症状：同じような表現ばかり
対策：Temperatureを0.1〜0.2上げる
```

### 問題2：出力が奇抜すぎる
```
症状：実用的でない提案
対策：Temperatureを0.2〜0.3下げる
```

### 問題3：選択肢が少ない
```
症状：バリエーション不足
対策：Top-Pを0.2〜0.3上げる
```

### 問題4：一貫性がない
```
症状：文脈がバラバラ
対策：Top-Pを0.2〜0.3下げる
```

## 最適化のコツ

### 1. 段階的調整
```
初期：Temperature=0.5, Top-P=0.5
↓
結果を見て微調整
↓
最適値を見つける
```

### 2. タスク特性の理解
- 創造的タスク → 高Temperature
- 分析的タスク → 低Temperature
- 多様性重視 → 高Top-P
- 一貫性重視 → 低Top-P

### 3. 複数回生成
同じ設定でも出力は変わるため、
重要なタスクは3〜5回生成して
最良のものを選ぶ
