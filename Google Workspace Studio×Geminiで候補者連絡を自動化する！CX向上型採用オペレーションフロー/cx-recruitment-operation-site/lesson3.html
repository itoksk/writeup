面接後作業をゼロにする！ 要約＆評価ポイントをGmail・Chatへ自動送信するフローを作る

面接終了をトリガーに、評価依頼の自動送信と次ステップの条件分岐処理を実装します。 Proceed / Hold / Decline の3パターンで自動処理を設計しましょう。

STEPS

エージェント作成手順

1

新規エージェント作成

Google Workspace Studio → 「Create agent」→ エージェント名「Post-Interview Workflow」とDescriptionを入力

2

Calendar イベントトリガー設定

「Add starter」→「Calendar event ends」を選択
Event filter: Title contains '面接' / Calendar: Primary calendar

3

Gmail 評価依頼送信

「Add step」→「Send Gmail」を選択 → 面接官へ評価シート記入依頼を送信
Calendarイベントの参加者情報から面接官メールアドレスを取得

4

フォーム回答トリガー

「Add trigger」→「Form response received」を選択
Form: 面接評価シート（Google Forms）を指定

5

Gemini 要約

「Add step」→「Generate with Gemini」→ 評価内容を構造化して要約
総合評価・強み・懸念点・推奨アクションを抽出

6

Chat 結果共有

「Add step」→「チャットで通知する」→ 自分へのDMで面接結果速報を受信 ※スペースへの投稿機能は削除されました

7

条件分岐設定

「Add condition」→ Geminiの判定結果（Proceed/Hold/Decline）に応じてアクションを分岐

BRANCHING

条件分岐ロジック

PROCEED

次に進む

→ 次の面接官にCalendar招待を自動作成
→ 候補者に次ステップ案内メール下書きを作成
→ 候補者管理シートのステータスを更新

HOLD

保留

→ 採用マネージャーに判断依頼通知を送信
→ 追加情報の収集を依頼
→ 候補者管理シートのステータスを「検討中」に更新

DECLINE

不採用

→ お見送りメールの下書きを作成
※送信は人間が確認
→ 候補者管理シートのステータスを更新

重要ポイント

最終判断は人間が行う設計です。エージェントは「準備」と「提案」を行い、実際の送信は人間が確認してから行います。特にお見送りメールは必ず人間がレビューしてから送信します。

CONDITION LOGIC

条件分岐の設定方法

IF RECOMMENDATION == "Proceed": → Create Calendar invite for next interviewer → Draft next-step email for candidate → Update Sheets status to "次面接予定" → Notify hiring manager via Chat ELSE IF RECOMMENDATION == "Hold": → Send decision request to hiring manager → Update Sheets status to "検討中" → Set reminder for 48h follow-up ELSE IF RECOMMENDATION == "Decline": → Draft rejection email (DO NOT SEND automatically) → Update Sheets status to "見送り予定" → Flag for human review

BEST PRACTICES

運用のポイント

POINT 01

評価シートの標準化

Geminiが正確に解析できるよう、評価シートの項目を統一。5段階評価と自由記述の組み合わせが推奨

POINT 02

記入期限の設定

面接翌日17時を期限とし、未記入の場合はリマインドを送信（第4回で設定）。CX向上のために迅速な対応を

POINT 03

プライバシー配慮

評価内容は採用チーム内のみで共有。DM通知で受け取った情報の取り扱いに注意すること

POINT 04

人間の最終判断

お見送りメールは下書きのみ作成し、送信は必ず人間が確認。誤送信を防ぐための重要なセーフガード

SUMMARY

今日のポイント
