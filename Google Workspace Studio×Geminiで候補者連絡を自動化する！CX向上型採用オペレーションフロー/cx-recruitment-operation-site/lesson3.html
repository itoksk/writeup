面接後作業をゼロにする！ 要約＆評価ポイントをGmail/Chatへ自動送信するフローを作る

面接終了をトリガーに、評価依頼の自動送信と次ステップの条件分岐処理を実装します。 Proceed / Hold / Decline の3パターンで自動処理を設計しましょう。

BEFORE：受講前
  • 面接後の評価依頼を手動で送信
  • 次ステップの判断に時間がかかる
  • 候補者への連絡が遅れがち
  • 面接後処理に毎回1時間以上
AFTER：受講後
  • 面接終了で評価依頼が自動送信
  • Proceed/Hold/Declineで自動分岐
  • 候補者へ即座に次ステップ連絡
  • 面接後作業がゼロに
Goal

学習目標

面接終了後、評価結果に応じて次ステップを自動判定し、Chat通知するエージェントを作成する

Workspace Studioの現在の制約について

現在のWorkspace Studioでは、以下の制約があります：

  • 外部へのGmail送信は未対応 - 自分へのChat/メール通知のみ
  • Calendar イベント終了トリガーは未対応 - スケジュールトリガーを使用

本レッスンでは、自動生成機能を活用してエージェントを作成し、必要に応じてプロンプトをカスタマイズします。

Decide Step

「決める」ステップの設定方法

Geminiの出力に含まれるキーワードで条件分岐を設定します。lesson2と同様のシンプルなパターンです。

ポイント：Gemini出力 → 決める の連携

Geminiステップで必ず「推奨アクション: Proceed / Hold / Decline」を出力させ、「決める」ステップでそのキーワードを検出します。

条件1

Proceed（次に進む）

条件： Gemini出力が「Proceed」を含む
アクション： 次面接の調整を開始

条件2

Hold（保留）

条件： Gemini出力が「Hold」を含む
アクション： マネージャーに判断依頼

条件3

Decline（見送り）

条件： Gemini出力が「Decline」を含む
アクション： お見送りメール下書き作成

設定手順

  1. 「決める」ステップを追加
  2. 条件として「Geminiの出力」を選択
  3. 「Proceed を含む」「Hold を含む」「Decline を含む」でそれぞれ分岐を設定
  4. 各分岐の先にChat通知ステップを追加
Branching

条件分岐ロジック

PROCEED

次に進む

→ 次の面接官にCalendar招待を自動作成
→ 候補者に次ステップ案内メール下書きを作成
→ 候補者管理シートのステータスを更新

HOLD

保留

→ 採用マネージャーに判断依頼通知を送信
→ 追加情報の収集を依頼
→ 候補者管理シートのステータスを「検討中」に更新

DECLINE

不採用

→ お見送りメールの下書きを作成
※送信は人間が確認
→ 候補者管理シートのステータスを更新

重要ポイント

最終判断は人間が行う設計です。エージェントは「準備」と「提案」を行い、実際の送信は人間が確認してから行います。特にお見送りメールは必ず人間がレビューしてから送信します。

Condition Logic

条件分岐の設定方法

クリックでコピー
IF 推奨アクション == "Proceed": → 次の面接官にCalendar招待を作成 → 候補者に次ステップ案内メールの下書きを作成 → Sheetsのステータスを「次面接予定」に更新 → Chatで採用マネージャーに通知 ELSE IF 推奨アクション == "Hold": → 採用マネージャーに判断依頼を送信 → Sheetsのステータスを「検討中」に更新 → 48時間後フォローアップのリマインダーを設定 ELSE IF 推奨アクション == "Decline": → お見送りメールの下書きを作成（自動送信しない） → Sheetsのステータスを「見送り予定」に更新 → 人間レビュー用にフラグを立てる
