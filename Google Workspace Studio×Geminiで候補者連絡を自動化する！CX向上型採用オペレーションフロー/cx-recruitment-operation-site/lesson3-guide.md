# 第3回：面接後作業をゼロにする！要約＆評価ポイントをGmail/Chatへ自動送信するフローを作る

## 学習目標

面接終了後、評価結果に応じて次ステップを自動判定し、Chat通知するエージェントを作成する

---

## 自動生成プロンプト

以下のプロンプトをWorkspace Studioの「Create agent」画面で入力すると、エージェントが自動生成されます。

```
面接終了後、評価結果に応じて次ステップを自動判定し、Chat通知するエージェントを作成する
```

---

## エージェントのフロー

| ステップ | 内容 | 詳細 |
|---------|------|------|
| S | スターター：スケジュール / フォーム送信 | Daily 18:00 または評価シート送信時に起動 |
| 1 | Geminiに質問する | 評価フィードバックを要約 |
| 2 | 決める：条件分岐 | Proceed/Hold/Declineで分岐 |
| 3 | チャットで通知する | 自分へのDMで結果を通知 |

---

## エージェント設定

| 項目 | 設定値 |
|------|--------|
| エージェント名 | Post-Interview Workflow |
| Description | 面接終了後、評価結果に応じて次ステップを自動判定し通知 |
| スターター | スケジュール（Daily 18:00）または フォーム送信時 |
| 処理 | Geminiで評価要約 → 「決める」で条件分岐 |
| 結果共有 | Chat (自分へのDM通知) |

## プロンプト：Gemini（評価フィードバック要約）

**重要：「決める」ステップとの連携**

このプロンプトの出力に「推奨アクション: Proceed」「推奨アクション: Hold」「推奨アクション: Decline」のいずれかを含めることで、「決める」ステップで条件分岐できます。

```
あなたは面接フィードバックを要約するHRコーディネーターです。

評価シートの回答：
{form_responses}

以下の形式で整理してください：

■ 総合印象: [ポジティブ/ニュートラル/ネガティブ]

■ 強み（最大3点）:
- [強み1]
- [強み2]
- [強み3]

■ 懸念点:
- [懸念点があれば記載、なければ「特になし」]

■ 推奨アクション: [Proceed/Hold/Decline のいずれか1つ]

判定基準:
- Proceed: 総合評価4以上 かつ 重大な懸念なし
- Hold: 総合評価3-4 または 確認が必要な懸念あり
- Decline: 総合評価3未満 または 重大な懸念あり

■ サマリー: [採用マネージャー向け2-3文]
```

---

## 「決める」ステップの設定方法

Geminiステップで必ず「推奨アクション: Proceed / Hold / Decline」を出力させ、「決める」ステップでそのキーワードを検出します。

| 条件 | 判定 | アクション |
|------|------|------------|
| Gemini出力が「Proceed」を含む | 条件1 | 次面接の調整を開始 |
| Gemini出力が「Hold」を含む | 条件2 | マネージャーに判断依頼 |
| Gemini出力が「Decline」を含む | 条件3 | お見送りメール下書き作成 |

### 設定手順

1. 「決める」ステップを追加
2. 条件として「Geminiの出力」を選択
3. 「Proceed を含む」「Hold を含む」「Decline を含む」でそれぞれ分岐を設定
4. 各分岐の先にChat通知ステップを追加

## 条件分岐ロジック

```
IF 推奨アクション == "Proceed":
    → 次の面接官にCalendar招待を作成
    → 候補者に次ステップ案内メールの下書きを作成
    → Sheetsのステータスを「次面接予定」に更新
    → Chatで採用マネージャーに通知

ELSE IF 推奨アクション == "Hold":
    → 採用マネージャーに判断依頼を送信
    → Sheetsのステータスを「検討中」に更新
    → 48時間後フォローアップのリマインダーを設定

ELSE IF 推奨アクション == "Decline":
    → お見送りメールの下書きを作成（自動送信しない）
    → Sheetsのステータスを「見送り予定」に更新
    → 人間レビュー用にフラグを立てる
```

## 次ステップ提案プロンプト

```
面接フィードバックに基づいて、次のステップを提案してください。

入力情報：
- 総合印象: {overall_impression}
- 強み: {strengths}
- 懸念点: {concerns}
- 面接官の推奨: {interviewer_recommendation}
- スキル評価: {skill_rating}/5
- カルチャー評価: {culture_rating}/5

次のステップを提案：

1. 「Proceed」の場合（評価4以上、重大な懸念なし）：
   - 次の面接ラウンドのCalendar招待を下書き
   - ポジションに応じた適切な次の面接官を提案

2. 「Hold」の場合（評価3-4、いくつかの懸念あり）：
   - 採用マネージャーへの判断依頼メッセージを下書き
   - 確認が必要な具体的ポイントをリスト化

3. 「Decline」の場合（評価3未満、重大な懸念あり）：
   - 丁寧なお見送りメールを下書き
   - 主な理由をまとめる（内部使用のみ）

出力内容：
- 推奨アクション
- メッセージ下書き
- 特記事項（あれば）
```

## 評価シートの標準項目（推奨）

| 項目 | 形式 | 説明 |
|------|------|------|
| 総合評価 | 5段階 | 1=不採用〜5=強く推薦 |
| スキル評価 | 5段階 | 職務に必要なスキルレベル |
| カルチャーフィット | 5段階 | 企業文化との適合性 |
| コミュニケーション力 | 5段階 | 受け答えの明確さ、論理性 |
| 強み | 自由記述 | 候補者の強みポイント（必須） |
| 懸念点 | 自由記述 | 採用にあたっての懸念事項（任意） |
| 質問メモ | 自由記述 | 面接中の質問と回答の要点 |
| 推奨アクション | 選択式 | Proceed / Hold / Decline |

## まとめ

- 「面接終了後、評価結果に応じて次ステップを自動判定し、Chat通知するエージェントを作成する」で自動生成
- Geminiプロンプトで「推奨アクション: Proceed/Hold/Decline」を出力させる
- 「決める」ステップで「Proceedを含む」「Holdを含む」「Declineを含む」で分岐
- Chat DM通知で結果を即座に確認し、次のアクションへ
- **CX向上：候補者への連絡が面接後1日以内に可能に（従来：数日〜1週間）**
