# 第4回：遅延しない採用運用！面接リマインド・評価提出リマインドを自動化する

## 学習目標

面接リマインドと評価シート提出リマインドを自動化する2つのエージェントを作成する

---

## 自動生成プロンプト

以下のプロンプトをWorkspace Studioの「Create agent」画面で入力すると、エージェントが自動生成されます。

### Agent 1: Interview Reminder

```
面接予定の前日と1時間前に、面接情報をChat通知するエージェントを作成する
```

### Agent 2: Evaluation Reminder

```
評価シート未提出の候補者をチェックし、経過時間に応じて段階的にChat通知するエージェントを作成する
```

---

## Interview Reminder 設計

| 項目 | 設定値 |
|------|--------|
| エージェント名 | Interview Reminder |
| Description | Sends reminders to candidates and interviewers before interviews |
| スターター1 | Schedule (Daily, 9:00 AM) - 前日リマインド |
| スターター2 | Schedule (Hourly) - 1時間前リマインド |
| データソース | Calendar（翌日/1時間後の面接イベント） |
| 出力 | Chat (自分へのDM) ※Gmailでの外部送信は現在未対応 |

---

## Interview Reminder 作成手順

| ステップ | 内容 | 詳細 |
|---------|------|------|

## Evaluation Reminder 設計

| 項目 | 設定値 |
|------|--------|
| エージェント名 | Evaluation Reminder |
| Description | Reminds interviewers to submit evaluation forms with escalation |
| スターター | Schedule (Daily, 10:00 AM) |
| データソース | 候補者管理シート（評価未提出レコード） |
| 条件分岐 | 24h, 48h, 72h経過でエスカレーション |
| 出力 | Chat (自分へのDM) ※Gmailでの外部送信は現在未対応 |

## 「決める」ステップの設定方法

Geminiステップで必ず「リマインドレベル: Level1 / Level2 / Level3 / NoAction」を出力させ、「決める」ステップでそのキーワードを検出します。

### Geminiプロンプト（経過時間判定用）

```
面接終了からの経過時間を計算し、リマインドレベルを判定してください。

入力データ:
- 面接終了日時: {interview_end_datetime}
- 評価シート提出状況: {evaluation_status}

判定基準:
- 提出済み → NoAction
- 未提出 & 24時間以上経過 → Level1
- 未提出 & 48時間以上経過 → Level2
- 未提出 & 72時間以上経過 → Level3

出力形式:
リマインドレベル: [Level1/Level2/Level3/NoAction]
```

### 「決める」ステップの条件設定

| 条件 | 判定 | アクション |
|------|------|------------|
| Gemini出力が「Level1」を含む | 条件1 | 丁寧なトーンでリマインド通知 |
| Gemini出力が「Level2」を含む | 条件2 | 緊急度を上げたリマインド通知 |
| Gemini出力が「Level3」を含む | 条件3 | マネージャーへエスカレーション通知 |

### 設定手順

1. 「Geminiに質問する」ステップで上記プロンプトを設定
2. 「決める」ステップを追加
3. 条件として「Geminiの出力」を選択
4. 「Level1 を含む」「Level2 を含む」「Level3 を含む」でそれぞれ分岐を設定
5. 各分岐の先に対応するChat通知ステップを追加

## 評価提出リマインド通知 - エスカレーション（72時間経過）

```
【エスカレーション】評価シート未提出

以下の面接について、評価シートが72時間以上未提出です。

■ 候補者: {candidate_name}
■ ポジション: {position}
■ 面接日: {interview_date}
■ 面接官: {interviewer}
■ 経過時間: {hours_elapsed}時間

■ 候補者の現在のステータス
{current_status}

■ 影響
・候補者への連絡が遅延しています
・CX（候補者体験）に影響する可能性があります

マネージャーからのフォローをお願いいたします。

■ 対応オプション
1. 面接官に直接フォロー
2. 口頭フィードバックを代理記入
3. 評価シートなしで次ステップ判断
```

## 運用のポイント

| ポイント | 内容 |
|----------|------|
| リマインド頻度の調整 | 候補者への連絡は前日と1時間前の2回に留める。多すぎると逆効果でCXを損なう可能性あり |
| トーンの使い分け | 1回目は丁寧に、2回目以降は緊急度アップ。ただし責めるトーンは避け、協力を依頼する姿勢で |
| エスカレーションルール | 何日でマネージャーに上げるかチームで合意。72時間が推奨。事前にルールを共有しておく |
| シート更新の習慣化 | 「最終連絡日」の更新を習慣化。第3回で作成したPost-Interview Workflowと連携して自動更新も可能 |

## テスト確認ポイント

- [ ] 1時間前リマインド情報が自分へのChat DMに届くか
- [ ] 評価未提出の検知が正しく動作するか
- [ ] 24h/48h/72hで段階的に通知内容が変わるか
- [ ] 通知に候補者名・面接官名・経過時間が正しく含まれているか

## まとめ

- **Interview Reminder:** 「面接予定の前日と1時間前に、面接情報をChat通知する」で自動生成
- **Evaluation Reminder:** 「評価シート未提出の候補者をチェックし、経過時間に応じて段階的にChat通知する」で自動生成
- Geminiプロンプトで「リマインドレベル: Level1/Level2/Level3」を出力させる
- 「決める」ステップで「Level1を含む」「Level2を含む」「Level3を含む」で分岐
- **CX向上：リマインド漏れゼロで候補者への連絡遅延を防止**
