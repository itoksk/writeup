# 第5回：採用オペレーションを完全自動化！選考フロー全体をStudioで一本化する方法

## 学習目標

日次の採用パイプラインサマリーをChat通知するエージェントを作成し、選考フロー全体を可視化する

---

## 自動生成プロンプト

以下のプロンプトをWorkspace Studioの「Create agent」画面で入力すると、Daily Dashboardエージェントが自動生成されます。

```
毎朝、候補者管理シートからパイプライン概況を集計し、日次サマリーをChat通知するエージェントを作成する
```

---

|---|---------------|------|----------|

| 03 | Interview Reminder | 面接リマインド | Daily/Hourly |

## Daily Recruiting Dashboard 設計

| 項目 | 設定値 |
|------|--------|
| エージェント名 | Daily Recruiting Dashboard |
| Description | Delivers daily summary of recruiting pipeline |
| スターター | Schedule (Daily, 9:00 AM) |
| データソース | 候補者管理シート（面接日・ステータス列を含む） |
| 出力 | Chat (自分へのDM) ※外部送信は未対応 |

---

## エージェント作成手順

| ステップ | 内容 | 詳細 |
|---------|------|------|
| 1 | エージェント自動生成 | 自動生成プロンプトを入力してエージェントを自動生成 |
| 2 | スターターの確認・調整 | 「スケジュール」: Daily 9:00 AM / 平日のみ |
| 3 | Sheetsステップの確認 | 候補者管理シートから全候補者データを取得 |
| 4 | Geminiステップのプロンプト調整 | パイプライン概況・本日の面接予定・フォロー必要候補者をサマリー化 |
| 5 | Chat通知ステップの確認 | 自分へのDMで日次ダッシュボードを受信 |

## プロンプト：Gemini（日次ダッシュボード生成）

```
あなたは日次の採用サマリーを作成するHRアナリティクスアシスタントです。

候補者管理シートのデータ：
{sheet_data}
※シートには「面接日」列があり、本日の面接予定はそこから抽出してください

日次の採用ダッシュボードを生成してください。以下の項目を含めること：

1. パイプライン概況（ステータス別人数）
2. 本日の面接スケジュール（シートの「面接日」列を参照）
3. フォローが必要な候補者（3日以上連絡なし）
4. 今週の指標（応募、面接、オファー）

Chatメッセージ形式で、一目で把握できアクションにつながる形式にしてください。

出力形式：
---
【採用パイプライン日次サマリー】

■ パイプライン概況
・応募受付: {count_応募受付}名
・書類選考中: {count_書類選考中}名
・面接予定: {count_面接予定}名
・評価待ち: {count_評価待ち}名
・内定: {count_内定}名

■ 本日の面接予定
{today_interviews_from_sheet}

■ 要フォロー（3日以上連絡なし）
{delayed_candidates}

■ 今週の採用活動
・新規応募: {week_applications}名
・面接実施: {week_interviews}件
・内定承諾: {week_acceptances}名
---
```

## ステータス定義

| ステータス | 意味 | トリガー |
|-----------|------|----------|
| 応募受付 | 応募書類受領 | 応募メール受信 |
| 書類選考中 | 履歴書確認中 | 手動 |
| 一次面接予定 | 面接日程確定 | シート更新（面接日列） |
| 一次面接完了 | 評価待ち | 評価フォーム送信 / 手動 |
| 二次面接予定 | 次面接確定 | シート更新（面接日列） |
| 最終面接完了 | 採否判定中 | 評価フォーム送信 / 手動 |
| 内定 | オファー提示 | 手動 |
| 内定承諾 | 入社確定 | 手動 |
| 辞退 | 候補者辞退 | 手動 |
| 不採用 | 選考終了 | 手動 |

## 運用ルール

| ルール | 内容 |
|--------|------|
| 命名規則の統一 | Calendar: 「面接：[候補者名]（[ポジション]）」、Drive: 「[候補者名]_[応募日]」、評価シート: 「評価_[候補者名]_[面接日]」 |
| ステータス更新タイミング | 自動: 応募受付・面接設定・面接完了・評価完了、手動: 次ステップ決定・内定・辞退 |
| エスカレーションフロー | 24h: 評価未提出→リマインド、48h: 再リマインド、72h: マネージャー通知、3日: 候補者連絡停滞→DM通知 |
| 例外処理 | エージェントエラー: Chatでエラー通知、候補者辞退: 手動でステータス更新、お見送り: 下書き作成→人間確認 |

## まとめ

- 「毎朝、候補者管理シートからパイプライン概況を集計し、日次サマリーをChat通知するエージェントを作成する」で自動生成
- このエージェントは「決める」ステップ不要のシンプルなサマリー生成型
- Geminiでパイプライン概況・要フォロー候補者を分析し、Chat通知
- 5つのエージェントで採用パイプライン全体を一元管理
- **CX向上：候補者の待機時間を可視化し、迅速なフォローを実現**
