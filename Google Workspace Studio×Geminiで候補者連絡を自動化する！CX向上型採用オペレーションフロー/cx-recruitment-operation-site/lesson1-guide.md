# 第1回：採用業務の手作業が消えない理由を見抜く！候補者連絡フローの全体像を理解する

## 学習目標

採用オペレーションにおける候補者連絡の課題を理解し、Google Workspace Studio × Geminiによる自動化の全体像を把握する

---

## 候補者連絡の4つの課題

| 課題 | 内容 |
|------|------|
| 連絡の遅延 | 面接日程調整に3日かかる、候補者からの質問への返信が翌日になる → 他社に流れるリスク |
| 連絡漏れ | 「次のステップの連絡したっけ？」という状況。スプレッドシート更新忘れで漏れが発生 |
| 連絡内容のバラつき | 担当者によって案内メールの内容が違う。候補者体験（CX）の品質が不安定 |
| 情報共有の断絶 | 面接官への事前情報共有が不十分、フィードバックが遅れて次アクションが決まらない |

---

## CX（Candidate Experience）向上の3つのポイント

1. **スピード** - 候補者を待たせない即時対応。応募確認は即時、日程調整は24時間以内
2. **一貫性** - どの担当者でも同じ品質の案内。テンプレート化と自動化で実現
3. **パーソナライズ** - 候補者に合わせた情報提供。Geminiで履歴書を要約し、適切なメッセージを生成

---

## コースで作成する5つのエージェント

| エージェント名 | 役割 | 作成回 |
|---------------|------|--------|
| Interview Prep Notification | 面接前日に候補者サマリーをGeminiで要約し、Chat DMで通知 | 第2回 |
| Post-Interview Workflow | 評価フォーム送信後、結果に応じて次ステップを条件分岐で判定・通知 | 第3回 |
| Interview Reminder | 面接前日と1時間前にChat DMでリマインド通知 | 第4回 |
| Evaluation Reminder | 24h・48h・72hでエスカレーション付き評価リマインド通知 | 第4回 |
| Daily Recruiting Dashboard | 毎朝パイプライン状況を集計してChat DMで配信 | 第5回 |

## プロンプト例：候補者情報要約

```
あなたは面接準備をサポートするHRアシスタントです。

以下の候補者情報を面接官向けに要約してください：

候補者名: {candidate_name}
応募ポジション: {position}
履歴書/職務経歴書の内容: {resume_content}
前回の面接メモ: {previous_notes}

以下の項目を含めて回答してください：
1. 主要なスキル・資格（3つのポイント）
2. 関連する経験のハイライト
3. 面接で深掘りすべきポイント
4. 懸念点や確認すべきギャップ

面接官がすぐに把握できる、簡潔で読みやすいサマリー形式で出力してください。
```

## CX評価指標

| 指標 | 内容 | 目標 |
|------|------|------|
| 応答時間 | 候補者からの問い合わせへの返信時間 | 24時間以内 → 自動化後：即時 |
| プロセス透明性 | 候補者が自分のステータスを把握できているか | 常に次のステップが明確 |
| コミュニケーション品質 | 連絡内容の一貫性と丁寧さ | パーソナライズされた連絡 |
| 候補者NPS | 採用プロセスを他者に勧めるか | +30以上 |

---

## まとめ

- 候補者連絡の課題は「遅延・漏れ・バラつき・断絶」
- CX（候補者体験）の向上が採用成功の鍵
- 「スピード・一貫性・パーソナライズ」を自動化で実現
- 5回のコースで5つのエージェントを構築
- 連絡業務時間を2〜3時間から30分以下に削減
