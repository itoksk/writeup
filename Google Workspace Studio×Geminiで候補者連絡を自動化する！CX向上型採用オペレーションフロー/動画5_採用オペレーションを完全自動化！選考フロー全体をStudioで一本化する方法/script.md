## 2. 統合アーキテクチャ（1:00-3:00）

### 講師セリフ

「まず、統合後のアーキテクチャを確認します。

**（画面：統合アーキテクチャ図）**

### 中心となるデータソース

**候補者管理シート**を中心に、全エージェントが連携します。

シートの構成：
- 候補者名
- 応募ポジション
- 現在のステータス
- 最終連絡日
- 担当面接官
- 評価結果
- 次アクション

### エージェント連携フロー

**（画面：連携フロー図）**

```
応募受信 → 候補者登録（シート）
    ↓
面接設定（Calendar）
    ↓
Interview Prep Notification（面接前準備）
    ↓
面接実施
    ↓
Post-Interview Workflow（面接後作業）
    ↓
Evaluation Reminder（評価リマインド）
    ↓
次ステップ決定
    ↓
Interview Reminder / Delay Alert（状況に応じて）
```

各エージェントがシートを参照・更新することで、データの一貫性を保ちます。」

## 3. 候補者ステータス管理（3:00-5:00）

### 講師セリフ

「統合の鍵となる**ステータス管理**を設定します。

**（画面：ステータス一覧）**

### ステータス定義

| ステータス | 意味 | 次アクション |
|-----------|------|-------------|
| 応募受付 | 応募書類受領 | 書類選考 |
| 書類選考中 | 履歴書確認中 | 結果連絡 or 面接設定 |
| 一次面接予定 | 面接日程確定 | 面接準備通知 |
| 一次面接完了 | 評価待ち | 評価リマインド |
| 二次面接予定 | 次面接確定 | 面接準備通知 |
| 最終面接完了 | 採否判定中 | 結果通知 |
| 内定 | オファー提示 | 回答待ち |
| 辞退/不採用 | 終了 | アーカイブ |

### ステータス自動更新

**（画面：自動更新設定）**

各エージェントの実行結果に応じて、シートのステータスを自動更新します。

例：面接終了検知 → ステータスを「一次面接完了」に更新

```
Update Google Sheets:
- Sheet: 候補者管理
- Row: {candidate_row}
- Column: ステータス
- Value: "一次面接完了"
- Column: 最終連絡日
- Value: TODAY()
```」

## 5. 運用ルールの設計（6:30-8:00）

### 講師セリフ

「自動化を持続的に機能させる**運用ルール**を設計します。

**（画面：運用ルール一覧）**

### ルール1：命名規則の統一

- Calendarイベント：「面接：[候補者名]（[ポジション]）」
- Driveフォルダ：「[候補者名]_[応募日]」
- 評価シート：「評価_[候補者名]_[面接日]」

### ルール2：ステータス更新タイミング

| アクション | 更新タイミング | 担当 |
|-----------|---------------|------|
| 応募受付 | 応募メール受信時 | 自動 |
| 面接設定 | Calendar登録時 | 自動 |
| 面接完了 | イベント終了時 | 自動 |
| 評価完了 | フォーム送信時 | 自動 |
| 次ステップ決定 | マネージャー判断後 | 手動 |

### ルール3：エスカレーションフロー

**（画面：エスカレーションフロー図）**

- 評価未提出48時間 → 面接官にリマインド
- 評価未提出72時間 → マネージャーに通知
- 候補者連絡3日停滞 → チームにアラート
- 候補者連絡5日停滞 → マネージャーに直接通知

### ルール4：例外処理

- エージェントがエラーの場合 → Chatでエラー通知
- 候補者から辞退連絡 → ステータスを「辞退」に手動更新」
