## Structuredの仕組み（1:00〜3:00）

### 基本構造の説明（1:00〜2:00）

#### 講師セリフ
「Structuredプロンプトでは、入力と出力の構造を明確に定義します。」

#### 画面表示
```
【基本構造】

入力部：
┌─────────────┐
│ 項目1：      │
│ 項目2：      │
│ 項目3：      │
└─────────────┘

出力部：
┌─────────────┐
│ 1. セクション1│
│ 2. セクション2│
│ 3. セクション3│
└─────────────┘
```

### 具体例の提示（2:00〜3:00）

#### 構造化の例
```
【入力項目】
・件名：[何についてのメールか]
・宛先部署：[送信先]
・要件：[主な内容]
・期限：[いつまでに]

【出力形式】
1. 挨拶（1行）
2. 要件説明（2-3行）
3. 詳細情報（箇条書き）
4. 締めの言葉（1-2行）
```

## 実演：社内研修案内（3:00〜7:00）

### タスク設定（3:00〜3:30）

#### 講師セリフ
「社内研修の案内メールをStructuredプロンプトで作成します。」

#### シナリオ
- 研修名：情報セキュリティ研修
- 対象：全社員
- 実施日：2024年2月15日
- 重要度：必須参加

### プロンプト作成（3:30〜4:30）

#### 完全なプロンプト
```
以下の情報を使って社内研修案内メールを作成してください。

【入力情報】
件名：情報セキュリティ研修
対象：全社員
日時：2024年2月15日 14:00-16:00
場所：3F会議室A
申込期限：2月10日
備考：必須参加、録画配信なし

【出力構成】
1. 件名（【重要】を付ける）
2. 宛名（各位）
3. 研修の目的（1-2文）
4. 開催詳細（箇条書き）
   - 日時
   - 場所
   - 対象者
   - 所要時間
5. 申込方法（具体的手順）
6. 注意事項（2-3点）
7. 問い合わせ先（担当部署と内線）
```

### 生成と確認（4:30〜5:30）

#### 講師セリフ
「構造が明確なので、必要な情報が漏れなく、決まった順序で出力されます。」

#### 出力例
```
件名：【重要】情報セキュリティ研修のご案内

各位

お疲れ様です。総務部より重要なお知らせです。

情報セキュリティの重要性が高まる中、全社員を対象とした
必須研修を実施いたします。最新の脅威と対策について
学ぶ重要な機会となります。

【開催詳細】
・日時：2024年2月15日（木）14:00-16:00
・場所：本社3F会議室A
・対象者：全社員（必須参加）
・所要時間：2時間

【申込方法】
1. 社内ポータルサイトにログイン
2. 「研修申込」メニューを選択
3. 「情報セキュリティ研修」を選んで申込
※申込期限：2月10日（土）17:00まで

【注意事項】
・必須研修のため、全員参加をお願いします
・当日参加できない方は、事前に総務部へご連絡ください
・録画配信はございません

【問い合わせ先】
総務部 研修担当（内線：2345）

以上、よろしくお願いいたします。
```

### 注意点とコツ（7:45〜8:30）

#### 講師セリフ
「注意点として、過度に構造化すると機械的な文面になります。最後に『トーン：親しみやすく』といった一行を追加すると、人間味のある文章になります。」

#### 改善例
```
【出力構成】
（既存の構成）
...
【トーン】
親しみやすく、前向きな雰囲気で
```
