# 動画3：Structuredプロンプトデモ集

## 基本テンプレート構造

### 社内研修案内テンプレート
```
以下の情報を使って社内研修案内メールを作成してください。

【入力情報】
件名：情報セキュリティ研修
対象：全社員
日時：2024年2月15日 14:00-16:00
場所：3F会議室A
申込期限：2月10日
備考：必須参加、録画配信なし

【出力構成】
1. 件名（【重要】を付ける）
2. 宛名（各位）
3. 研修の目的（1-2文）
4. 開催詳細（箇条書き）
   - 日時
   - 場所
   - 対象者
   - 所要時間
5. 申込方法（具体的手順）
6. 注意事項（2-3点）
7. 問い合わせ先（担当部署と内線）
```

### 会議招集メールテンプレート
```
以下の情報から会議招集メールを作成してください。

【入力情報】
会議名：月次売上レビュー会議
日時：2024年2月20日 10:00-11:00

【出力構成】
1. 件名（会議名と日時を含む）
2. 宛先（関係者各位）
3. 会議の目的（簡潔に）
4. 会議詳細（表形式）
5. アジェンダ（番号付きリスト）
6. 事前準備のお願い
7. 出欠確認の依頼
```

## 業務連絡用テンプレート

### システムメンテナンス告知
```
以下の情報でシステムメンテナンス告知メールを作成してください。

【入力情報】
対象システム：社内ポータルサイト
メンテナンス日時：2024年2月25日 22:00-翌2:00
影響：全サービス停止
代替手段：緊急時は電話連絡
事前対応：必要なデータは事前にダウンロード

【出力構成】
1. 件名（【メンテナンス】を先頭に）
2. 影響を受ける方（全社員）
3. メンテナンス概要（1段落）
4. 詳細情報（箇条書き）
   - 対象システム
   - 作業日時
   - 影響範囲
   - 復旧予定時刻
5. ご協力いただきたいこと
6. 緊急連絡先
7. お詫びの言葉
```

### 業務変更通知
```
以下の内容で業務プロセス変更の通知メールを作成してください。

【入力情報】
変更内容：経費精算の申請方法
変更日：2024年3月1日から
旧方法：紙の申請書を経理部へ提出
新方法：Webシステムで申請
移行期間：3月末まで両方受付
マニュアル：社内ポータルに掲載

【出力構成】
1. 件名（【業務変更】で始める）
2. 対象者明記
3. 変更の背景（2-3文）
4. 変更内容の比較表
5. 移行スケジュール
6. 新システムの使い方
7. サポート体制
8. FAQ（3-4個）
```

```
以下の内容で予算申請却下の通知を作成してください。

2. 宛名

4. 却下理由の説明（建設的に）
5. 改善提案
6. 再申請の案内
7. 相談窓口の案内
8. 今後の協力姿勢を示す締め
```

## ハイブリッド型（構造+トーン指定）

### 親しみやすい構造化メール
```
以下の構造で、親しみやすいトーンのメールを作成してください。

【入力情報】
イベント：社内懇親会
日時：2024年2月28日 18:00〜
場所：会社近くのレストラン
会費：3,000円
申込：2月20日まで

【出力構成】
1. 件名（楽しそうな雰囲気で）
2. カジュアルな挨拶
3. イベントの魅力的な説明
4. 詳細情報（見やすく整理）
5. 参加を促す一言
6. 申込方法（簡単に）
7. ワクワクする締めの言葉

【トーン】
堅くなりすぎず、でも失礼にならない程度にフレンドリーに
```

### プロフェッショナルな構造化メール
```
以下の構造で、信頼感のあるプロフェッショナルなメールを作成してください。

【入力情報】
報告内容：プロジェクト進捗
完了率：75%
課題：リソース不足
対策：外部委託を検討
次回報告：2週間後

【出力構成】
1. 件名（プロジェクト名と報告種別）
2. 簡潔な挨拶
3. エグゼクティブサマリー
4. 詳細報告（数値含む）
5. 課題と対策（論理的に）
6. 次のアクション
7. 締めの言葉

【トーン】
データに基づいた客観的で信頼できる報告調
```

## テンプレート改善チェックリスト

### 効果的なテンプレートの条件
✅ 必要な情報が網羅されている
✅ 論理的な順序で構成されている
✅ 再利用しやすい汎用性がある
✅ カスタマイズポイントが明確
✅ 出力の品質が安定している

### 避けるべきパターン
❌ 構造が複雑すぎる
❌ 入力項目が多すぎる
❌ 柔軟性がない
❌ 文脈を無視した機械的な構成
❌ トーンの指定がない

## 実践演習

### 演習1：既存メールの構造化
自分がよく書くメールを1つ選び、Structuredテンプレートに変換してください。

### 演習2：チーム用テンプレート作成
チーム全体で使える汎用的なテンプレートを1つ作成してください。

### 演習3：トーンの追加
既存のStructuredテンプレートに、適切なトーン指定を追加してください。
