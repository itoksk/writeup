## シーン別最適プロンプト選択ガイド

### クレーム対応（フューショット＋Structured）
```
以下の例を参考に、クレーム対応メールを作成してください。

【過去の成功例】
[例1：配送遅延]
お客様、この度は配送の遅れにより大変ご迷惑をおかけし、
心よりお詫び申し上げます。お客様のご不便を真摯に受け止め、
即座に配送状況を確認し、明日中にお届けする手配を整えました。

[例2：商品不良]
この度は弊社商品の不良により、ご迷惑をおかけしましたこと、
深くお詫び申し上げます。代替品を本日中に発送させていただき、
不良品は着払いでご返送いただければ幸いです。

[例3：サービス不具合]
システムの不具合によりサービスをご利用いただけなかったこと、
誠に申し訳ございません。現在は復旧し、再発防止策も実施済みです。
ご不便をおかけした分、次回ご利用時に特別割引を適用させていただきます。

【今回の状況】
クレーム内容：注文と異なる商品が届いた
顧客名：田中様
対応方針：即日交換、送料当社負担

【出力構成】
1. 件名（お詫びの意を明確に）
2. 冒頭の謝罪（心からの謝罪）
3. 状況の確認と共感
4. 具体的な解決策の提示
5. 今後の改善策
6. 締めの謝罪とお願い
```

### セミナー参加お礼（Structure＋Freeform＋トーン）
```
セミナー参加者へのお礼メールを作成してください。

【基本情報】
イベント：AI活用セミナー
参加者：100名
次回：3月20日

【構成】
1. 件名：感謝の気持ちが伝わるもの
2. 導入：温かみのある挨拶
3. 本文：
   - 参加への心からの感謝（Freeform的表現で）
   - セミナーのハイライト（3点を箇条書き）
   - 次回案内（期待感を持たせる表現）
4. 締め：また会えることを楽しみにする言葉

【トーン】
プロフェッショナルでありながら親しみやすく、
次回も参加したくなるような前向きな雰囲気
```

### プロジェクト進捗報告（Structure＋数値＋感情）
```
プロジェクト進捗報告メールを作成してください。

【データ】
プロジェクト：新システム開発
進捗：65%（予定通り）
完了タスク：要件定義、基本設計
進行中：詳細設計
課題：テスト環境の準備遅れ

【構成要件】
1. 件名（進捗率を含める）
2. エグゼクティブサマリー（3行以内）
3. 詳細報告
   - 完了事項（達成感を込めて）
   - 進行中タスク（具体的に）
   - 課題と対策（建設的に）
4. 今後の予定（明確な日付入り）
5. チームへの感謝（メンバーのモチベーション向上）

【表現方針】
・数値は正確に、でも機械的にならない
・課題は正直に、でも前向きに
・チームの頑張りが伝わる表現
```

## よくある失敗と対策

### 失敗例1：過度な構造化
```
❌ 悪い例：
【項目1】【項目2】【項目3】【項目4】【項目5】
【項目6】【項目7】【項目8】【項目9】【項目10】
→ 複雑すぎて使いづらい

✅ 改善例：
【基本情報】3項目
【本文構成】3-4セクション
【トーン】1行で指定
```

### 失敗例2：曖昧な指示
```
❌ 悪い例：
「いい感じのメールを書いて」

✅ 改善例：
「プロフェッショナルだが親しみやすい、
読み手が次のアクションを取りたくなるメール」
```

### 失敗例3：例文の選択ミス
```
❌ 悪い例：
フォーマルな場面でカジュアルな例文を使用

✅ 改善例：
相手と状況に応じた適切なレベルの例文を選択
```
