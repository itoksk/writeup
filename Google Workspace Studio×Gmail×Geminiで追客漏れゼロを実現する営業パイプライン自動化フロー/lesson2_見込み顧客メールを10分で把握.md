# 第2回：見込み顧客メールを10分で把握！
## 要約・重要顧客優先・緊急案件アラートの自動化

---

## この回の目標

営業担当者が行う「メール確認」を自動化します。**メール受信時にリアルタイムで実行**され、見込み顧客からのメールをGeminiが即座に要約。重要顧客を優先表示し、緊急案件があればアラートで通知します。

---

## Before / After

### BEFORE：受講前
- 毎朝メールを1通ずつ開いて確認
- 重要顧客のメールを見落とす
- 緊急案件に気づくのが遅れる
- メール確認に1時間以上かかる

### AFTER：受講後
- メール受信時に即座に要約が届く
- 重要顧客が優先表示される
- 緊急案件はアラートで即通知
- メール確認が10分で完了（80%削減）

---

## 作成するエージェント

| 項目 | 内容 |
|------|------|
| 名前 | 見込み顧客メール要約エージェント |
| スターター | Gmailスターター（メール受信時） |
| ステップ | Geminiステップで要約・優先度判定 → Chat DMステップで通知 |
| 効果 | 重要メール受信時に即座に把握可能 |

> **80%** メール確認時間を削減（毎朝1時間 → 10分以内に）

---

## エージェントのフロー

```
Gmailスターター → Geminiステップ → Chat DMステップ
```

### このフローのポイント
- **Gmailスターターでメール受信時にリアルタイム実行**：重要メールを即座に把握
- **フィルター設定で重要顧客からのメールのみをトリガー**：ノイズを排除して効率化
- **Geminiステップで優先度判定**：緊急案件を見落とさない
- **Chat DMステップで通知**：メールを開かずに内容を把握

---

## 作成手順

### Step 01: Studioにアクセス

ブラウザで studio.workspace.google.com にアクセスします。「Create」ボタンをクリックして新規エージェントを作成します。

```
URL: studio.workspace.google.com
```

> **アクセスできない場合**
> 管理者によってStudioが有効化されていない可能性があります。管理コンソール > 生成AI > Gemini for Workspace での設定を管理者に確認してください。

### Step 02: Gmailスターター設定

重要顧客からメールを受信した時に自動起動するよう設定します。「Starter」セクションで「Gmail」を選択します。

**設定値**
- **スターター種類:** Gmail（メール受信時）
- **トリガー:** 新規メール受信時
- **フィルター:** 重要顧客からのメール

```
# フィルター例（メールアドレス指定）
from:yamada@a-corp.co.jp OR from:tanaka@b-inc.com OR from:suzuki@c-group.jp
```

> **フィルターのポイント**
> - Gmailの`from:`演算子では**ドメイン指定（@xxx.co.jp）は使用できません**
> - 重要顧客の**メールアドレスを個別に指定**してください
> - 複数のメールアドレスは`OR`で結合します
> - 全メールでトリガーすると処理量が多くなるため、対象を絞ることを推奨します

### Step 03: Geminiステップ（要約・分類）

受信したメールをGeminiステップで分析・要約します。「Add step」から「Gemini」を選択し、プロンプトを設定します。

Gmailスターターで受信したメールの内容が自動的に変数として利用可能です。

### Step 04: Chat DMステップ（通知）

要約結果をGoogle Chatのダイレクトメッセージで送信します。「Add step」から「Google Chat - Send direct message」を選択します。

> ⚠️ **Chat通知についての重要な変更**
> 正式リリースでは**Chatスペースへの投稿機能は削除**されました。Chat通知は**個人へのDM（ダイレクトメッセージ）のみ**利用可能です。チーム共有が必要な場合は、Gmailでのメール送信を検討してください。

**通知テンプレート:**
```
📬 見込み顧客メール要約

⏰ 受信日時: {when_email_was_sent}

{gemini_summary}

---
対応漏れがないかご確認ください。
```

## Geminiプロンプト

Geminiに渡すプロンプトは、出力形式を明確に指定することがポイントです。以下のプロンプトをGeminiステップに設定してください。

### 推奨プロンプト

```
以下の営業メールを分析して、要約してください。

{trigger_email}

以下の形式で出力してください：
1. 【送信者】送信者名と会社名
2. 【件名】メールの件名
3. 【要約】3文以内で要約
4. 【緊急度】HIGH/MEDIUM/LOW
5. 【案件ステージ】問い合わせ/提案中/見積もり/契約交渉
6. 【推奨アクション】次に取るべきアクション

簡潔かつ実用的に出力してください。
```

### プロンプトのポイント
- **{trigger_email}**：Gmailスターターで受信したメールの変数（変数メニューから選択）
- **出力形式を明確に指定**：項目ごとに【】で区切る
- **緊急度判定**：案件の優先度を即座に把握
- **推奨アクション**：次のステップを明確に

### 出力例

```
📬 見込み顧客メール要約

⏰ 受信日時: 2025/01/15 14:32

【送信者】山田太郎様（株式会社ABC）
【件名】提案書の件でご相談
【要約】先日ご提案いただいた内容について、社内で検討しました。予算感が合えば進めたいとのことで、見積もりの再調整をお願いしたいとのご連絡です。
【緊急度】HIGH
【案件ステージ】見積もり
【推奨アクション】本日中に見積もり再調整の連絡を入れる

---
即座にご確認ください。
```

## テストと有効化

エージェントが完成したら、必ずテストを実行してから有効化します。

### テスト確認項目
- [ ] Gmailスターターのフィルターが正しく動作しているか
- [ ] 指定した重要顧客からのメールのみがトリガーされているか
- [ ] Geminiが日本語で要約を出力しているか
- [ ] 緊急度・案件ステージが正しく判定されているか
- [ ] Chatに通知が届いているか
- [ ] 変数（{trigger_email}など）が正しく展開されているか
- [ ] メールの内容が正しく分析されているか

> ⚠️ **テスト時の注意点**
> テスト実行時は実際にGoogle ChatのDMに通知が送信されます。

## 運用のコツ

### 重要顧客リストの更新
検索クエリに含める顧客ドメインは定期的に見直しましょう。新規取引先の追加、取引終了先の削除を忘れずに。

### フィルターの最適化
トリガーするメールの範囲を調整できます。最初は重要顧客のみに絞り、運用が安定してきたら対象を広げていくのがおすすめです。

### プロンプトの改善
使いながら出力を見て、プロンプトを改善していきましょう。「案件金額も出力してほしい」「顧客の業種も知りたい」など、必要に応じて項目を追加できます。
