## 2. 確度高案件監視エージェント（1:00-4:00）

### 講師セリフ

「まず、確度高案件監視エージェントを作成します。

**（画面：エージェントフロー図）**

### フロー

```
[メール受信] → [キーワード検出] → [Gemini確度判定] → [優先アラート]
```

### 2-1. 監視するキーワード（1:30-2:00）

**（画面：キーワードリスト）**

営業の確度が高いメールには、特定のキーワードが含まれることが多いです：

**受注シグナル（確度HIGH）**
- 契約、発注、注文、決定
- お願いします、進めてください
- 稟議通りました、承認されました

**検討シグナル（確度MEDIUM）**
- 見積もり、提案、検討
- 予算、スケジュール、導入時期
- 比較、候補、選定

**（画面：Create → 新規エージェント）**

エージェント名：『確度高案件監視エージェント』

### 2-2. スターター設定（2:00-2:30）

**（画面：Starter選択）**

『Gmail - Email received』を選択します。

**（画面：検索条件）**

検索条件でキーワードを指定：

```
from:(@重要顧客) (契約 OR 発注 OR 注文 OR 見積 OR 提案 OR 検討)
```

これで、重要顧客から上記キーワードを含むメールを受信したときに起動します。

### 2-3. Geminiで確度判定（2:30-3:30）

**（画面：Add Step → Gemini）**

プロンプト：

```
Analyze this sales email for deal probability:

{email_content}

Provide analysis in Japanese:

【検出キーワード】見つかった営業キーワードをリスト
【買いシグナル】ポジティブ/ネガティブ/ニュートラル
【確度】HIGH/MEDIUM/LOW
【確度スコア】1-10（10が最も高い）
【根拠】判定の根拠（2-3文）
【推奨アクション】今すぐ取るべきアクション
【対応優先度】1-5（1が最優先）
```

**（画面：変数確認）**

出力は `probability_analysis` に保存します。

### 2-4. 優先アラート（3:30-4:00）

**（画面：条件分岐）**

確度HIGHの場合、即座にアラート：

条件：`{probability_analysis} contains "HIGH"`

**（画面：Chat設定）**

True → 優先アラート送信：

```
🔥 高確度案件を検知しました！

【送信者】{sender_name}
【件名】{email_subject}

{probability_analysis}

---
⚡ 今すぐ対応してください！
📧 メールを確認: {email_link}
```

## 3. エージェントの統合と連携（4:00-6:00）

### 講師セリフ

「ここまでで4つのエージェントが完成しました。これらを連携させて、一つの営業フローとして機能させます。

**（画面：4つのエージェント連携図）**

### 連携図

```
[メール受信]
    │
    ├─→ [見込み顧客メール要約] ──→ 毎朝の要約通知
    │
    ├─→ [追客漏れゼロ] ──────→ ラベル付与 + 次アクション
    │
    ├─→ [営業資料自動整理] ───→ Drive保存 + 週次レポート
    │
    └─→ [確度高案件監視] ────→ 優先アラート
```

### 重複を避ける設計（4:30-5:00）

**（画面：フィルター設計）**

複数のエージェントが同じメールに反応しないよう、フィルターを設計します：

| エージェント | フィルター | 起動条件 |
|--------------|------------|----------|
| メール要約 | 毎朝9時（Scheduled） | 全ての未読メール |
| 追客漏れゼロ | Email received | 重要顧客からの全メール |
| 資料自動整理 | has:attachment | 添付ファイル付きのみ |
| 確度高案件監視 | キーワード含む | 特定キーワードを含む |

### 通知のまとめ方（5:00-5:30）

**（画面：通知先の整理）**

通知が多すぎると逆効果です。以下のように整理しましょう：

- **個人DM**: 緊急アラート（確度HIGH、追客漏れ）
- **チームスペース**: 日次要約、週次レポート
- **マネージャースペース**: 確度HIGHのみ

### ダッシュボード化（5:30-6:00）

**（画面：Chat通知のイメージ）**

毎朝のChat通知を「営業ダッシュボード」として活用：

```
📊 本日の営業ダッシュボード

【確度HIGH案件】
🔥 株式会社A: 契約交渉中（要対応）

【要対応メール】
📌 5件（「要対応」ラベルで確認）

【今週の資料】
📁 新規3件、更新2件

【本日の優先アクション】
1. A社に契約書を送付
2. B社にフォローアップ電話
3. C社の見積もり確認
```

## 4. 運用時の注意点とトラブルシューティング（6:00-8:00）

### 講師セリフ

「エージェントを運用する際の注意点とトラブルシューティングを解説します。

**（画面：注意点リスト）**

### 4-1. よくある問題と対処法（6:00-7:00）

**問題1：エージェントが起動しない**
- 原因：フィルター条件が厳しすぎる
- 対処：条件を緩めてテスト

**問題2：通知が多すぎる**
- 原因：条件が緩すぎる
- 対処：フィルターを追加、条件分岐で絞り込み

**問題3：Geminiの出力が不安定**
- 原因：プロンプトが曖昧
- 対処：出力形式を厳密に指定

**問題4：変数が展開されない**
- 原因：変数名の入力ミス
- 対処：変数メニューから選択

### 4-2. パフォーマンス最適化（7:00-7:30）

**（画面：最適化のポイント）**

- **不要なエージェントは無効化**：使わないエージェントはDisable
- **テスト環境と本番を分ける**：まずテストで確認
- **定期的な見直し**：月1回、フィルターと条件を見直し

### 4-3. セキュリティ考慮事項（7:30-8:00）

**（画面：セキュリティ）**

- **機密情報の取り扱い**：Geminiの出力に機密情報が含まれる可能性
- **共有範囲の確認**：Chat通知先のメンバーを確認
- **監査ログの活用**：管理者は実行ログを確認可能
