## 1. 確度判定プロンプト

### 英語版（推奨）
```
Analyze this sales email for deal probability:

{email_content}

Provide analysis in Japanese:

【検出キーワード】見つかった営業キーワードをリスト
【買いシグナル】ポジティブ/ネガティブ/ニュートラル
【確度】HIGH/MEDIUM/LOW
【確度スコア】1-10（10が最も高い）
【根拠】判定の根拠（2-3文）
【推奨アクション】今すぐ取るべきアクション
【対応優先度】1-5（1が最優先）
```

```
以下の営業メールを分析し、案件の確度を判定してください：

【確度】高/中/低
【確度スコア】1-10
【根拠】
【推奨アクション】
```

## 5. 条件分岐の条件式

```

```

```
{probability_analysis} contains "HIGH" OR {probability_analysis} contains "MEDIUM"
```

### 確度スコア8以上
```
{probability_analysis} matches "確度スコア】[8-9]|10"
```

## 8. トラブルシューティング

### 問題1：エージェントが起動しない
```
確認事項：
- フィルター条件が厳しすぎないか
- 対象メールが存在するか
- エージェントがEnabledになっているか

対処法：
- 条件を緩めてテスト
- テストモードで動作確認
```

### 問題2：通知が多すぎる
```
確認事項：
- フィルターが緩すぎないか
- 条件分岐が正しく設定されているか

対処法：
- フィルターを追加
- 条件分岐でHIGHのみに絞る
- 通知頻度を調整（まとめて通知）
```

### 問題3：Geminiの出力が不安定
```
確認事項：
- プロンプトが曖昧でないか
- 出力形式を厳密に指定しているか

対処法：
- 出力形式を明確に指定
- 例を含める
- 「必ず以下の形式で」と強調
```

## 9. エージェント構成まとめ

```
エージェント名: 確度高案件監視エージェント

Starter: Gmail - Email received
  - Filter: from:(重要顧客) (契約 OR 発注 OR 見積 OR 提案)

Step 1: Gemini - Generate content
  - Prompt: [確度判定プロンプト]
  - Input: {email_content}
  - Output: probability_analysis

Step 2: Conditional
  - Condition: {probability_analysis} contains "HIGH"

  True branch:
    Step 2a: Google Chat - Send message
      - Space: 個人DM
      - Message: [優先アラート]

    Step 2b: Gmail - Add label
      - Label: 確度HIGH

    Step 2c: Google Chat - Send message
      - Space: マネージャースペース
      - Message: [マネージャー用アラート]

  False branch:
    Step 3: Conditional
      - Condition: {probability_analysis} contains "MEDIUM"

      True branch:
        Step 3a: Gmail - Add label
          - Label: 検討中

      False branch:
        - 終了
```
