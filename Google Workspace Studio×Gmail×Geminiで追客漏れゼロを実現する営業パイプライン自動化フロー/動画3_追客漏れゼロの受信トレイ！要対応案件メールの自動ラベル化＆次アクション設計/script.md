# 第3回：追客漏れゼロの受信トレイ！要対応案件メールの自動ラベル化＆次アクション設計

**動画時間**: 10分
**学習目標**: 要対応案件メールを自動でラベル付けし、Geminiに次のアクションを提案させるエージェントを作成する

---

## 1. 導入（0:00-1:00）

### 講師セリフ

「皆さん、こんにちは。営業パイプライン自動化フロー、第3回をお届けします。

前回は、毎朝メールを自動で要約するエージェントを作成しました。

今回は、さらに一歩進めて『追客漏れゼロエージェント』を作成します。

**（画面：追客漏れのイメージ）**

営業で最も痛いのは、追客すべきタイミングを逃すこと。返信を忘れて1週間経ってしまい、他社に決まっていた...そんな経験、ありませんか？

このエージェントは、要対応の案件メールを検知し、自動でラベルを付け、Geminiが次のアクションを提案してくれます。

**（画面：自動ラベル付けのイメージ）**

これで、追客漏れはゼロになります。」

---

## 2. エージェントの全体像（1:00-2:00）

### 講師セリフ

「今回作るエージェントの構成を確認しましょう。

**（画面：エージェントフロー図）**

### フロー

```
[メール受信] → [Gemini分析] → [ラベル付与] → [Chat通知]
```

### 機能

1. **トリガー**：メールを受信したら即座に起動
2. **Gemini分析**：要対応かどうか判定、次アクションを提案
3. **ラベル付与**：「要対応」ラベルを自動付与
4. **Chat通知**：対応リストと次アクションを通知

**（画面：Before/After）**

前回のエージェントは「まとめて確認」でしたが、今回は「リアルタイム」で反応します。重要な案件は受信した瞬間に把握できます。」

---

## 3. スターター設定（2:00-3:00）

### 講師セリフ

「新しいエージェントを作成します。

**（画面：Create → 新規エージェント）**

エージェント名は『追客漏れゼロエージェント』とします。

**（画面：Starter選択）**

### スターター設定

今回は『Gmail - Email received』を選択します。メールを受信するたびにエージェントが起動します。

**（画面：Gmail Starter設定）**

### フィルター設定

すべてのメールで起動すると多すぎるので、フィルターを設定します：

- **From**: 重要顧客のドメイン（@a-corp.co.jp, @b-inc.com など）

または、

- **Label**: 「見込み顧客」ラベルが付いたメール

**（画面：設定完了）**

これで、重要顧客からメールが届くたびにエージェントが起動します。」

---

## 4. Geminiで分析・判定するステップ（3:00-5:00）

### 講師セリフ

「次に、受信したメールをGeminiで分析します。

**（画面：Add Step → Gemini）**

『Add Step』から『Gemini』を選択します。

**（画面：プロンプト入力）**

### プロンプト

```
Analyze this sales email and determine if follow-up action is required:

{email_content}

Provide response in Japanese with the following format:

【要対応判定】YES または NO
【理由】判定の理由（1文）
【緊急度】HIGH/MEDIUM/LOW
【案件ステージ】問い合わせ/提案中/見積もり/契約交渉
【次アクション】具体的に取るべきアクション
【対応期限】推奨対応期限（例：本日中、3日以内）
【返信ドラフト】簡潔な返信案（50文字程度）
```

**（画面：変数挿入）**

`{email_content}` には、スターターで受信したメールの内容を挿入します。件名、本文、送信者情報などがすべて含まれます。

**（画面：出力変数設定）**

出力は `gemini_analysis` という変数に保存します。次のステップで使用します。」

---

## 5. 条件分岐でラベルを付与するステップ（5:00-6:30）

### 講師セリフ

「Geminiの判定結果に基づいて、ラベルを付与します。

**（画面：Add Step → Conditional）**

『Add Step』から『Conditional』を選択します。

**（画面：条件設定）**

### 条件

```
{gemini_analysis} contains "YES"
```

Geminiが「要対応判定：YES」と回答した場合にTrueになります。

**（画面：True分岐 → Gmail）**

### True分岐：ラベル付与

『Add Step』→『Gmail』→『Add label』を選択します。

- **Email**: {trigger_email}（受信したメール）
- **Label**: 「要対応」

**（画面：ラベル作成）**

ラベルが存在しない場合は、事前にGmailで作成しておくか、Studioが自動で作成してくれます。

**（画面：False分岐）**

### False分岐

要対応でない場合は、特にアクションなしで終了します。」

---

## 6. Chat通知ステップ（6:30-8:00）

### 講師セリフ

「要対応の場合、Chatで通知します。

**（画面：True分岐にステップ追加）**

条件分岐のTrue側に、もう1つステップを追加します。

**（画面：Add Step → Chat）**

『Google Chat』→『Send message』を選択します。

**（画面：メッセージ設定）**

### メッセージテンプレート

```
📌 要対応案件メール

【送信者】{sender_name} ({sender_email})
【件名】{email_subject}

{gemini_analysis}

---
📧 元のメールを確認: {email_link}
ラベル「要対応」を付与しました。
```

**（画面：変数の確認）**

各変数には以下の情報が入ります：
- `{sender_name}`: 送信者名
- `{sender_email}`: 送信者メールアドレス
- `{email_subject}`: 件名
- `{gemini_analysis}`: Geminiの分析結果
- `{email_link}`: メールへのリンク

**（画面：緊急度による分岐）**

### 緊急度HIGHの場合

さらに条件を追加して、緊急度HIGHの場合は別途アラートを送信することもできます：

```
{gemini_analysis} contains "HIGH"
```

True → 追加アラート
```
🚨 緊急対応が必要な案件です！

{gemini_analysis}

至急確認してください。
```
