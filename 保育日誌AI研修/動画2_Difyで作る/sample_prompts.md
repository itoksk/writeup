# 動画2：Difyで作る - サンプルプロンプト集

## システムプロンプト設定

### 標準版
```
あなたは保育園の日誌作成を支援するアシスタントです。
入力された活動内容をもとに、以下の形式で日誌を作成してください：

【今日の活動】
- 活動内容を2〜3文で具体的に記載
- 子どもたちの反応や様子を含める

【子どもの様子】
- 個別の観察記録を含める
- 成長や変化を具体的に記載
- ポジティブな表現を心がける

【明日への連携】
- 継続観察ポイントを1文で
- 次の活動への展開を意識

文体は温かく、保護者が読んで安心する表現を使用してください。
```

### シンプル版（初心者向け）
```
保育園の日誌を作成するアシスタントです。
入力内容から、今日の活動と子どもの様子をまとめてください。
温かい文体で、200字程度にまとめてください。
```

### 詳細版（カスタマイズ用）
```
あなたは経験豊富な保育士として、日誌作成を支援します。

入力形式：簡単なメモや箇条書き
出力形式：
1. 活動内容（何をしたか）
2. 子どもの様子（どんな反応だったか）
3. 教育的観点（何が育まれたか）
4. 次回への課題（何を観察するか）

文字数：300字以内
文体：です・ます調で統一
注意：個人名はイニシャルで表記
```

## テスト用入力サンプル

### テスト1：シンプルな入力
```
積み木遊び。Bくんが高いタワーを作った。みんなで拍手。
```

**期待される出力例：**
> 【今日の活動】
> 積み木遊びを楽しみました。Bくんが集中して高いタワーを作り上げ、完成した時にはみんなで拍手をして喜び合いました。
> 
> 【子どもの様子】
> Bくんの集中力と創造力が光っていました。他の子どもたちも友達の成功を称える優しさを見せてくれました。
> 
> 【明日への連携】
> 協力して大きな作品を作る活動も取り入れていきたいと思います。

### テスト2：複数の活動
```
午前：外遊び、砂場で山作り
午後：絵本読み聞かせ、みんな集中
```

**期待される出力例：**
> 【今日の活動】
> 午前中は園庭で外遊びを楽しみ、砂場では協力して大きな山を作りました。午後は絵本の読み聞かせを行い、子どもたちは物語に夢中になっていました。
> 
> 【子どもの様子】
> 砂場では役割分担をしながら協力する姿が見られました。絵本の時間には、静かに集中して聞く力が育っています。
> 
> 【明日への連携】
> 絵本の内容を活かした表現遊びにつなげていきます。

### テスト3：個別観察を含む
```
水遊び。
Eちゃん：最初は見ているだけ→友達に誘われて参加
Fくん：率先してホースで水まき
全体：楽しそうに遊んでいた
```

**期待される出力例：**
> 【今日の活動】
> 水遊びで暑い日を楽しく過ごしました。水の感触を楽しみながら、それぞれのペースで活動に参加していました。
> 
> 【子どもの様子】
> Eちゃんは最初慎重に見ていましたが、友達の優しい誘いで徐々に参加し、最後は笑顔いっぱいでした。Fくんはリーダーシップを発揮してホースで水まきを担当し、みんなを楽しませてくれました。
> 
> 【明日への連携】
> Eちゃんの積極性の芽生えを大切に見守っていきます。

## Difyでの変数設定

### 入力変数の設定例

| 変数名 | 説明 | タイプ | 必須 |
|--------|------|--------|------|
| `activity` | 活動内容 | テキスト | Yes |
| `observation` | 観察記録 | テキスト | No |
| `note` | 特記事項 | テキスト | No |

### プロンプト内での変数使用
```
今日の活動：{{activity}}
観察記録：{{observation}}
特記事項：{{note}}

上記の内容から保育日誌を作成してください。
```

## トラブルシューティング

### 問題：出力が長すぎる
**解決策：** プロンプトに文字数制限を追加
```
「出力は200字以内でまとめてください。」
```

### 問題：文体が固い
**解決策：** 温かみのある表現を指示
```
「親しみやすく、温かい表現を使ってください。」
```

### 問題：形式が崩れる
**解決策：** 出力形式を明確に指定
```
必ず以下の3つの見出しで構成してください：
【今日の活動】
【子どもの様子】
【明日への連携】
```

## ベストプラクティス

1. **段階的なテスト**
   - まずシンプルなプロンプトから始める
   - 徐々に詳細を追加していく

2. **フィードバックループ**
   - 実際の出力を確認
   - 必要に応じてプロンプトを調整

3. **バージョン管理**
   - 良いプロンプトは保存
   - 改善履歴を記録

4. **安全性の確保**
   - 個人情報の扱いを明記
   - 出力内容の確認プロセス
