# 第2回：重要ポイントだけを抽出して確認時間を5分の1にする！日報と週報を自動要約する確認時短フロー

**動画時間**: 10分
**学習目標**: 提出された日報・週報をGeminiが自動要約し、管理者が要点だけを確認できるエージェントを作成する

---

## 1. 導入（0:00-1:30）

### 講師セリフ

「前回は、日報・週報の提出依頼を自動化しました。

でも、提出された報告書を全部読むのは大変ですよね？

**（画面：日報の山）**

- 10人のチームなら、毎日10件の日報
- 週報も合わせると、週60件以上
- 全部読むと1〜2時間かかる

今回は、**Geminiによる自動要約**で、確認時間を**5分の1**に短縮します。

**（画面：Before/After）**

| Before | After |
|--------|-------|
| 全文を読む | 要点だけ確認 |
| 1件5分 | 1件1分 |
| 重要ポイントを見落とし | 課題・相談を自動ハイライト |」

---

## 2. エージェント設計（1:30-3:00）

### 講師セリフ

「今回作成するエージェントの設計です。

**（画面：エージェント構成図）**

### Report Summary Agent

**目的**: 提出された日報・週報を自動要約して管理者に配信

**トリガー**: Gmail受信（日報・週報メール）

**処理フロー**:

```
Gmail受信（件名: 日報 or 週報）
    ↓
メール本文を取得
    ↓
Geminiで要約・分析
    ↓
要約結果をChatに送信
    ↓
（オプション）Sheetsに記録
```

### 要約のポイント

**（画面：要約観点）**

Geminiに抽出させる項目：

1. **業務サマリー**（3行以内）
2. **達成した成果**
3. **課題・懸念事項**（★重要）
4. **相談・依頼事項**（★重要）
5. **翌日/翌週の予定**」

---

## 3. エージェント作成（3:00-6:30）

### 講師セリフ

「それでは、Studioでエージェントを作成します。

**（画面：Studio画面）**

### Step 1: 新規エージェント作成

- Name: **Report Summary Agent**
- Description: Summarizes daily/weekly reports and highlights key points

### Step 2: トリガー設定

**（画面：Trigger設定）**

- Trigger Type: **Gmail**
- Condition: **When email received**
- Filter:
  - Subject contains: 日報 OR 週報
  - From: @yourcompany.com

### Step 3: Gemini要約プロンプト

**（画面：Gemini設定）**

```
You are a management assistant analyzing team reports.

Report content:
---
{email_body}
---

Analyze this report and extract:

1. **業務サマリー** (3 bullet points max)
2. **達成した成果** (if any)
3. **課題・懸念事項** (mark with 🚨 if critical)
4. **相談・依頼事項** (mark with 📢 if needs response)
5. **明日/来週の予定**

Also provide:
- Overall assessment: 順調 / 注意 / 要フォロー
- Recommended action for manager (if any)

Output in Japanese. Be concise.
```

### Step 4: Chat送信設定

**（画面：Chat設定）**

- Space: 管理者用チャンネル
- Format:

```
📋 **日報要約** - {sender_name} ({date})

{gemini_summary}

---
📎 元メール: {email_link}
```

### Step 5: Sheets記録（オプション）

**（画面：Sheets設定）**

記録する項目：
- 日付
- 送信者
- 評価（順調/注意/要フォロー）
- 課題有無
- 相談有無」

---

## 4. 実践的な活用例（6:30-8:00）

### 講師セリフ

「このエージェントの活用シーンを見てみましょう。

**（画面：活用例）**

### 活用例1: 朝の一括確認

毎朝、前日の日報要約がChatにまとまっている
→ 5分で全員の状況を把握

### 活用例2: 課題の早期発見

「🚨」マークの報告を優先チェック
→ 問題が大きくなる前に対処

### 活用例3: 週次レビュー

Sheetsに蓄積したデータから傾向分析
→ 誰がいつ課題を抱えやすいか把握

### カスタマイズ例

**（画面：カスタマイズオプション）**

- 「要フォロー」評価の場合のみ通知
- 特定キーワード（退職、体調不良など）を検知してアラート
- 部門別にChat送信先を分岐」

---

## 5. テストと確認（8:00-9:00）

### 講師セリフ

「エージェントをテストします。

**（画面：テスト実行）**

### テスト手順

1. テスト用の日報メールを送信
2. エージェントの実行を確認
3. Chat通知の内容を確認
4. 要約の精度を確認

### 確認ポイント

- 課題・相談が正しく抽出されているか
- 評価（順調/注意/要フォロー）が適切か
- 日本語の品質は問題ないか」
