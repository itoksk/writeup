属人管理から卒業するための基礎設計！
条件分岐で日報・週報提出依頼を自動化する

管理業務の課題を整理し、Google Workspace Studioで曜日判定による条件分岐を実装。 日報・週報提出依頼の自動化エージェントを作成して、属人管理から卒業するための基礎を構築します。

BEFORE：受講前
  • 日報確認に毎日30分〜1時間
  • 提出依頼を毎日手動送信
  • 送り忘れが月に数回発生
  • 管理業務が属人化
AFTER：受講後
  • 日報・週報依頼が自動送信
  • 曜日判定で条件分岐
  • 送り忘れゼロを実現
  • 管理業務70%削減の第一歩
Goal

学習目標

管理業務の課題を整理し、Google Workspace Studioで曜日判定による条件分岐を実装。 日報・週報提出依頼の自動化エージェントを作成して、属人管理から卒業するための基礎を構築する
Problems

管理業務の4つの課題

ISSUE 01

提出依頼の手間

毎日17時に日報提出依頼を送信。金曜日は週報も追加で依頼する必要がある

ISSUE 02

確認作業の負荷

10人分の日報を毎日読んで確認。内容を理解するだけで60分以上かかる

ISSUE 03

品質のばらつき

記載内容の質が人によって異なり、個別に指摘や修正依頼が必要

ISSUE 04

進捗把握の遅れ

チーム全体の状況把握に時間がかかり、課題の早期発見が困難

Before / After

導入効果

Before：属人管理
  • 日報確認に毎日30分〜1時間かかる
  • 未提出者への個別リマインドが必要
  • 承認待ち案件を放置してしまう
  • 進捗把握が属人化している
  • 提出依頼を毎日手動送信
  • 送り忘れが月に数回発生
After：ゼロタッチ
  • 提出依頼は自動送信で完全自動化
  • 未提出は自動検知・リマインド
  • 承認依頼は自動通知で漏れゼロ
  • ダッシュボードで即時可視化
  • 17時に自動配信、送り忘れゼロ
  • 金曜日は自動で週報依頼も追加

Steps

作成ステップ

1

新規エージェント作成

studio.workspace.google.com にアクセス → 「新規作成」をクリック → エージェント名「日報週報提出依頼」を入力

2

スケジュールスターター設定

スターターで「Scheduled（スケジュール）」を選択 → 頻度: 毎日 → 時刻: 17:00 を設定して定時実行を有効化

3

Geminiステップで曜日判定

ステップを追加 → 「Gemini」を選択 → 下記の曜日判定プロンプトを設定し、曜日に応じた出力を生成

4

条件分岐ステップ設定

ステップを追加 → 「条件分岐」を選択 → Geminiの出力が「NO_SEND」以外の場合のみ次のステップを実行

5

Chat DMステップ設定

Chat DM送信のステップを追加。自分への通知を受け取り、必要に応じて手動でチームへ転送。※Chat通知は個人へのDMのみ対応（スペースへの投稿・外部送信は未対応）

6

テスト実行と有効化

「テスト」で動作確認を行い、曜日判定とメッセージ生成が正しく機能することを確認後、「有効化」で本番稼働を開始

Prompt

曜日判定プロンプト

クリックでコピー
Today is {current_date}. Determine the day of the week and generate the appropriate message: If Monday to Thursday: - Generate a daily report request in Japanese - Include deadline (today 23:59) - Keep it brief and professional If Friday: - Generate a weekly report request in Japanese - Include deadline (today 23:59) - Ask for weekly summary and next week's plan If Saturday or Sunday: - Output: "NO_SEND" Output the message only, no explanations.
Templates

メッセージテンプレート

日報提出依頼（月〜木）

クリックでコピー
件名：【提出依頼】本日の日報 チームの皆さん お疲れ様です。 本日の日報提出をお願いいたします。 ■ 提出期限 本日 23:59まで ■ 記載項目 ・本日の業務内容 ・進捗状況 ・課題・相談事項 ・明日の予定 よろしくお願いいたします。

週報提出依頼（金曜日）

クリックでコピー
件名：【提出依頼】今週の週報 チームの皆さん お疲れ様です。 今週の週報提出をお願いいたします。 ■ 提出期限 本日 23:59まで ■ 記載項目 ・今週の業務サマリー ・達成した成果 ・課題・懸念事項 ・来週の計画 ・共有事項 よろしくお願いいたします。

注意: Chat通知は自分へのDM（ダイレクトメッセージ）のみ送信可能です。スペースへの投稿機能・外部への送信機能は正式リリースで削除されました。チーム全体への通知が必要な場合は、自分へのDMを受け取った後、手動で転送してください。

Condition

条件分岐ステップ設定

クリックでコピー
IF Gemini出力 != "NO_SEND": → Chat DM送信ステップ実行（自分へ） → 必要に応じて手動でチームへ転送 ELSE: → 何もしない (土日・祝日は自動スキップ)
Test

テスト確認ポイント

動作確認チェックリスト

  • 曜日判定が正しく行われているか（月〜木、金、土日の判別）
  • メッセージの内容が適切か（日報/週報の切り替えが正しい）
  • 送信先（Chat DM）が正しく設定されているか
  • 土日は送信されないか（NO_SEND判定が機能しているか）
