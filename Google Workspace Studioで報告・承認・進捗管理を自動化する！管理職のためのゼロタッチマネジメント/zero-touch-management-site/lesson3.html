合格・要修正・再提出を条件分岐で仕分ける！
チェックリストを自動で判定する審査フロー

日報・週報の品質をチェックリストで自動判定し、APPROVED（合格）/ NEEDS_REVISION（要修正）/ RESUBMIT（再提出）の3段階で条件分岐処理を実装。 提出者への即時フィードバックと管理者への通知を自動化するQuality Check Agentを構築します。

BEFORE：受講前
  • 品質チェックを毎日30分手動で実施
  • 判定基準が曖昧で属人的
  • フィードバックが翌日以降に遅延
  • 同じ指摘を繰り返す非効率
AFTER：受講後
  • AIが自動判定でチェック時間90%削減
  • 判定基準が統一・明確・公平に
  • 即時フィードバックで改善サイクル加速
  • Quality Check Agentを構築
Goal

学習目標

Judgment

判定基準の定義

APPROVED

全必須項目OK
軽微な指摘0〜1件

→ 承認通知を送信

NEEDS_REVISION

必須項目は満たす
軽微な指摘2件以上

→ 修正依頼を送信

RESUBMIT

必須項目に不備あり
重大な欠落がある

→ 再提出依頼 + 管理者通知

エージェントフロー

提出物受信

Configuration

エージェント設計

項目 設定値
エージェント名 品質チェック
説明 提出物の品質を自動チェックし、適切なアクションにルーティング
スターター Gmail - Email received または Forms - Form submitted
処理ステップ Geminiによるチェックリスト評価
条件分岐 APPROVED / NEEDS_REVISION / RESUBMIT
出力（提出者へ） Chat DM（自分）でフィードバック受信 → 手動で転送 ※外部送信は未対応
出力（Sheets） ステータス更新と判定詳細の記録
出力（管理職へ） Chat DM通知（RESSUBMITの場合のみ）
Checklist

日報チェックリスト項目

項目 必須 チェック内容 NG例
業務内容 必須 具体的な作業が記載されているか 「業務した」のみ
進捗状況 必須 数値や達成率が含まれているか 「順調」のみ
課題 任意 課題がある場合に記載されているか 課題あるのに未記載
翌日予定 必須 翌日の計画が記載されているか 「引き続き」のみ
文字数 必須 最低100文字以上か 50文字未満

ポイント: チェックリストは業務に合わせてカスタマイズ可能。週報の場合は「今週の成果」「来週の計画」などを追加します。

Steps

作成ステップ

1

新規エージェント作成

studio.workspace.google.com → 「新規作成」→ エージェント名「品質チェック」を入力

2

スターター設定

スターター: Gmail - Email received または Forms - Form submitted → フィルター: 件名に「提出」「報告」を含む

3

Geminiステップでチェック

ステップを追加 → 「Gemini」を選択 → 下記のチェックリスト評価プロンプトを設定。各項目をOK/NGで判定

4

条件分岐ステップ設定

ステップを追加 → 「条件分岐」→ Gemini出力のJUDGMENTフィールドでAPPROVED / NEEDS_REVISION / RESUBMIT に分岐

5

Chat DMステップ設定

各分岐にChat DM送信ステップを追加。自分宛に通知を受け取り、承認・修正依頼・再提出依頼の結果を確認後、手動で提出者へ転送 ※外部送信は未対応

6

Sheets・Chat DMステップ設定

Sheetsステップでステータス列を更新。RESSUBMITの場合のみChat DMで管理職に通知（※スペースへの投稿は削除済み）

Prompt

品質チェックプロンプト

クリックでコピー
You are a quality control assistant checking submissions against a checklist. Submission content: --- {submission_content} --- Checklist: 1. 業務内容: 具体的な作業が記載されているか (Required) 2. 進捗状況: 数値や達成率が含まれているか (Required) 3. 課題: 課題がある場合に記載されているか (Optional) 4. 翌日予定: 翌日の計画が記載されているか (Required) 5. 文字数: 最低100文字以上か (Required) For each item, evaluate: - Status: OK / NG / N/A - Comment: If NG, provide specific improvement suggestion Then provide overall judgment: - APPROVED: All required items OK, minor issues 0-1 - NEEDS_REVISION: All required items OK, minor issues 2+ - RESUBMIT: Any required item NG Output format: --- CHECKLIST_RESULT: 1. 業務内容: [OK/NG] - [具体的なコメント] 2. 進捗状況: [OK/NG] - [具体的なコメント] 3. 課題: [OK/NG/N/A] - [具体的なコメント] 4. 翌日予定: [OK/NG] - [具体的なコメント] 5. 文字数: [OK/NG] - [現在の文字数] JUDGMENT: [APPROVED/NEEDS_REVISION/RESUBMIT] FEEDBACK: [提出者向けの具体的なフィードバック（日本語）] ---
Logic

条件分岐ロジック

クリックでコピー
IF JUDGMENT == "APPROVED": → Chat DM（自分）で承認結果を受信 → Update status to "承認済" (Sheets) → 手動で提出者へ承認通知を転送 ELSE IF JUDGMENT == "NEEDS_REVISION": → Chat DM（自分）で修正依頼内容を受信 → Update status to "要修正" (Sheets) → Include specific feedback from CHECKLIST_RESULT → 手動で提出者へ修正依頼を転送 (+24h期限) ELSE IF JUDGMENT == "RESUBMIT": → Chat DM（自分）で再提出依頼内容を受信 → Update status to "再提出" (Sheets) → 手動で提出者へ再提出依頼を転送 (+48h期限)

注意: 現在のWorkspace Studioでは外部へのメール・Chat送信は未対応です。フィードバックは自分へのDMで受け取り、内容を確認後、手動で提出者へ転送してください。

Test

動作確認チェックリスト
