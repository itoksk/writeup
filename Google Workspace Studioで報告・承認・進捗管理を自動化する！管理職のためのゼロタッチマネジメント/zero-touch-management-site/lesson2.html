重要ポイントだけを抽出して確認時間を1/5にする！
日報と週報を自動要約する確認時短フロー

日報・週報メールをGeminiで自動要約し、業務サマリー・課題・依頼事項を構造化して抽出。 管理職の確認時間を60分から12分に短縮し、重要情報の見落としをゼロにするReport Summary Agentを構築します。

BEFORE：受講前
  • 日報10人分を毎日30〜60分かけて読む
  • 重要情報が本文中に埋もれる
  • 依頼事項を見落としがち
  • 対応優先度が不明確
AFTER：受講後
  • AI要約で確認時間12分/日（80%削減）
  • 課題・依頼事項が自動抽出される
  • 評価ラベルで対応要否が一目瞭然
  • Report Summary Agentを構築
Goal

学習目標

以下のプロンプトをコピーして、Google Workspace Studio で実行すると、このレッスンのエージェントを自動生成できます。

クリックでコピー
日報・週報要約エージェントを作成してください。 【エージェント名】 日報週報要約 【説明】 日報・週報を自動要約し、重要ポイントをハイライト 【スターター】 Gmail - Email received（メール受信時） フィルター: 件名に「日報」または「週報」を含む 【処理フロー】 1. Geminiステップで以下を抽出: - 業務サマリー（3項目まで） - 達成した成果 - 課題・懸念事項（重要な場合はALERTマーク） - 相談・依頼事項（対応が必要な場合はREQUESTマーク） - 明日/来週の予定 - 評価: 順調 / 注意 / 要フォロー 2. Chat DMステップ: - 要約結果を自分へ通知 3. Sheetsステップ（オプション）: - 日付、送信者、評価、要約を記録 【出力先】 - Chat DM（自分へ通知） - Sheets（ログ記録用） このエージェントを作成してください。

ポイント: 上記プロンプトをコピーして Google Workspace Studio に貼り付けると、エージェントのベースが自動生成されます。

Steps

作成ステップ

1

新規エージェント作成

studio.workspace.google.com → 「新規作成」 → エージェント名「日報週報要約」を入力

2

Gmailスターター設定

スターター: Gmail - Email received（メール受信時）→ フィルター: 件名に「日報」または「週報」→ 送信元: @yourcompany.com

3

Geminiステップで要約生成

ステップを追加 → 「Gemini」を選択 → 下記の日報要約プロンプトを設定し、構造化された要約を自動生成

4

Chat DMステップ設定

ステップを追加 → Chat DM送信を選択 → 自分へ通知を送信。※スペースへの投稿・外部送信は未対応のため、必要に応じて手動転送

5

Sheetsステップ設定（オプション）

ステップを追加 → Sheets「行を追加」→ スプレッドシート: 日報管理シート → 日付・送信者・評価・要約を自動記録

6

テスト実行と有効化

「テスト」でテスト日報メールを送り要約生成を確認 → Chat DM通知とSheets記録の動作確認 → 「有効化」で本番稼働

Prompt

日報要約プロンプト

クリックでコピー
You are a management assistant analyzing team reports. Report content: --- {email_body} --- Analyze this report and extract: 1. **業務サマリー** (3 bullet points max) 2. **達成した成果** (if any) 3. **課題・懸念事項** (mark with ALERT if critical) 4. **相談・依頼事項** (mark with REQUEST if needs response) 5. **明日/来週の予定** Also provide: - Overall assessment: 順調 / 注意 / 要フォロー - Recommended action for manager (if any) Output in Japanese. Be concise.

ポイント: ALERTとREQUESTのマークを使うことで、確認時に目を引く重要項目を視覚的に識別できます。

Assessment

評価ラベルの定義

順調

問題なし、予定通り進行

注意

軽微な遅れ、経過観察

要フォロー

課題あり、支援検討

要支援

緊急対応、即時介入

活用法: 「要フォロー」以上のレポートは優先的に原文を確認し、必要に応じて1on1をスケジュール。

Template

Chat通知テンプレート

クリックでコピー
**日報要約** - {sender_name} ({date}) ■ 業務サマリー {business_summary} ■ 達成した成果 {achievements} ■ 課題・懸念事項 {issues} ■ 相談・依頼事項 {requests} ■ 明日の予定 {tomorrow_plan} --- 評価: {assessment} 元メール: {email_link}

注意: Chat通知は自分へのDM（ダイレクトメッセージ）のみ送信可能です。スペースへの投稿機能・外部への送信機能は正式リリースで削除されました。

Sheets

Sheets記録テンプレート

列 内容 形式
A 日付 YYYY-MM-DD
B 送信者 氏名
C レポートタイプ 日報 / 週報
D 評価 順調 / 注意 / 要フォロー / 要支援
E 課題有無 あり / なし
F 相談有無 あり / なし
G 要約（短縮版） テキスト（200文字以内）

活用法: このデータを使って月次で「課題発生頻度」「評価トレンド」をピボットテーブルで分析し、チームの傾向を把握できます。

Usage

出力の活用シーン

朝一 一括確認

毎朝、前日の日報要約がChatにまとまっている → 5分で全員の状況を把握して1日をスタート

優先対応 課題の早期発見

ALERTマークの報告を優先チェック → 問題が大きくなる前に対処し、チームの安定稼働を維持

週次 レビュー会議

週報要約を会議資料として活用 → 各メンバーの状況を事前に把握した上でディスカッション

月次 傾向分析

Sheetsに蓄積したデータから傾向分析 → 誰がいつ課題を抱えやすいか、パターンを把握
