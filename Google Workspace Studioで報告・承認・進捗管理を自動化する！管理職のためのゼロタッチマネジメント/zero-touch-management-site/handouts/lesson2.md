# 第2回：重要ポイントだけを抽出して確認時間を1/5にする！日報と週報を自動要約する確認時短フロー

## 学習目標

日報・週報メールをGeminiで自動要約し、業務サマリー・課題・依頼事項を構造化して抽出。管理職の確認時間を60分から12分に短縮し、重要情報の見落としをゼロにするReport Summary Agentを構築します。

---

## BEFORE / AFTER

| Before（受講前） | After（受講後） |
|---|---|
| 日報10人分を毎日30〜60分かけて読む | AI要約で確認時間12分/日（80%削減） |
| 重要情報が本文中に埋もれる | 課題・依頼事項が自動抽出される |
| 依頼事項を見落としがち | 評価ラベルで対応要否が一目瞭然 |
| 対応優先度が不明確 | Report Summary Agentを構築 |

---

## エージェント作成プロンプト

以下のプロンプトをコピーして、Google Workspace Studio（https://studio.workspace.google.com/）で実行してください。

```
日報・週報要約エージェントを作成してください。

【エージェント名】
日報週報要約

【説明】
日報・週報を自動要約し、重要ポイントをハイライト

【スターター】
Chat - チャットメッセージを受け取ったとき
会話の種類: スペースと名前付きグループDM
スペース: 日報専用スペース（事前に作成しておく）

【処理フロー】
1. Geminiステップ（要約生成）:
   - 業務サマリー（3項目まで）
   - 達成した成果
   - 課題・懸念事項（重要な場合はALERTマーク）
   - 相談・依頼事項（対応が必要な場合はREQUESTマーク）
   - 明日/来週の予定
   - 評価: 順調 / 注意 / 要フォロー

2. 抽出ステップ（Sheets記録用データ抽出）:
   Geminiの出力から以下を個別変数として抽出:
   - report_type: 日報 または 週報
   - assessment: 順調 / 注意 / 要フォロー
   - has_issues: あり / なし
   - has_requests: あり / なし
   - summary_short: 要約（200文字以内）

3. Chat DMステップ:
   - 要約結果を自分へ通知

4. Sheetsステップ:
   - 抽出した変数を使って行を追加
   - 列: 日付, 送信者, レポートタイプ, 評価, 課題有無, 相談有無, 要約

【出力先】
- Chat DM（自分へ通知）
- Sheets（ログ記録用）

【事前準備】
1. Google Chatで「日報提出」などの名前でスペースを作成し、チームメンバーを招待
2. Googleスプレッドシートで日報管理シートを作成（列: 日付, 送信者, レポートタイプ, 評価, 課題有無, 相談有無, 要約）

このエージェントを作成してください。
```

## 作成ステップ

1. **新規エージェント作成**: studio.workspace.google.com → 「新規作成」 → エージェント名「日報週報要約」を入力

2. **Chatスターター設定**: スターター: Chat - チャットメッセージを受け取ったとき → 会話の種類: スペースと名前付きグループDM → スペース: 日報専用スペースを選択

3. **Geminiステップで要約生成**: ステップを追加 → 「Gemini」を選択 → 下記の日報要約プロンプトを設定し、構造化された要約を自動生成

4. **抽出ステップでデータを変数化**: ステップを追加 → 「抽出」を選択 → Gemini出力から report_type, assessment, has_issues, has_requests, summary_short を個別変数として抽出

5. **Chat DMステップ設定**: ステップを追加 → Chat DM送信を選択 → 自分へ通知を送信。※スペースへの投稿・外部送信は未対応のため、必要に応じて手動転送

6. **Sheetsステップ設定**: ステップを追加 → Sheets「行を追加」→ 日報管理シートを選択 → 抽出した変数（日付, 送信者, report_type, assessment, has_issues, has_requests, summary_short）をマッピング

7. **テスト実行と有効化**: 「テスト」でテスト日報をスペースに投稿し要約生成を確認 → Chat DM通知とSheets記録の動作確認 → 「有効化」で本番稼働

## 日報要約プロンプト

```
You are a management assistant analyzing team reports.

Report content:
---
{email_body}
---

Analyze this report and extract:

1. **業務サマリー** (3 bullet points max)
2. **達成した成果** (if any)
3. **課題・懸念事項** (mark with ALERT if critical)
4. **相談・依頼事項** (mark with REQUEST if needs response)
5. **明日/来週の予定**

Also provide:
- Overall assessment: 順調 / 注意 / 要フォロー
- Recommended action for manager (if any)

Output in Japanese. Be concise.
```

---

## 評価ラベルの定義

| 評価 | 説明 |
|---|---|
| 順調 | 問題なし、予定通り進行 |
| 注意 | 軽微な遅れ、経過観察 |
| 要フォロー | 課題あり、支援検討 |
| 要支援 | 緊急対応、即時介入 |

**活用法:** 「要フォロー」以上のレポートは優先的に原文を確認し、必要に応じて1on1をスケジュール。

## Sheets記録テンプレート

| 列 | 内容 | 抽出変数 | 形式 |
|---|---|---|---|
| A | 日付 | （自動: 現在日時） | YYYY-MM-DD |
| B | 送信者 | （自動: メッセージ送信者） | 氏名 |
| C | レポートタイプ | `report_type` | 日報 / 週報 |
| D | 評価 | `assessment` | 順調 / 注意 / 要フォロー |
| E | 課題有無 | `has_issues` | あり / なし |
| F | 相談有無 | `has_requests` | あり / なし |
| G | 要約（短縮版） | `summary_short` | テキスト（200文字以内） |

**活用法:** このデータを使って月次で「課題発生頻度」「評価トレンド」をピボットテーブルで分析し、チームの傾向を把握できます。

## 注意事項

Chat通知は自分へのDM（ダイレクトメッセージ）のみ送信可能です。スペースへの投稿機能・外部への送信機能は正式リリースで削除されました。チーム全体への通知が必要な場合は、自分へのDMを受け取った後、手動で転送してください。
