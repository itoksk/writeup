# Google Workspace Studioで"報告・承認・進捗管理"を自動化する！管理職のためのゼロタッチマネジメント

## コース概要

Google Workspace Studio と Gemini を活用して、管理職の「報告確認」「承認処理」「進捗管理」を70%効率化するコースです。

### 対象者
- Google Workspace を導入している組織の管理職
- 日報・週報の確認、承認処理に時間を取られている方
- チームの進捗管理を効率化したい方

### 前提条件
- Google Workspace Business Starter/Standard/Plus、Enterprise Standard/Plus、または Education Fundamentals/Standard/Plus エディション
- または Google AI Ultra for Business / Google AI Pro for Education アドオン
- **18歳未満のユーザーはAI機能を使用できません**

## カリキュラム（全5回）

### 第1回：属人管理から卒業するための基礎設計！条件分岐で"日報・週報提出依頼"を自動化する

**学習目標**: 日報・週報の提出依頼を自動化し、催促作業をゼロにする

**Before → After**:
- Before: 日報・週報を提出してくれない、催促メッセージを毎週手で送っている
- After: エージェントが提出期限前に自動リマインド、提出状況がスプレッドシートに自動集約

**構築するエージェント**:
- スターター: スケジュール（Scheduled）- 毎日9:00 / 毎週金曜17:00
- ステップ:
  1. Gemini: 提出依頼メッセージを生成
  2. Gmail: メンバーに一斉送信
  3. Sheets: 送信ログを記録
  4. 条件分岐: 未提出者への追加リマインド

**ポイント**:
- 曜日・時間に応じたスケジュール設定
- メール文面はGeminiで自然な文章を生成
- 提出フォームへのリンクを自動挿入

---

### 第2回：重要ポイントだけを抽出して確認時間を1/5にする！日報と週報を自動要約する確認時短フロー

**学習目標**: Geminiで日報・週報を自動要約し、確認時間を80%削減する

**Before → After**:
- Before: 日報を読むだけで毎日30〜60分、重要事項を読み飛ばすリスク
- After: 「重要ポイント」「懸念」「フォロー必要箇所」が整理されて通知、確認は5〜7分

**構築するエージェント**:
- スターター: フォーム回答（Form submitted）
- ステップ:
  1. Gemini: 日報内容を要約
     - プロンプト例: 「以下の日報から【重要点】【懸念】【次アクション】を抽出してください」
  2. 条件分岐: 懸念ありの場合のみChat DM送信
  3. Gmail: 管理職へ要約サマリー送信
  4. Sheets: 要約結果を記録

**ポイント**:
- 要約形式を統一すると日々の比較が容易
- 「懸念あり」の場合のみ即時通知で、ノイズを削減
- Chat通知は個人へのDMのみ利用可能（スペースへの投稿機能は削除済み）

---

### 第3回：合格・要修正・再提出を条件分岐で仕分ける！チェックリストを自動で判定する審査フロー

**学習目標**: 提出物の品質を自動判定し、適切なフィードバックを自動送信する

**Before → After**:
- Before: 資料を開いて1ページずつ読む、判断基準が曖昧で評価がブレる
- After: Geminiが基準に照らして自動判定、合格物だけ管理職へ届く

**構築するエージェント**:
- スターター: フォーム回答（Form submitted）- ファイル添付あり
- ステップ:
  1. Drive: 添付ファイルを保存
  2. Gemini: チェック基準に基づき判定
     - プロンプト例: 「以下の資料が基準を満たしているか判定し、合格/要修正/再提出を出力してください」
  3. 条件分岐:
     - 合格 → 承認通知を管理職へ
     - 要修正 → 修正依頼をメンバーへ
     - 再提出 → 再提出案内をメンバーへ
  4. Sheets: 判定結果を記録

**ポイント**:
- チェック基準を明確にするほど判定精度UP
- フォーマットを統一すると自動判定が安定
- 再提出追跡も自動化で負荷ゼロ

### 第4回：Driveとスプレッドシートを連携する！進捗ダッシュボードが自動で更新される仕組み

**学習目標**: 資料提出・承認状況をリアルタイムで可視化するダッシュボードを構築

**Before → After**:
- Before: 手作業でスプレッドシート更新、資料提出状況が把握しづらい
- After: Driveへの資料追加で自動更新、ダッシュボード常時最新版

**構築するエージェント**:
- スターター: Driveファイル追加（File added to folder）
- ステップ:
  1. ファイル情報を取得（名前・日付・アップロード者）
  2. Sheets: 管理シートに行追加
  3. （オプション）Looker Studioでダッシュボード化

**ポイント**:
- 部署別フォルダを作成して整理
- Sheetsをデータソースにすればダッシュボード構築が容易
- 進捗監視作業が完全ゼロに

---

### 第5回：誰に何をいつ伝えるかを整理し遅延ゼロを実現する！承認プロセスを自動化するワークフロー

**学習目標**: 承認依頼の自動送信・リマインド・完了通知を実装し、承認渋滞を解消

**Before → After**:
- Before: 承認依頼がメールに埋もれて気づかない、「まだですか？」の催促が精神的に負担
- After: 提出物があれば自動で承認依頼が届く、期限前に自動リマインド

**構築するエージェント**:
- スターター: フォーム回答（Form submitted）
- ステップ:
  1. Gmail: 管理職へ「承認依頼」を送信（承認フォームURLを挿入）
  2. スケジュールトリガー: 期限前に自動リマインド
  3. 承認フォーム回答をトリガーに:
     - 条件分岐: 承認 → メンバーへ承認完了通知
     - 条件分岐: 差し戻し → 修正依頼メール
  4. Sheets: 承認結果を記録

**ポイント**:
- 承認担当者を明確化（役職ごとに分岐可能）
- 承認期限を設定し、期限前リマインドを自動化
- 承認履歴は1on1・評価面談でも活用可能

## 正式リリース版での注意点

### 用語の変更
- 「Flow」「Workspace Flow」→「Workspace Studio」「エージェント」
- 「トリガー」→「スターター（Starter）」
- 「アクション」→「ステップ（Step）」

### 機能の変更点
1. **Chat通知**: 個人へのDM（ダイレクトメッセージ）のみ利用可能。スペースへの投稿機能は正式リリースで削除されました。
2. **スターターの種類**:
   - Gmail: メール受信（Email received）
   - Calendar: 会議に基づいて（イベントの前後に実行）
   - Drive: ファイル追加（File added to folder）
   - Forms: フォーム回答（Form submitted）
   - スケジュール: 定期実行（Scheduled）
   - 手動実行（Manual）

### カスタムステップについて

Google Workspace Studioでは、Apps Scriptを使って独自のカスタムステップを作成できます。ただし、**ノーコードの良さを活かすため、標準ステップで対応できる場合はカスタムステップは不要**です。

**カスタムステップが有効なケース**:
- 社内独自システムとのAPI連携
- 標準ステップにない特殊な計算・変換処理
- 複雑なビジネスロジックの実装

**注意**: カスタムステップの作成にはApps Scriptの知識が必要です。まずは標準ステップで業務を自動化し、どうしても対応できない場合にのみ検討しましょう。

参考: [カスタムステップのクイックスタート](https://developers.google.com/workspace/add-ons/studio/quickstart-calculator?hl=ja)

## Workspace内で完結するメリット

Google Workspace Studioの最大の特徴は、**すべてがGoogle Workspaceの中で完結する**ことです。

- **APIキー管理が不要**: OpenAI APIキーやGoogle Cloud APIキーの取得・管理が不要
- **認証情報の漏洩リスクなし**: APIキーをコードに埋め込む必要がない
- **データがGoogle内に留まる**: サードパーティサービスにデータを送信する必要がない
- **課金・コスト管理が簡単**: API使用量の計算や従量課金の心配がない
- **管理者による一元管理**: Google管理コンソールから一元的に制御可能
