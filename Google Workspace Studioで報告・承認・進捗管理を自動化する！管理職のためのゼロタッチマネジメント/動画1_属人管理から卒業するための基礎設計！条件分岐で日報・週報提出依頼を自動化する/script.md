# 第1回：属人管理から卒業するための基礎設計！条件分岐で日報・週報提出依頼を自動化する

**動画時間**: 10分
**学習目標**: 管理業務の課題を整理し、条件分岐を使った日報・週報提出依頼の自動化エージェントを作成する

---

## 1. 導入（0:00-2:00）

### 講師セリフ

「管理職のみなさん、こんな悩みを抱えていませんか？

- 日報・週報の提出確認に毎日時間を取られる
- 承認依頼が来ても気づかず放置してしまう
- 進捗状況の把握が属人化している
- リマインドを手動で送り続けている

このコースでは、**Google Workspace Studio**を使って、これらの管理業務を**ゼロタッチ**で自動化します。

**（画面：コース全体像）**

5回のレッスンで、以下のエージェントを作成します：

1. **日報・週報提出依頼**の自動化
2. **日報・週報の自動要約**
3. **チェックリストの自動判定**
4. **進捗ダッシュボード**の自動更新
5. **承認プロセス**の自動化

今回は、条件分岐を使った**日報・週報提出依頼の自動化**を実装します。」

## 2. 管理業務の課題分析（2:00-3:30）

### 講師セリフ

「まず、管理業務でよくある課題を整理しましょう。

**（画面：課題一覧）**

### よくある課題

| 課題 | 影響 |
|------|------|
| 日報・週報の提出確認 | 毎日30分〜1時間 |
| 未提出者へのリマインド | 個別対応で負担大 |
| 承認待ち案件の放置 | 業務の遅延 |
| 進捗把握の属人化 | 引き継ぎ困難 |
| 報告内容の確認 | 重要ポイントを見落とし |

### ゼロタッチマネジメントとは

**（画面：ゼロタッチの概念図）**

「ゼロタッチ」とは、**管理者が手を動かさなくても**業務が回る仕組みです。

- 提出依頼 → **自動送信**
- 未提出確認 → **自動検知**
- リマインド → **自動送信**
- 進捗把握 → **ダッシュボード自動更新**
- 承認依頼 → **自動通知**」

## 3. エージェント設計（3:30-5:00）

### 講師セリフ

「今回作成するエージェントの設計を見ていきましょう。

**（画面：エージェント構成図）**

### Daily Report Request Agent

**目的**: 毎日定時に日報提出依頼を自動送信

**トリガー**: Schedule（毎日17:00）

**条件分岐**:
- 月〜木 → 日報提出依頼
- 金 → 週報提出依頼
- 土日 → 送信しない

**出力先**: Gmail または Google Chat

### フロー図

```
Schedule (17:00)
    ↓
曜日判定
    ↓
[月〜木] → 日報提出依頼メール送信
[金] → 週報提出依頼メール送信
[土日] → 何もしない
```」

---

## 4. エージェント作成（5:00-8:00）

### 講師セリフ

「それでは、Google Workspace Studioでエージェントを作成します。

**（画面：Studio画面）**

### Step 1: 新規エージェント作成

1. Studioにアクセス
2. 「Create Agent」をクリック
3. エージェント名: **Daily Report Request**
4. Description: Sends daily/weekly report request based on day of week

### Step 2: スターター設定

**（画面：Trigger設定）**

- Trigger Type: **Schedule**
- Frequency: **Daily**
- Time: **17:00**
- Timezone: **Asia/Tokyo**

### Step 3: 条件分岐の設定

**（画面：Gemini設定）**

Geminiに曜日を判定させ、適切なメッセージを生成します。

**プロンプト**:

```
Today is {current_date}.

Determine the day of the week and generate the appropriate message:

If Monday to Thursday:
- Generate a daily report request in Japanese
- Include deadline (today 23:59)
- Keep it brief and professional

If Friday:
- Generate a weekly report request in Japanese
- Include deadline (today 23:59)
- Ask for weekly summary and next week's plan

If Saturday or Sunday:
- Output: "NO_SEND"

Output the message only, no explanations.
```

### Step 4: 条件分岐アクション

**（画面：Action設定）**

```
IF Gemini output != "NO_SEND":
    → Send Gmail to {team_distribution_list}
    → Send Chat message to {team_space}
ELSE:
    → Do nothing
```

### Step 5: メールテンプレート設定

**（画面：Gmail設定）**

- To: {team_distribution_list}
- Subject: 【提出依頼】{report_type}
- Body: {gemini_output}」

## 5. テストと確認（8:00-9:00）

### 講師セリフ

「作成したエージェントをテストします。

**（画面：テスト実行）**

### テスト手順

### 確認ポイント

- メッセージの内容が適切か

- 土日は送信されないか」
