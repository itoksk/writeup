カスタムリソースとは

カスタムリソースは、複数のフィールドを持つ複合データ構造を定義する仕組みです。 例えば「顧客」リソースには、名前、メールアドレス、会社名などの複数フィールドが含まれます。

FEATURE 01

複合データ構造

単一の変数では表現できない複数フィールドを持つデータを定義できます。

FEATURE 02

動的なデータ取得

providerFunction を使って外部APIからリアルタイムにデータを取得できます。

FEATURE 03

再利用可能

一度定義したリソースは複数のステップで再利用できます。

マニフェストでのカスタムリソース定義

workflowResourceDefinitions でカスタムリソースを定義します。

"flows": { "workflowResourceDefinitions": [ { "id": "customer", "displayName": "顧客", "description": "CRMシステムの顧客情報", "fields": [ { "id": "customerId", "displayName": "顧客ID", "dataType": { "basicType": "STRING" } }, { "id": "customerName", "displayName": "顧客名", "dataType": { "basicType": "STRING" } }, { "id": "email", "displayName": "メールアドレス", "dataType": { "basicType": "EMAIL_ADDRESS" } }, { "id": "companyName", "displayName": "会社名", "dataType": { "basicType": "STRING" } } ], "providerFunction": "getCustomerList" } ], "workflowElements": [...] }

リソースの使用方法

定義したリソースは、入力変数や出力変数のdataTypeで "resourceType": "customer" のように参照できます。

providerFunction の実装

providerFunction は、カスタムリソースのデータを外部から取得する関数です。 ユーザーが構成カードでリソースを選択する際に呼び出されます。

/** * 顧客リストを取得する providerFunction */ function getCustomerList(event) { try { // 外部CRM APIから顧客一覧を取得 const customers = fetchCustomersFromCRM(); // リソースアイテムを構築 const resourceItems = customers.map(customer => { return AddOnsResponseService.newResourceItem() .setId(customer.id) .setDisplayName(customer.name) .setFieldValue("customerId", customer.id) .setFieldValue("customerName", customer.name) .setFieldValue("email", customer.email) .setFieldValue("companyName", customer.company); }); // レスポンスを返す return AddOnsResponseService.newResourceItemsResponseBuilder() .setResourceItems(resourceItems) .build(); } catch (error) { console.error("顧客取得エラー:", error); throw new Error("顧客情報の取得に失敗しました"); } }

UrlFetchApp による外部API連携

UrlFetchApp は、Apps Script で外部APIにHTTPリクエストを送信するためのサービスです。

GET リクエスト

function fetchCustomersFromCRM() { const apiKey = PropertiesService.getScriptProperties() .getProperty('CRM_API_KEY'); const options = { method: 'GET', headers: { 'Authorization': 'Bearer ' + apiKey, 'Content-Type': 'application/json' }, muteHttpExceptions: true }; const response = UrlFetchApp.fetch( 'https://api.crm-system.com/v1/customers', options ); if (response.getResponseCode() !== 200) { throw new Error('API Error: ' + response.getContentText()); } return JSON.parse(response.getContentText()).customers; }

POST リクエスト

function createCustomerInCRM(customerData) { const apiKey = PropertiesService.getScriptProperties() .getProperty('CRM_API_KEY'); const options = { method: 'POST', headers: { 'Authorization': 'Bearer ' + apiKey, 'Content-Type': 'application/json' }, payload: JSON.stringify(customerData), muteHttpExceptions: true }; const response = UrlFetchApp.fetch( 'https://api.crm-system.com/v1/customers', options ); if (response.getResponseCode() !== 201) { throw new Error('顧客作成に失敗: ' + response.getContentText()); } return JSON.parse(response.getContentText()); }

エラーハンドリングの重要性

muteHttpExceptions: true を設定すると、HTTPエラー時も例外が発生せず、 レスポンスコードを確認して適切にエラー処理できます。

PropertiesService による認証情報管理

APIキーなどの認証情報は、コードに直接書かず PropertiesService で安全に管理します。

// 認証情報の保存（初期設定時に一度だけ実行） function setApiCredentials() { const scriptProperties = PropertiesService.getScriptProperties(); scriptProperties.setProperties({ 'CRM_API_KEY': 'your-api-key-here', 'CRM_API_SECRET': 'your-api-secret-here' }); } // 認証情報の取得 function getApiKey() { return PropertiesService.getScriptProperties() .getProperty('CRM_API_KEY'); } // 認証情報の削除 function clearApiCredentials() { const scriptProperties = PropertiesService.getScriptProperties(); scriptProperties.deleteAllProperties(); }
1

Script Properties

プロジェクト全体で共有。デプロイ時も保持される。APIキーの保存に最適。

2

User Properties

ユーザーごとに分離。ユーザー固有の設定や認証トークンに使用。

3

Document Properties

ドキュメントごとに保存。スプレッドシート連携時などに使用。

CRM連携ステップの完全実装

学んだ内容を組み合わせて、CRM連携カスタムステップを実装しましょう。

/** * 構成カードを返す */ function onConfigCRMUpdate(event) { // 顧客選択ドロップダウン（カスタムリソースを使用） const customerSelect = CardService.newSelectionInput() .setFieldName("customer") .setTitle("顧客を選択") .setType(CardService.SelectionInputType.DROPDOWN) .setExternalDataSource( CardService.newWorkflowDataSource() .setResourceType("customer") ); // ステータス選択 const statusSelect = CardService.newSelectionInput() .setFieldName("status") .setTitle("ステータス") .setType(CardService.SelectionInputType.DROPDOWN) .addItem("商談中", "negotiating", false) .addItem("提案中", "proposed", false) .addItem("成約", "closed_won", false) .addItem("失注", "closed_lost", false); // メモ入力 const noteInput = CardService.newTextInput() .setFieldName("note") .setTitle("メモ") .setMultiline(true); const section = CardService.newCardSection() .setHeader("CRM更新") .addWidget(customerSelect) .addWidget(statusSelect) .addWidget(noteInput); return CardService.newCardBuilder() .addSection(section) .build(); } /** * ステップ実行 */ function onExecuteCRMUpdate(event) { const inputs = event.workflow.actionInvocation.inputs; // 入力値を取得 const customerId = inputs["customer"].resourceValues[0] .fieldValues["customerId"].stringValue; const status = inputs["status"].stringValues[0]; const note = inputs["note"]?.stringValues[0] || ""; // CRM APIを呼び出して更新 const result = updateCustomerStatus(customerId, status, note); // 出力変数を返す const variableDataMap = { "updateResult": AddOnsResponseService .newVariableData() .addBoolValue(result.success), "updatedAt": AddOnsResponseService .newVariableData() .addTimestampValue(Date.now()) }; return outputVariables(variableDataMap); }
