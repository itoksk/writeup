# セットアップガイド

## 必要ツール
- ChatGPT (GPT-4o) または Claude 3.5 などHTML生成が得意なモデル
- コードエディタ（VS Code推奨）
- ブラウザのデベロッパーツール

## 推奨ファイル構成
- `design/` フォルダ：AIが生成したHTML・CSSを保存
- `assets/` フォルダ：画像やアイコンを格納
- `notes/` フォルダ：プロンプトとレビュー結果の記録

## 動画のデモを再現する手順
1. 動画1で整えた要件シートを開き、プロンプトの冒頭に貼り付ける。
2. 5要素テンプレート（テーマ/トーン/構造/アクセント/アクセシビリティ）を箇条書きする。
3. ChatGPTにプロンプトを投げ、出力されたHTMLをVS Codeに貼り付けて整形する。
4. Live Server拡張機能などでブラウザ表示を確認し、配色と余白を微調整する。
5. ナビゲーションやアニメーションが動いているか、ブラウザの開発者ツールでチェック。

## よくある調整ポイント
- フォントが想定どおりに反映されない → `<link rel="preconnect">` と `<link href="https://fonts.googleapis.com/...">` の記述を確認。
- コードブロックの改行が崩れる → `.code-block { white-space: pre-wrap; }` をCSSに追加。
- アクセントカラーが弱い → CSS変数`--electric`の値を明るめに調整し、hover時にコントラストを上げる。

## 品質チェックリスト
- [ ] `meta viewport`が指定されている
- [ ] ダークモード用の`.dark-mode`または`data-theme="dark"`が定義されている
- [ ] セクションごとのコンテンツ量が均等になっている
- [ ] 必要に応じてSVGアイコンまたはGoogle Iconsの呼び出しが記載されている

## トラブルシューティング
- CSSが冗長な場合 → AIに「ユーティリティクラスを整理して」と追加依頼。
- セクションのスクロールが機能しない → `IntersectionObserver`のオブザーバー設定と`data-target`属性を確認。
- ダークモードで文字が読みにくい → 背景色と文字色のコントラスト比を4.5:1以上に調整。
