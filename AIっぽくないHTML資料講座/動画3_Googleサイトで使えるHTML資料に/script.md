# Googleサイトで“使える”HTML資料を埋め込む方法

## 講師用台本（全10分）

### 【0:00-1:00】導入
- これまでに作成したHTMLを社内に展開する最終ステップだと説明し、「閲覧できない・崩れる」をゼロにする回と宣言。
- 今日のゴールは「Googleサイトで安全に埋め込み、公開権限まで整えた状態にすること」と伝える。
- Before/Afterカードで「余白崩れ・アクセス遅延が続くBefore」と「調整済みiframeと共有フローで即展開できるAfter」を比較して期待を高める。

### 【1:00-4:00】デモ・実演
- 画面共有でGoogleサイトを開き、新規ページを作成する操作をそのまま見せる。
- テンプレート選択からブランドカラーの設定までを操作しながら、「元のHTMLの世界観に揃える」ポイントを解説。
- 埋め込みボタンをクリックし、ホスト済みHTMLのURLを入力する様子を実演。
- サイズ調整で幅を100%、高さを90vhに設定し、モバイルプレビューで余白を確認。
- Before/AfterカードのBeforeで触れた「二重スクロール」の再現→Afterで改善したビューを順に見せ、効果を体感してもらう。

### 【4:00-9:00】構築・設定
- 埋め込みパターンを2種類解説：URL埋め込みと`<iframe>`コード埋め込み。
  - URL埋め込みのメリット：更新が一箇所で済む、JavaScript制限を回避しやすい。
  - コード埋め込みの注意：Googleサイトでブロックされやすい要素があるため、静的HTMLに整える必要がある。
- サンプルコードを貼り付け、`style="border:0; width:100%; height:90vh; overflow:hidden;"`の意味を順番に解説。
- 権限設定ステップを実演：右上の「共有」から「ドメイン内のユーザー」に限定し、特定グループへアクセス権を付与。
- プレビューでPC・タブレット・スマホ表示を順番に確認し、スクロールバーが二重にならないかチェック。
- ダークモード切り替えが資料側で機能するか、Googleサイト上でテストする流れも紹介。
- Before/AfterカードのAfter側を参照し、「公開前チェックリストに沿えばモバイル確認や通知フローも漏れなく整う」と言語化。

### 【9:00-10:00】まとめ・次回予告
- チェックポイントを3つで振り返り：①URL/コード埋め込みの使い分け ②権限とデバイス検証 ③公開URLの共有方法。
- Before/Afterカードを再掲し、「Afterの状態＝余白崩れゼロ・共有フロー整備」を目指して復習を促す。
- セリフ：「これで“AIっぽくない”HTMLが社内ポータルに並びます。今後は更新履歴を残しながら運用を回していきましょう。」
- アクション：公開リンクをSlackの#ai_shareチャンネルに投げ、第三者レビューを依頼するよう促す。
