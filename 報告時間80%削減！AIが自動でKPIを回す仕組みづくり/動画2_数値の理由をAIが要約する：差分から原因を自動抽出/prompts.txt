# プロンプト集：第2回 数値の理由をAIが要約する：差分から原因を自動抽出

**対象**: 初心者〜中級者
**使用ツール**: ChatGPT（無料版OK）、GPTs（応用編・有料）
**作業方法**: コピー&ペースト、プログラミング不要

---

## プロンプト1：基本的なKPI変動理由分析（最も使う）

スプレッドシートからKPIデータ、営業日報、イベント情報をコピーして、ChatGPTに以下のプロンプトと一緒に貼り付けます。

```
以下のKPIデータと営業日報、イベント情報から、今週のKPI変動理由を分析して要約してください。

【KPIデータ】
売上：前週148万円 → 今週165万円（+11.5%）
成約件数：前週26件 → 今週30件（+15.4%）
問い合わせ数：前週118件 → 今週135件（+14.4%）

【営業日報（今週分）】
2024-02-10	B社プロジェクト	4,200,000	成約	新規大口案件
2024-02-11	C社継続案件	1,500,000	成約	既存顧客リピート
2024-02-12	D社提案	2,000,000	商談中	価格交渉中

【イベント情報（今週分）】
2024-02-08	ウェビナー開催	50	25
2024-02-10	展示会出展	120	18

【出力形式】
以下の形式で要約してください：

✅ プラス要因（売上増加の理由）
- 具体的な案件名、金額、日付を含めて記載
- イベントやキャンペーンの効果も含める

⚠️ マイナス要因・注意すべき点
- リスクや懸念事項があれば記載
- 前週比がマイナスの項目があれば理由を分析

💡 次週への示唆
- 改善提案やアクション案を簡潔に
- データに基づいた実行可能な提案
```

**使い方**：
1. Google Sheetsで営業日報とイベント情報の今週分をコピー
2. ChatGPTを開く
3. 上記プロンプトの【営業日報】【イベント情報】部分にコピーしたデータを貼り付け
4. KPIデータ部分も実際の数値に更新
5. 送信
6. ChatGPTの回答をコピーして、スプレッドシートの『変動理由』列に貼り付け

```
【営業日報（今週分）】
（コピーしたデータを貼り付け）

【イベント情報（今週分）】
（コピーしたデータを貼り付け）

【出力形式】
1. 最も大きな増加要因（案件名と金額）

簡潔に、各項目1行でまとめてください。
```

## プロンプト6：異常値検知とアラート

異常値や注意すべき変動を検知したい場合：

```
以下のKPIデータから、異常値や注意すべき変動を検知してください。

【KPIデータ（過去4週分）】
2024-01-22	売上135万円	成約22件	問い合わせ105件
2024-01-29	売上142万円	成約24件	問い合わせ110件
2024-02-05	売上148万円	成約26件	問い合わせ118件
2024-02-12	売上165万円	成約30件	問い合わせ135件

【営業日報（今週分）】
（コピーしたデータを貼り付け）

【検知条件】
- 前週比±20%以上の変動
- 3週連続で同じ方向に変動している指標
- 急激な変化があった指標

【出力形式】
🚨 アラート：[異常値の内容と理由]
📈 トレンド：[継続的な傾向]
✅ 正常範囲：[問題なし]

異常値があれば、その理由を営業日報とイベント情報から特定してください。
```

## GPTs Instructions（応用編・オプション）

### GPTsを作成する場合のInstructions（ChatGPT Plus必要）

GPT作成画面の「Instructions」欄に貼り付けるテキスト：

```
あなたはKPI分析の専門家です。
ユーザーがKPIデータと営業日報、イベント情報を貼り付けたら、以下を実行してください：

【役割】
1. KPIの変動理由を分析
2. プラス要因、マイナス要因、次週への示唆の3つに分けて要約
3. 具体的な案件名、金額、日付を含めて記載

【出力形式】
✅ プラス要因（売上増加の理由）
- 具体的な案件名、金額、日付を含めて記載
- イベントやキャンペーンの効果も含める
- 数値を含めて具体的に（例：「B社プロジェクト（420万円）が成約（2/10）」）

⚠️ マイナス要因・注意すべき点
- リスクや懸念事項があれば記載
- 前週比がマイナスの項目があれば理由を分析

💡 次週への示唆
- 改善提案やアクション案を簡潔に
- データに基づいた実行可能な提案

【注意事項】
- 推測ではなく、提供されたデータに基づいた分析のみ行う
- 固有名詞（案件名、イベント名）と数値（金額、日付）を必ず含める
- 抽象的な表現は避け、具体的に記述する
- 簡潔に、3〜5項目に絞る
```

**使い方**：
1. ChatGPTで「Explore GPTs」→「Create a GPT」
2. 「Configure」タブを開く
3. 「Instructions」欄に上記をコピー&ペースト
4. 「Name」欄に「KPI変動理由分析アシスタント」などを入力
5. 保存

**GPTsの利点**：
- 毎回プロンプトを書く必要がなくなる
- データをコピー&ペーストするだけで自動で分析してくれる
- 分析の視点が統一される

**GPTsの注意点**：
- ChatGPT Plus（月20ドル）が必要
- Actions設定は不要（シンプルにInstructionsだけでOK）
- データは毎回コピー&ペーストで渡す

## よくある質問（FAQ）

### Q1: ChatGPT無料版でも使えますか？
**A**: はい、基本的な分析なら無料版（GPT-3.5）で十分です。より詳しい分析が欲しい場合は有料版（GPT-4）を使うと精度が上がります。

### Q2: 営業日報のフォーマットは決まっていますか？
**A**: いいえ、自由です。ただし、日付、案件名、金額、ステータス、備考の5列があると分析精度が上がります。

### Q3: データをコピーする際、どの範囲を選択すればいいですか？
**A**: 今週分（または分析対象期間）のデータだけを選択してコピーしてください。見出し行は含めてもOKです。

### Q4: ChatGPTの分析結果が間違っている気がします。
**A**: 『計算過程も教えて』と追加で質問すれば、ChatGPTが根拠を説明してくれます。データが不足している場合は、補足情報を追加してください。

### Q5: GPTsを作るメリットは？
**A**: 毎回同じプロンプトを書く必要がなくなります。ただし、ChatGPT Plus（有料）が必要なので、無料版でも十分使えます。

### Q6: 複数週分をまとめて分析できますか？
**A**: はい、『過去4週間の変動理由を全て分析して』と指示すれば、複数週分まとめて分析できます。

## トラブルシューティング

### 問題1: ChatGPTが分析してくれない

**症状**: プロンプトを送っても、分析せずに説明だけ返ってくる

**解決策**: プロンプトの最後に「必ずこの形式で出力してください」と追加
```
【出力形式】
以下の形式で必ず出力してください：
✅ プラス要因
⚠️ マイナス要因
💡 次週への示唆
```

### 問題2: 分析が浅い

**症状**: 「売上が増えました」のような抽象的な分析しか返ってこない

**解決策**: データの粒度を上げる
- 営業日報の『備考』欄に背景情報を追加
- 『具体的な案件名、金額、日付を含めて』と明示
- 『3〜5項目に絞って』と指示

### 問題3: 不要な情報が含まれる

**症状**: 分析結果に無関係な情報が含まれる

**解決策**: 出力形式を明確に指定
```
【出力形式】
以下の3つだけを出力してください。他の情報は不要です：
✅ プラス要因
⚠️ マイナス要因
💡 次週への示唆
```

## 活用のコツ

### コツ1: 営業日報の備考欄を充実させる
一行でいいので、案件の背景情報を書いておきます：
- 「新規大口案件」「既存顧客リピート」「キャンペーン経由」など
- ChatGPTの分析精度が大幅に上がります

### コツ2: プロンプトをテンプレート化
一度プロンプトを作ったら、テキストファイルに保存：
- 毎週、【データ】部分だけ差し替えればOK
- 5分で分析完了

### コツ3: 分析の深さを調整
状況に応じて指示を変える：
- 忙しい時：『簡潔に3つに絞って』
- 詳しく知りたい時：『詳しく分析して』
- 報告書用：『詳細版プロンプト』を使用

### コツ4: GPTsの使い分け
- 基本編（無料版）：毎回プロンプトを入力
- 応用編（GPTs）：データをコピペするだけ
- まずは基本編から試して、必要性を感じたらGPTsを検討

### コツ5: 他のデータも追加可能
営業日報とイベント情報以外にも：
- 競合情報
- 顧客満足度
- 市場トレンド
など、関連データを追加すると多角的な分析ができます

## 応用編：他の用途にも使える

### 応用1: 月次分析
「前週比」を「前月比」に変更するだけで、月次データにも使えます：
```
【KPIデータ】
売上：前月500万円 → 今月650万円（+30%）
```

### 応用2: 部門別分析
複数部門のデータを一度に分析：
```
【営業部KPIデータ】
売上：前週100万円 → 今週120万円（+20%）

【マーケティング部KPIデータ】
リード数：前週50件 → 今週65件（+30%）

各部門の変動理由を分析してください。
```

### 応用3: プロジェクト別分析
プロジェクトごとの進捗を分析：
```
【プロジェクトA】
進捗率：前週60% → 今週75%（+15%）

【営業日報（プロジェクトA関連）】
（コピーしたデータを貼り付け）

プロジェクトAの進捗が上がった理由を分析してください。
```
