## 🔰 初心者向け：まずはこれだけ！（最小限プロンプト）

**使い方（3ステップ）:**
1. 下のプロンプトをすべてコピー
2. ChatGPTに貼り付け
3. 【ここを編集】の部分だけ、あなたのプロジェクトに書き換えて送信

```

【ここを編集：あなたのプロジェクト情報を書く】
- プロジェクト名: 新ECサイト構築
- 何をする: ECサイトをリニューアルして売上を上げる
- 期間: 6ヶ月
- 予算: 5,000万円
- メンバー: PM 1名、エンジニア 5名、デザイナー 2名

カテゴリ別に分けて、表で教えてください。
```

**💡 ポイント:**
- 「プロジェクト名」「何をする」「期間」「予算」「メンバー」だけ書けばOK
- 詳しく書けば書くほど、AIの回答が具体的になります
- まずは簡単な情報だけでも試してみてください

## プロンプト1: 基本的なリスク洗い出し（標準版）

**使い方:**
1. 下のプロンプトをすべてコピー
2. ChatGPTに貼り付け
3. 【プロジェクト概要】の部分を、あなたのプロジェクトに書き換え
4. 送信ボタンを押す

```
あなたはプロジェクトマネジメントの専門家です。
以下のプロジェクトにおいて、考えられるリスクをカテゴリ別に網羅的にリストアップしてください。

【プロジェクト概要】
【ここを編集：あなたのプロジェクト情報を書く】
- プロジェクト名: 新ECサイト構築
- 目的: 既存のECサイトをリニューアルし、売上を30%向上させる
- スコープ: フロントエンド刷新、決済システム統合、在庫管理システム連携
- 期間: 6ヶ月（2025年6月〜12月）
- 予算: 5,000万円
- チーム構成: PM 1名、エンジニア 5名、デザイナー 2名、外部ベンダー 1社
- 主要ステークホルダー: 営業部、カスタマーサポート部、情報システム部

【出力形式】
以下のカテゴリごとに、リスクを箇条書きで出力してください。
1. 技術的リスク
2. スケジュールリスク
3. コストリスク
4. 人的リスク
5. 外部依存リスク
6. セキュリティリスク
7. 法務・コンプライアンスリスク
8. ステークホルダーリスク
9. 運用・保守リスク
10. その他のリスク

各リスクは「リスク名: 簡潔な説明（1行）」の形式で記載してください。
```

### 想定出力例

ChatGPTはこんな風に答えてくれます：

**1. 技術的リスク**
- フロントエンド刷新の遅延: 新技術の学習コストで開発が遅れる
- 決済システム統合の失敗: 既存システムとの互換性問題
- 在庫管理システム連携の不具合: データ形式の不一致

**2. スケジュールリスク**
- 要件定義の遅延: ステークホルダーの合意形成に時間がかかる
- テスト期間の不足: 十分なテストができない

## プロンプト3: スプレッドシート形式で出力（実務で便利）

**こんな時に使います:**
- ExcelやGoogleスプレッドシートに貼り付けたい
- リスク管理表を作りたい

```
あなたはプロジェクトマネジメントの専門家です。
以下のプロジェクトにおいて、考えられるリスクをスプレッドシート形式で出力してください。

【プロジェクト概要】
【ここを編集：あなたのプロジェクト情報を書く】
- プロジェクト名: [プロジェクト名を記入]
- 目的: [目的を記入]
- スコープ: [スコープを記入]
- 期間: [期間を記入]
- 予算: [予算を記入]
- チーム構成: [チーム構成を記入]

【出力形式】
以下の列でタブ区切り（TSV形式）で出力してください。そのままスプレッドシートに貼り付けられる形式でお願いします。

※発生確率・影響度・重大度は空欄で出力してください（後で記入するため）
```

2. Excelまたは Googleスプレッドシートを開く

4. 列がきれいに分かれた表が完成

## ❓ よくある質問（FAQ）

### Q1: リスク台帳って何ですか？
A: プロジェクトで起こりそうな問題（リスク）を一覧にしたものです。「何が起こるか」「どう対処するか」を事前に考えておくための表です。

### Q2: プロジェクト情報は、どこまで詳しく書けばいいですか？
A: 最初は「プロジェクト名」「何をする」「期間」「予算」「メンバー」だけでOKです。詳しく書けば書くほど、AIの回答が具体的になります。

### Q3: リスクが100個以上出てきました。多すぎませんか？
A: プロンプト2を使って「重要度の高いリスクを20項目程度に絞って」と指示してください。

### Q4: AIが出したリスクを、そのまま使っていいですか？
A: AIの提案は参考にして、自分のプロジェクトに合わせて調整してください。AIはあなたのプロジェクトの細かい事情を知らないので、100%信じないでください。

### Q5: スプレッドシート形式で出力したのに、列がバラバラです
A: ChatGPTの出力をコピーする時、表全体を選択してコピーしてください。それでもダメな場合は、プロンプト3を使って「タブ区切り（TSV形式）」で出力してもらってください。


## 🚨 注意点

### 注意点1：AIの提案を100%信じない
AIはあなたのプロジェクトの細かい事情を知りません。AIの提案を参考にして、自分で判断してください。

例：
- AIが「セキュリティリスク」を挙げても、あなたの会社にセキュリティ専門チームがいるなら、リスクは低い
- AIが「スケジュールリスク」を挙げても、あなたのチームは過去に同じプロジェクトを成功させているなら、リスクは低い

### 注意点2：プロジェクト情報は正確に書く
プロジェクト情報が曖昧だと、AIの回答も曖昧になります。できるだけ具体的に書いてください。

悪い例：
- プロジェクト名: システム開発
- 目的: システムを作る

良い例：
- プロジェクト名: 新ECサイト構築
- 目的: 既存のECサイトをリニューアルし、売上を30%向上させる

## プロンプト6: 特定のリスクを深掘りする

**こんな時に使います:**
- 1つのリスクについて、もっと詳しく知りたい
- 対策案を知りたい

```
以下のリスクについて、さらに詳しく掘り下げてください。

【リスク】
【ここを編集：知りたいリスク名を書く】
例：決済システム統合の失敗

【深掘りポイント】
1. このリスクが発生する具体的な原因・シナリオ
2. 発生確率（高/中/低）とその根拠
3. 発生した場合の影響範囲（スケジュール、コスト、品質、スコープへの影響）
4. 類似プロジェクトでの発生事例
5. 検討すべき予防策の候補

上記5点について、詳細に説明してください。
```

```
【リスクリスト】
【ここを編集：ChatGPTが出力したリスクリストをここに貼り付け】

【出力形式】
重複を削除し、カテゴリごとに整理して出力してください。
統合した項目については、元のリスク名を併記してください。
```
