# ChatGPTで始めるリスクマネジメント実践：抜け漏れゼロ・"先読み対応"でプロジェクトを守る

## 1. 想定する現場課題

プロジェクト管理におけるリスクマネジメントは、多くの現場で以下のような課題に直面しています：

### 主な課題
- **プロジェクトリスク台帳の更新が属人化**し、最新化が追いつかない
  - 特定の担当者に依存し、その人が不在だと更新が止まる
  - リスク洗い出しの視点が偏り、重要な視点が抜け落ちる

- **「何がリスクになるか」が抜けてしまい**、後手対応になりやすい
  - 経験豊富なメンバーの知見が共有されず、若手は網羅的な検討ができない
  - 想定漏れによる手戻りが頻発し、スケジュールとコストに影響

- **月次報告が"報告のための作業"**になっており、対策が形骸化
  - リスクレビューに毎回2時間以上かかるが、実質的な改善につながらない
  - 報告書作成が目的化し、予防的な対策が後回しに

- **Excel台帳の管理に時間がかかり**、分析・可視化まで手が回らない
  - リスク洗い出しに毎回3時間、フォーマット調整に1時間
  - データの集計・分析を手作業で行い、全体像の把握が困難

- **リスク検討会議で「想定漏れ」「手戻り」**が頻発している
  - 会議後に「あれも検討すべきだった」という指摘が出る
  - 対策の具体性が不足し、実行段階で詰まる

## 2. 理想の状態（Before / After）

| 状態 | Before（導入前） | After（導入後） |
|------|------------------|-----------------|
| **作業時間** | リスク洗い出しに3時間／月次更新に2時間 | 生成と更新が10分以内に完了 |
| **精度** | 抜け漏れ多数・属人化 | AIによる自動生成で網羅性95%以上 |
| **感情** | 「手間ばかり多く成果が見えない」 | 「予兆を先に見つけ、安心して進行できる」 |
| **組織効果** | 対応が後手 | 予防型のリスク文化が定着 |
| **月次レビュー** | 毎月レポート作成に1〜2時間 | ChatGPTが5分で自動生成 |
| **判断品質** | 感覚で影響度を判断 | ChatGPTが数値シナリオを生成し、影響比較を自動化 |

### 効率化目標（現実的な数値）
- **リスク洗い出し時間**: 80%削減（3時間 → 30分）
- **月次レビュー時間**: 75%削減（2時間 → 30分）
- **重大リスク検出率**: 95%に向上（属人的な視点からAIによる網羅的視点へ）
- **対策の具体性**: 実行可能レベルまで自動提案（予兆・防止策・対応手順の3連鎖）

## 7. 各動画の詳細構成

詳細な台本・セリフ・プロンプトは各動画フォルダ内の `script.md` を参照してください。

### 動画1: ChatGPTで一括生成！抜け漏れゼロのリスク台帳づくり入門
- プロジェクト概要をChatGPTに入力 → カテゴリ別リスク一覧を自動生成
- 50項目の網羅的なリスクを10分で作成
- スプレッドシートへの貼り付けと整形

### 動画2: 発生前に防ぐ：予兆→防止策→対応手順をAIで自動連鎖
- 各リスクに対する「発生予兆」「予防策」「発生時の対応手順」を3連鎖で自動生成
- リスクを"点"でなく"線"で管理する思考をAIがサポート
- プロジェクト体制に合わせた調整方法

### 動画3: もしもに備える：ChatGPTでスケジュール/コスト影響をシナリオ比較
- リスク発生時の工期遅延・追加コストを3シナリオで比較
- 「もしも分析」を数値化し、意思決定の質を向上
- 現場データでの補正方法

### 動画4: たった5分で！重大度スコア×是正計画を自動出力するリスクレビュー術
- 発生確率×影響度で重大度を算出し、上位リスクの是正案を提示
- 月次レビューの質を上げ、判断スピードを高める
- 報告書フォーマットへの自動貼り付け
