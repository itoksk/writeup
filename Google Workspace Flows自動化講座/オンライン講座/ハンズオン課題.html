ハンズオン練習問題

学んだ知識を実践で定着させよう

練習問題の進め方

この練習問題の目的

今日学んだつまずきやすいポイントを意識しながら、実際にエージェントを作成・動作させることで、知識を定着させます。

  • レベル1（基礎）: 講座で一緒に作ったものを復習
  • レベル2（応用）: 少し異なるシナリオに挑戦
  • レベル3（発展）: 複数のステップを組み合わせた自動化

レベル1：基礎練習

1
基礎

練習1-1：お問い合わせフォーム通知エージェント

シナリオ

お客様からのお問い合わせフォームに回答が来たら、即座にサポートチームのChatスペースに通知するエージェントを作成してください。

実装要件

  • テスト用のGoogle Formを作成（質問は3つ以上）
  • 通知先のChatスペースを準備（または新規作成）
  • 「New form response」をトリガーに設定
  • Chatスペースへの通知を設定（候補から選択）
  • 通知に以下の変数を含める：
    • 回答者のメールアドレス
    • タイムスタンプ
    • 各質問への回答内容
  • Test runで動作確認
  • エージェントを有効化（Turn on）
  • 実際にフォームから回答を送信して、本番動作を確認
成功基準
  • フォーム回答後、1〜5分以内にChatに通知が届く
  • 通知に回答者情報と回答内容が含まれている
  • エージェントの状態が「On」になっている
つまずきポイントの復習
  • ポイント①: Chatスペースは候補から選択（Webhook URL不要）
  • ポイント②: フォームも候補から選択（手動入力NG）
  • ポイント③: 変数は「Insert variable」から挿入
  • ポイント④: Test run後、必ず「Turn on」する
2
基礎

練習1-2：Activityログで動作を確認

シナリオ

練習1-1で作成したエージェントのActivityログを確認し、正常に動作していることを確かめる練習をします。

実装手順

  • 練習1-1のエージェントで、フォーム回答を1回送信
  • 「My agents」画面で作成したエージェントを選択
  • 「Activity」タブを開く
  • 最新の実行ログを確認
  • ログの各ステップ（トリガー、通知送信など）が成功しているか確認
  • 通知に含まれた変数の値を確認
成功基準
  • Activityログにアクセスできた
  • 各ステップの実行結果を確認できた
  • 変数に正しい値が入っていることを確認できた
ヒント

Activityログは、エージェントが実際に動いた履歴を確認できる重要な機能です。エラーが出た時だけでなく、正常に動いている時も確認する習慣をつけましょう。

レベル2：応用練習

3
応用

練習2-1：Driveファイルアップロード通知

シナリオ

共有Driveフォルダにファイルがアップロードされたら、チームのChatスペースに通知するエージェントを作成してください。

実装要件

  • 監視用のDriveフォルダを作成または選択
  • 「New file in folder」をトリガーに設定
  • Driveフォルダを候補から選択（URL貼り付けNG）
  • Chatスペースへの通知を設定
  • 通知に以下の変数を含める：
    • ファイル名
    • アップロード者
    • ファイルへの直接リンク（URL）
  • Test runで動作確認
  • 有効化後、実際にファイルをアップロードして動作確認
成功基準
  • ファイルアップロード後、通知が届く
  • 通知からファイルに直接アクセスできる
  • ファイル名とアップロード者が表示される
ヒント

Driveフォルダの選択も、Chatスペースと同じく候補から選択します。フォルダ名の一部を入力して、候補を表示させましょう。

4
応用

練習2-2：複数の質問への条件分岐（オプション）

シナリオ

フォーム回答の内容に応じて、異なるChatスペースに通知を送るエージェントを作成してください。（この練習問題は、条件分岐機能がある場合のみ）

実装要件

  • 「緊急度」を選択する質問をフォームに追加（例：「緊急」「通常」「低」）
  • 「緊急」が選択された場合は、特定のChatスペースに通知
  • それ以外の場合は、別のChatスペースに通知
ヒント

Workspace Flowsの条件分岐機能（If/Else）を使います。「Add step」から「Condition」を選択してみましょう。

レベル3：発展練習

5
発展

練習3-1：Gemini要約付きファイル通知

シナリオ

Driveにアップロードされたドキュメントファイルを、GeminiでAI要約し、その要約をChatに通知するエージェントを作成してください。

実装要件

  • 「New file in folder」をトリガーに設定
  • 「Summarize with Gemini」ステップを追加
  • Geminiに要約させるプロンプトを設定（例：「このドキュメントを3行で要約してください」）
  • Chatスペースへの通知を設定
  • 通知に以下を含める：
    • ファイル名
    • Geminiによる要約結果（変数を使用）
    • ファイルへの直接リンク
  • 有効化後、テストファイルをアップロードして動作確認
成功基準
  • ファイルアップロード後、AI要約付きの通知が届く
  • 要約が適切に生成されている
  • 通知からファイルに直接アクセスできる
ヒント
  • 「Add step」から「Summarize with Gemini」を選択
  • 要約結果も変数として、Chat通知に埋め込めます
  • Gemini for Workspaceが有効化されている必要があります
6
発展

練習3-2：実務で使える自動化を設計

シナリオ

あなたの実務で実際に使える自動化エージェントを1つ設計し、実装してください。

実装要件

  • 現在の業務で手作業している作業を1つ選ぶ
  • その作業を自動化するエージェントを設計
  • トリガー、通知内容、変数の使い方を計画
  • 実装し、動作確認
  • チームメンバーに共有（可能であれば）
成功基準
  • 実務で使える自動化が実装できた
  • 手作業の時間を削減できた
  • チームで共有できた（オプション）
アイデアの例
  • 会議議事録がDriveにアップロードされたら、要約して関係者に通知
  • 週次レポートフォームの回答を自動集計・通知
  • プロジェクトファイルのアップロードをチーム全体に共有
  • 定期的なタスクリマインダーを自動送信

完了確認

完了チェックリスト

すべての練習問題が完了したら、以下を確認してください：

  • 各エージェントが「On」の状態になっている
  • Test runだけでなく、実際の動作確認も完了している
  • Activityログでエラーがないことを確認
  • 通知内容が適切に整形されている
