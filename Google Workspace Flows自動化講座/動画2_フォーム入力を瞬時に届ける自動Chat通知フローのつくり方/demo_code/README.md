# デモ用資料・参考情報

## 📋 テスト用フォームの準備

動画撮影・実習用のサンプルフォームを作成してください。

### フォーム名
「お問い合わせフォーム」（サンプル）

### 質問項目（例）

#### 質問1：お名前 *
- 種類：記述式（短文）
- 必須：はい

#### 質問2：メールアドレス *
- 種類：記述式（短文）
- 必須：はい
- 検証：メールアドレス形式

#### 質問3：お問い合わせ内容 *
- 種類：記述式（長文）
- 必須：はい

#### 質問4：緊急度
- 種類：ラジオボタン（選択肢）
- 選択肢：高 / 中 / 低
- 必須：はい

#### 質問5：連絡希望時間帯
- 種類：チェックボックス
- 選択肢：午前 / 午後 / 夕方以降

## 🎬 デモ実演の流れ

### 1. テスト回答の準備
フォームに、以下のようなテスト回答を用意してください：

```
お名前: 山田太郎
メールアドレス: yamada@example.com
お問い合わせ内容: 貴社サービスについて詳しく知りたいです。
緊急度: 高
連絡希望時間帯: 午後
```

### 2. フロー作成のデモ手順
1. Flowsホーム画面を開く
2. 「新しいフォーム回答をチームに通知」テンプレートを選択
3. トリガー設定：「お問い合わせフォーム」を選択
4. Chat送信設定：「マーケティングチーム」スペースを選択
5. メッセージ内容を確認（必要に応じてカスタマイズ）
6. テスト実行
7. Chatに通知が届くことを確認
8. 「有効化」ボタンをクリック

### 3. 本番テスト
フローを有効化したら、実際にフォームに回答を送信して、
Chat通知が届くことを確認します。

## ⚠️ デモ時の注意点

### Chatスペース選択の部分は特に強調
- **手動入力ではなく、候補から選択する**ことを何度も強調してください
- 実際に手動入力して失敗する例も見せると、より理解しやすくなります

### 画面操作はゆっくり丁寧に
- クリックする場所が明確にわかるように
- マウスカーソルの動きもゆっくりと

### 成功体験を重視
- テストが成功したら、「できましたね！」と明るくコメント
- 受講者に「自分にもできそう」と思ってもらうことが重要

## 🔧 カスタマイズのアイデア

### アイデア1：緊急度別の通知先
条件分岐を使って、緊急度が「高」の場合は、
「緊急対応チーム」スペースに通知することもできます。
（第4回で条件分岐を学びます）

### アイデア2：スプレッドシートにも記録
Chat通知と同時に、スプレッドシートにも回答を記録することで、
後から集計・分析しやすくなります。

### アイデア3：自動返信メール
フォーム回答者に対して、自動で返信メールを送ることもできます。
「お問い合わせありがとうございました」という確認メールを自動送信できます。

## 💡 受講者からの想定Q&A

### Q1: 複数のフォームに対して、同じChatスペースに通知できますか？
A: はい、できます。各フォームに対して別々のフローを作成し、
同じChatスペースを選択してください。

### Q2: フォームの質問を後から変更したら、フローも更新が必要ですか？
A: 質問を追加・削除した場合、フローのメッセージ内容も更新する必要があります。
ただし、質問の文言を変更しただけであれば、フローの更新は不要です。

### Q3: 1つのフォームに対して、複数の通知先を設定できますか？
A: はい、複数のフローを作成することで、複数の通知先に送ることができます。

### Q4: 通知が届かない場合、どうすればいいですか？
A:
1. Chatスペースに参加しているか確認
2. フローが「有効」になっているか確認
3. テスト実行でエラーが出ていないか確認
