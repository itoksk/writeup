## 📊 Before/After 比較データ

### Before（現状の課題）

#### フォーム通知の手作業
- 頻度：1日10〜15件
- 1件あたりの作業時間：5分
- 合計作業時間：50〜75分/日
- 月間作業時間：約20〜30時間

#### ファイル確認・共有の手作業
- 頻度：週5件
- 1件あたりの作業時間：10分
- 週間作業時間：50分
- 月間作業時間：約3.5時間

#### 合計
- 月間作業時間：約25〜35時間

### After（Flows導入後）

#### 自動化による削減効果
- フォーム通知：0秒（完全自動化）
- ファイル要約・共有：0秒（完全自動化）
- 月間削減時間：約25〜35時間
- **削減率：90%**

#### 残る作業
- 自動化フローの初期設定：1〜2時間（初回のみ）
- フローのメンテナンス：月30分程度

## 🎯 Google Workspace Flows の主要機能一覧

### トリガー（きっかけ）
1. **フォーム送信時**
   - Googleフォームに回答が送信されたとき

2. **ファイルアップロード時**
   - 指定したDriveフォルダにファイルがアップロードされたとき

3. **Chat メンション時**
   - Chatで@メンションされたとき

4. **スケジュール**
   - 毎日、毎週など、指定した時刻に実行

5. **カレンダーイベント**
   - カレンダーに予定が追加されたとき

### アクション（実行すること）
1. **Chat通知**
   - 指定したChatスペースにメッセージを送信

2. **Gemini要約**
   - Geminiを使ってファイルやテキストを要約

3. **スプレッドシート記録**
   - データをスプレッドシートに追加・更新

4. **メール送信**
   - Gmailでメールを送信

5. **条件分岐**
   - 条件に応じて、異なるアクションを実行

6. **ループ処理**
   - 複数のデータに対して、同じアクションを繰り返し実行

## 🔧 アカウント要件の詳細

### 必須要件

| 項目 | 要件 | 備考 |
|------|------|------|
| アカウント種別 | Google Workspace（Business/Enterprise） | 個人アカウント、Educationアカウントは一部機能制限あり |
| Gemini for Workspace | 有効化 | Gemini連携機能を使用する場合は必須 |
| 管理者権限 | 組織の設定により異なる | 一部の機能は管理者承認が必要 |
| Beta機能 | Google Workspace ラボで有効化 | Flowsはベータ版機能のため |

### 確認方法

#### アカウント種別の確認
1. Google Workspace 管理コンソールにアクセス
2. 「アカウント」→「アカウント設定」で確認

#### Gemini for Workspace の確認
1. Google Workspace 管理コンソールにアクセス
2. 「アプリ」→「Google Workspace」→「Gemini」で確認

## 📝 講師への補足メモ

### デモ時の注意点

1. **画面共有の準備**
   - Flowsのホーム画面をあらかじめ開いておく
   - テンプレート一覧が見やすいようにブラウザのズームを調整
   - 不要なタブは閉じておく

2. **説明のポイント**
   - 「難しくない」というメッセージを強調
   - 「自分にもできそう」と思ってもらえるように
   - 次回以降の動画への期待感を高める

3. **時間配分**
   - 導入：1:30
   - Before/After：1:30
   - デモ：2:30
   - 構築手順：1:30
   - 設定実演：1:30
   - 注意点：1:00
   - まとめ：0:30
   - **合計：10:00**
