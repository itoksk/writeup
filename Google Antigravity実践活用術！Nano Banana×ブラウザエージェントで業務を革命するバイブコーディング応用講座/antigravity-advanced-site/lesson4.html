GitHubもNotionも思いのまま！MCP連携で開発ワークフローを革命する

外部ツールとの連携で生産性を最大化

このレッスンのゴール

MCP（Model Context Protocol）を使って、Google AntigravityからGitHub、Notion、Slackなどの外部サービスを操作できるようになります。Issue管理、ドキュメント作成、通知送信など、開発ワークフロー全体をチャットで完結させる方法を習得しましょう。

MCPとは？

Model Context Protocol

MCPは、AIモデルと外部ツールを接続するためのオープンプロトコルです。Anthropicが2024年11月に発表し、Claude、Gemini、GPTなど主要なAIが対応を進めています。AntigravityもMCPに標準対応しており、GitHub、Notion、Slackなどの「MCPサーバー」を追加することで機能を拡張できます。

従来の開発フロー
  • コードを書く→GitHubを開く→コミット
  • PRを作成→Slackで通知→Notionに記録
  • ツール間を行き来する手間
  • コンテキストスイッチでロスが発生
MCP連携後
  • 「GitHubにコミットして」で完了
  • 「PRを作成してSlackに通知」で自動実行
  • 全てAntigravityのチャットで完結
  • コーディングに集中できる

MCPサーバーのインストール

1

設定ファイルを開く

Antigravityの設定（Command + ,）から「Extensions」→「MCP Servers」を選択。または、直接 mcp_config.json ファイルを編集します。

2

MCPサーバーを追加

使用したいサービスのMCPサーバーを追加します。公式リポジトリから取得するか、コミュニティ製を使用。

3

認証情報を設定

各サービスのAPIトークンやアクセスキーを設定します。GitHub Personal Access Token、Notion API Key など。

// mcp_config.json の例 { "mcpServers": { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx" } }, "notion": { "command": "npx", "args": ["-y", "@notionhq/mcp-server"], "env": { "NOTION_API_KEY": "secret_xxxxxxxxxxxx" } }, "slack": { "command": "npx", "args": ["-y", "@anthropic/mcp-server-slack"], "env": { "SLACK_BOT_TOKEN": "xoxb-xxxxxxxxxxxx" } } } }

APIトークンの取得方法

各サービスの開発者設定からトークンを発行します。GitHubは Settings → Developer Settings → Personal Access Tokens から取得。Notionは Integrations ページから作成。

GitHub連携でできること

操作1

リポジトリ操作

リポジトリの作成、クローン、ブランチ作成などの基本操作。

操作2

コミット・プッシュ

変更をステージング、コミット、リモートにプッシュ。

Issue管理

Issueの作成、一覧取得、コメント追加、クローズ。

操作4

PR作成・レビュー

Pull Requestの作成、差分確認、マージ。

// GitHub操作の指示例 「このプロジェクトをGitHubにプッシュして」 「feature/login-form ブランチを作成して、現在の変更をコミット」 「main ブランチにPRを作成。タイトルは『ログインフォームの追加』」 「リポジトリのIssue一覧を表示して」 「Issue #3 に『修正完了しました』とコメントして」

実践：GitHub連携プロジェクト

新規プロジェクトをGitHubで管理

新しいWebアプリを作成し、GitHubリポジトリの作成からPRマージまでをAntigravityだけで完結させてみましょう。

1

プロジェクトとリポジトリを作成

新規プロジェクトを作成し、GitHubリポジトリも同時に作成します。

「新しいフォルダ my-todo-app を作成して、シンプルなToDoアプリを作って。 GitHubにもリポジトリを作成して、初期コミットをプッシュして」
2

機能追加ブランチを作成

新機能の開発はブランチを分けて行います。

「feature/dark-mode ブランチを作成して、ダークモード機能を追加して。 完了したらコミットして」
3

PRを作成

変更をプッシュしてPull Requestを作成します。

「変更をプッシュして、main ブランチに対してPRを作成。 タイトル: ダークモード機能の追加 説明: ユーザーがダークモードを切り替えられる機能を追加しました」
4

PRをマージ

問題なければマージします。

「PRをマージして、feature/dark-mode ブランチを削除して」

全てチャットで完結

ブラウザでGitHubを開くことなく、リポジトリ作成からPRマージまで全てAntigravityのPlaygroundで完了しました！

MCPサーバーは増え続けている

MCPはオープンプロトコルなので、コミュニティによって新しいサーバーが日々追加されています。公式リポジトリで最新のリストを確認できます。

カスタムMCPサーバーの作成

自分だけのツール連携

公式にMCPサーバーが存在しないサービスでも、自分でMCPサーバーを作成することで連携できます。Node.jsやPythonで簡単に作成可能。

// カスタムMCPサーバーの基本構造（Node.js） import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js"; import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js"; const server = new McpServer({ name: "my-custom-server", version: "1.0.0" }); // ツールを定義 server.tool("send_email", "メールを送信する", { to: { type: "string", description: "送信先" }, subject: { type: "string", description: "件名" }, body: { type: "string", description: "本文" } }, async (args) => { // メール送信ロジック return { success: true, message: "メール送信完了" }; }); const transport = new StdioServerTransport(); await server.connect(transport);

社内ツールとの連携も可能

社内の独自システムやAPIとも、MCPサーバーを作成することで連携できます。「勤怠システムに出勤記録を登録して」のような操作も実現可能。
