総合実践！日本語だけで作る本格Webアプリ開発

全ての技術を組み合わせた実践プロジェクト

このレッスンのゴール

これまで学んだ全てのスキル（Nano Banana Pro、ブラウザエージェント、Walkthrough、MCP連携）を組み合わせて、本格的なWebアプリを開発します。企画から完成、ドキュメント作成、デプロイまでを一人で完遂できるようになりましょう。

今回作るもの：タスク管理アプリ

「TaskFlow」- チーム向けタスク管理アプリ

シンプルながら実用的なタスク管理アプリを作成します。タスクの作成・編集・削除、ステータス管理、カテゴリ分類、ダークモード対応など、本格的な機能を盛り込みます。

機能1

タスクCRUD

タスクの作成・表示・編集・削除の基本機能

機能2

ステータス管理

未着手・進行中・完了の3段階ステータス

機能3

カテゴリ分類

仕事・プライベート・学習などのカテゴリ

機能4

フィルタ・検索

ステータス・カテゴリでのフィルタリング

機能5

ダークモード

ライト/ダークモードの切り替え

機能6

データ永続化

LocalStorageでデータを保存

開発の流れ

Phase 1

企画・設計

要件整理とUI設計。Nano Banana Proでワイヤーフレームを作成。

Phase 2

ビジュアル素材作成

Nano Banana Proでアイコン、ロゴ、ヒーロー画像を生成。

Phase 3

基本機能実装

HTML/CSS/JSでタスク管理の基本機能を実装。

Phase 4

動作検証

ブラウザエージェントで自動テスト、問題があれば修正。

Phase 5

ドキュメント作成

Walkthroughで操作マニュアルを自動生成。

Phase 6

公開・共有

GitHubにプッシュ、Notionに開発ログを記録。

Phase 1: 企画・設計

1

要件を整理

まずAIに要件を伝えて整理してもらいます。

「TaskFlowというタスク管理アプリを作りたいです。以下の機能を含めてください： - タスクの追加・編集・削除 - ステータス管理（未着手/進行中/完了） - カテゴリ分類（仕事/プライベート/学習） - ステータスとカテゴリでフィルタリング - ダークモード対応 - LocalStorageでデータ保存 まず、この要件で設計を整理してください」
2

ワイヤーフレームを作成

Nano Banana ProでUIイメージを生成します。

「このタスク管理アプリのUIワイヤーフレームを作成してください。 - 左サイドバー: カテゴリフィルタ、ステータスフィルタ - メインエリア: タスク一覧 - ヘッダー: アプリ名、ダークモード切替ボタン、新規タスクボタン モダンでミニマルなデザインで」

ビジュアルで認識合わせ

いきなりコーディングせず、まずワイヤーフレームを作ることで「思っていたのと違う」を防げます。生成されたイメージを見て、必要に応じて調整を依頼しましょう。

Phase 2: ビジュアル素材作成

// アイコンセットを生成 「TaskFlowアプリ用のアイコンセットを作成してください。 スタイル: フラットデザイン、オレンジ基調（#F97316） サイズ: 24x24px 以下のアイコンを順番に作成： 1. 新規追加（+）アイコン 2. 編集（鉛筆）アイコン 3. 削除（ゴミ箱）アイコン 4. フィルタ（漏斗）アイコン 5. ダークモード（月）アイコン 6. ライトモード（太陽）アイコン」
// ロゴを生成 「TaskFlowのロゴを作成してください。 - チェックマークとフローを組み合わせたシンボル - オレンジ（#F97316）のグラデーション - モダンでプロフェッショナルな印象 - 透過背景」
// 生成した画像を保存 「生成した全てのアイコンとロゴを /assets/images フォルダに保存してください」

Phase 3: 基本機能実装

1

プロジェクト作成

フォルダ構造を作成し、基本ファイルを生成します。

「taskflow-app フォルダを作成して、以下の構造でプロジェクトを初期化してください： taskflow-app/ ├── index.html ├── css/ │ └── style.css ├── js/ │ └── app.js └── assets/ └── images/ （先ほど生成したアイコン） まずはHTMLとCSSを作成してください」
2

機能を段階的に実装

一度に全てではなく、機能ごとに分けて実装を依頼します。

// タスク追加機能 「まず、タスクの追加機能を実装してください。 - 新規タスクボタンをクリックするとモーダルが開く - タイトル（必須）、説明（任意）、カテゴリ（選択）を入力 - 保存ボタンでタスクがリストに追加される - LocalStorageにも保存」
// タスク表示・編集・削除 「次に、以下の機能を追加してください： - タスク一覧の表示 - タスクをクリックすると編集モーダルが開く - 削除ボタンで確認後にタスクを削除 - ステータス変更（未着手→進行中→完了）」
// フィルタ機能 「サイドバーにフィルタ機能を追加してください： - ステータスで絞り込み（全て/未着手/進行中/完了） - カテゴリで絞り込み（全て/仕事/プライベート/学習） - フィルタをクリックするとリストが即座に更新」
// ダークモード 「ダークモード機能を追加してください： - ヘッダーの切り替えボタンでトグル - CSS変数を使ったテーマ切り替え - 選択をLocalStorageに保存して次回も維持」

段階的に実装するメリット

一度に全機能を依頼すると、AIが混乱したり、意図と異なる実装になることがあります。機能ごとに分けて依頼し、都度確認しながら進めるのがポイントです。

Phase 4: 動作検証

// ブラウザエージェントで自動テスト 「ブラウザでTaskFlowアプリを開いて、以下のテストを実行してください： 【正常系テスト】 1. 新規タスク「買い物リスト作成」をカテゴリ「プライベート」で追加 2. タスクがリストに表示されることを確認 3. タスクのステータスを「進行中」に変更 4. 「仕事」カテゴリでフィルタリング 5. リストに「買い物リスト作成」が表示されないことを確認 6. フィルタを「全て」に戻す 【ダークモードテスト】 7. ダークモードに切り替え 8. 画面が暗くなることを確認 9. ページをリロードしてもダークモードが維持されることを確認 各ステップでスクリーンショットを撮影してください」
// 問題があれば自動修正 「テスト中に問題を発見した場合は、自動で修正してから再テストしてください」

テストの自動化

従来なら手動で行っていたUIテストが、自然言語で指示するだけで自動実行されます。問題があれば修正も自動。開発効率が劇的に向上します。

Phase 5: ドキュメント作成

// Walkthroughでマニュアル生成 「TaskFlowアプリの使い方マニュアルをWalkthroughで作成してください。 以下の操作を実演しながらスクリーンショット付きで記録： 1. アプリの起動方法 2. 新規タスクの追加手順 3. タスクの編集方法 4. ステータス変更の仕方 5. フィルタリング機能の使い方 6. ダークモードの切り替え 完了したらMarkdown形式で docs/MANUAL.md に保存」
// READMEも生成 「GitHub用のREADME.mdを作成してください。 - プロジェクト概要 - 機能一覧 - スクリーンショット（ライトモード/ダークモード） - 使い方 - 技術スタック - ライセンス」
