スクショ撮影からマニュアル作成まで！Walkthroughで操作説明書を自動生成

ドキュメント作成の自動化

このレッスンのゴール

Google AntigravityのWalkthrough機能を使って、スクリーンショット付きの操作マニュアルを自動生成できるようになります。開発中の操作をそのまま記録し、ドキュメントとして出力する効率的なワークフローを習得しましょう。

Walkthroughとは？

操作の記録＋自動ドキュメント生成

Walkthroughは、Antigravityでのタスク実行時に「何をしたか」を自動的に記録し、スクリーンショット付きの操作説明書として出力する機能です。変更サマリーと検証ステップが含まれた、チーム共有可能なドキュメントが自動生成されます。

従来のマニュアル作成
  • 操作しながら手動でスクショを撮影
  • Word/Notionに一枚ずつ貼り付け
  • 説明文を手書きで追加
  • 更新があれば全てやり直し
Walkthrough
  • 操作するだけでスクショが自動取得
  • 説明文もAIが自動生成
  • Markdown形式で即座に出力
  • 更新も再実行で自動反映

Walkthroughの仕組み

構成要素1

変更サマリー

タスク完了後に「何が変わったか」を自動でまとめます。ファイルの追加・変更・削除、主要な変更点などが一覧化されます。

構成要素2

検証ステップ

「正しく動作するか確認するための手順」が自動生成されます。スクリーンショットと説明文がセットで出力。

構成要素3

Artifacts

生成された成果物（画像、コード、ドキュメント）が一覧で表示され、ダウンロードや共有が可能。

Walkthroughの使い方

1

通常通りタスクを実行

Playgroundで普通にタスクを依頼します。Walkthroughは自動的に操作を記録しています。

2

タスク完了を確認

AIがタスクを完了すると、Playground下部に「Walkthrough」セクションが表示されます。

3

ドキュメントを確認・エクスポート

自動生成されたドキュメントを確認し、「Export」ボタンでMarkdown/HTML/PDFとして出力。

// Walkthrough付きでタスクを実行 「お問い合わせフォームを作成して。 完了したらWalkthroughドキュメントを生成して」 // または明示的に依頼 「今実行した操作のWalkthroughを出力して」

自動生成されるドキュメントの内容

// 自動生成されるWalkthroughの例（Markdown形式） # お問い合わせフォーム作成 - Walkthrough ## 概要 シンプルなお問い合わせフォームを作成しました。名前、メールアドレス、 お問い合わせ内容の入力フィールドと送信ボタンで構成されています。 ## 変更サマリー - 新規作成: `contact.html` - 新規作成: `styles.css` - 追加されたコード行: 145行 ## 検証ステップ ### ステップ1: ページを開く ブラウザで `contact.html` を開きます。 ![ステップ1](screenshots/step1.png) フォームが正しく表示されていることを確認してください。 ### ステップ2: 入力テスト 各フィールドにテストデータを入力します。 - 名前: テスト太郎 - メールアドレス: test@example.com - お問い合わせ内容: テストメッセージ ![ステップ2](screenshots/step2.png) ### ステップ3: 送信確認 送信ボタンをクリックし、成功メッセージが表示されることを確認。 ![ステップ3](screenshots/step3.png)

そのまま引き継ぎドキュメントに

生成されたWalkthroughは、そのままチームメンバーへの引き継ぎ資料や、QAチームへのテスト手順書として使えます。

実践：操作マニュアルの自動生成

ToDoアプリの操作マニュアルを作成

シンプルなToDoアプリを作成し、その使い方マニュアルをWalkthroughで自動生成してみましょう。

1

ToDoアプリを作成

まずベースとなるアプリを生成します。

「シンプルなToDoアプリを作成して。 - タスクの追加 - タスクの完了チェック - タスクの削除 ローカルストレージに保存する仕様で」
2

操作をブラウザで実演

ブラウザエージェントで実際の操作を実行させます。

「ブラウザでこのアプリを開いて、以下の操作を実行して。 各ステップでスクリーンショットを撮影しながら進めて。 1. 「牛乳を買う」というタスクを追加 2. 「レポートを提出する」というタスクを追加 3. 「牛乳を買う」を完了チェック 4. 「レポートを提出する」を削除」
3

Walkthroughドキュメントを生成

操作が完了したら、マニュアルを出力します。

「今実行した操作をもとに、このToDoアプリの使い方マニュアルをWalkthroughとして出力して。 スクリーンショット付きで、Markdown形式で保存して」
4

HTMLマニュアルに変換

必要に応じてHTMLページとしても出力できます。

「このWalkthroughを見やすいHTMLページに変換して。 CSSも適用して、印刷にも対応したレイアウトで」

Artifactsの種類と活用

種類1

スクリーンショット

操作中に取得した画面キャプチャ。PNG形式で保存され、マニュアルに自動埋め込み。

種類2

コードスニペット

生成・変更されたコードの抜粋。技術ドキュメントに埋め込み可能。

種類3

動画録画

ブラウザエージェントの操作を動画として記録。GIF/MP4形式で出力可能。

種類4

差分レポート

変更前後のコード差分。コードレビューやデバッグに活用。

// Artifactsの取得 「今回の変更の差分レポートを表示して」 「ブラウザ操作を動画で記録して」 「生成した全てのArtifactsを /docs フォルダに保存して」

エクスポート形式と活用シーン

形式 特徴 活用シーン
Markdown テキストベース、バージョン管理しやすい GitHub README、Notion、技術ドキュメント
HTML そのままブラウザで表示可能 社内ポータル、ヘルプページ
PDF 印刷・配布に最適 クライアント納品、印刷マニュアル
Notion Notionページとして直接出力 チームのナレッジベース
// エクスポート指示の例 「WalkthroughをPDF形式で出力して」 「このドキュメントをNotionに直接エクスポートして」 「GitHubのREADME.mdとして保存して」

チームへの共有方法

方法1

GitHubにプッシュ

Markdownファイルとスクリーンショットをリポジトリにプッシュ。README.mdとして公開。

方法2

Notionに同期

MCP連携を使えば、Notionのページとして直接出力可能（次のレッスンで詳しく解説）。

方法3

Slackで共有

PDFに変換してSlackのチャンネルに投稿。または要約だけをメッセージとして送信。
