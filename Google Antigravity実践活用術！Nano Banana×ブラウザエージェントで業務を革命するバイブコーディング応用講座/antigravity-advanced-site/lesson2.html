AIがブラウザを勝手に動かす！ブラウザエージェントで動作確認を完全自動化

Chrome連携で開発・テストを革命する

このレッスンのゴール

Google AntigravityのブラウザエージェントとChrome拡張機能を連携させ、作成したWebアプリのUI検証を自動化できるようになります。「ブラウザでテストして」の一言で、クリック・入力・スクロールが自動実行される驚きの開発体験を習得しましょう。

ブラウザエージェントとは？

AIがChromeを自律操作する仕組み

ブラウザエージェントは、AntigravityのAIがChromeブラウザを直接操作できる機能です。マウスカーソルの移動、クリック、テキスト入力、スクロールなど、人間がブラウザで行う操作をAIが自動で実行します。

従来の動作確認
  • コードを書く→ブラウザを開く→手動で確認
  • エラー発見→コードに戻る→修正→再確認
  • フォーム入力を何度も手動で繰り返す
  • スクリーンショットを手動で撮影
ブラウザエージェント
  • 「ブラウザで開いて確認して」と指示するだけ
  • エラー検知→自動で修正→再確認まで全自動
  • フォーム入力もAIが自動で実行
  • 必要なスクリーンショットを自動取得

ブラウザエージェントのセットアップ

1

Chrome拡張機能をインストール

Chrome ウェブストアで「Google Antigravity Connector」を検索してインストールします。または、Antigravity内の設定画面からインストールリンクにアクセスできます。

2

拡張機能を有効化

インストール後、Chromeのツールバーに表示されるAntigravityアイコンをクリックし、「Antigravityと接続」を選択。

3

接続を確認

Antigravityの設定画面で「ブラウザエージェント: 接続済み」と表示されていればOK。

セキュリティについて

ブラウザエージェントはローカル環境でのみ動作します。外部サイトへのログイン情報などはAntigravityに送信されません。ただし、開発中のサイト以外での使用は推奨されません。

基本的な使い方

// Playgroundでの指示例 「作成したindex.htmlをブラウザで開いて」 → Chromeが起動し、ファイルが表示される 「ページが正しく表示されているか確認して」 → AIがページを見て、問題がないかチェック 「フォームに名前を入力して送信ボタンを押して」 → AIがフォームに自動入力して送信 「ページを下までスクロールして」 → AIがスクロール操作を実行

自然言語で操作できる

「クリック」「入力」「スクロール」といった操作を、普通の日本語で指示するだけ。プログラミング的な記述は一切不要です。

自動操作できること

操作1

クリック

ボタン、リンク、メニューなど任意の要素をクリック。「送信ボタンをクリックして」のように指示。

操作2

テキスト入力

フォームフィールドへのテキスト入力。「メールアドレス欄に test@example.com と入力して」など。

操作3

スクロール

ページ全体や特定領域のスクロール。「ページ下部までスクロール」「商品一覧を右にスクロール」など。

操作4

ホバー

マウスオーバー動作。「メニューにマウスを合わせて」でドロップダウン表示を確認。

操作5

ページ遷移

リンクをたどって別ページへ移動。「お問い合わせページに移動して」など。

操作6

スクリーンショット

現在の画面をキャプチャ。「今の画面のスクリーンショットを撮って」と指示。

実践：フォーム入力の自動テスト

お問い合わせフォームの動作確認を自動化

シンプルなお問い合わせフォームを作成し、ブラウザエージェントで入力・送信のテストを自動化してみましょう。

1

フォームを作成

まずAntigravityでシンプルなフォームを生成します。

// Playgroundへの指示 「お問い合わせフォームを作成して。 - 名前（必須） - メールアドレス（必須） - お問い合わせ内容（テキストエリア） - 送信ボタン バリデーション付きで、送信成功時はアラートを表示」
2

ブラウザで開く

作成したフォームをブラウザエージェントで表示します。

「作成したフォームをChromeで開いて」
3

自動入力テスト

フォームに自動でデータを入力させます。

「以下のテストデータでフォームに入力して： - 名前: テスト太郎 - メールアドレス: test@example.com - お問い合わせ内容: これはテスト送信です。 入力後、送信ボタンをクリックして」
4

バリデーションテスト

エラーケースも自動でテストできます。

「次は、名前を空欄にしたまま送信ボタンを押して。 バリデーションエラーが表示されるか確認して」

結果報告

ブラウザエージェントは操作を実行した後、「アラートが表示されました」「エラーメッセージ『名前は必須です』が表示されています」のように結果を報告してくれます。

エラー検知と自動修正ループ

AIが問題を発見したら自動で修正

ブラウザエージェントは単に操作するだけでなく、問題を検知したら自動でコードを修正し、再度テストする「自動修正ループ」機能を持っています。

Step 1

問題検知

ブラウザエージェントがページを確認し、レイアウト崩れ、エラーメッセージ、動作不良などを発見。

Step 2

原因分析

AIがコードを分析し、問題の原因を特定。「CSSのflexboxの設定が間違っている」など。

Step 3

自動修正

コードを自動で修正し、ファイルを保存。

Step 4

再確認

修正後のページを再度読み込み、問題が解決したか確認。必要に応じて再修正。

// 自動修正を依頼する指示 「このページをブラウザで開いて、何か問題があれば自動で修正して」 「フォームをテストして、バグがあれば直して」 「ページの見た目を確認して、レイアウトが崩れていたら修正して」

Gemini 2.5 UI Checkpointの役割

AIがUIを「見て」判断する

ブラウザエージェントには「Gemini 2.5 UI Checkpoint」というマルチモーダルモデルが搭載されています。これはスクリーンショットを「見て」理解し、UIの問題を検知できる技術です。

検知できること 例
レイアウト崩れ 「ボタンがはみ出している」「テキストが重なっている」
デザインの不整合 「フォントサイズが不統一」「色が見づらい」
アクセシビリティ問題 「コントラスト比が低い」「クリック領域が小さい」
エラー表示 「404エラーが表示されている」「JavaScript例外」
動作不良 「ボタンを押しても反応がない」「ローディングが終わらない」

複数ページにまたがるテスト

// ページ遷移を含む自動テスト 「以下の操作を順番に実行して： 1. index.htmlをブラウザで開く 2. 『お問い合わせ』リンクをクリック 3. contact.htmlに遷移したことを確認 4. フォームにテストデータを入力して送信 5. 送信完了ページが表示されることを確認 6. 各ステップのスクリーンショットを保存」
