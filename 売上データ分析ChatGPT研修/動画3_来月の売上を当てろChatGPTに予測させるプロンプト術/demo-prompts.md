# 動画3：デモプロンプト集
## 来月の売上を当てろ：ChatGPTに予測させるプロンプト術

### 1. 基本的な売上予測プロンプト
```
過去12ヶ月の売上データを使って、来月の売上予測を行ってください。

データ：月次売上データ（日付、売上金額）
要求：
1. トレンド分析（上昇・下降・横ばいの判定）
2. 季節性の有無を確認
3. 線形回帰による予測値の算出
4. 予測の信頼区間を計算
5. 予測の根拠と精度を説明

結果をグラフと数値で分かりやすく表示してください。
```

### 2. ARIMA モデル予測プロンプト
```
時系列売上データにARIMAモデルを適用して、6ヶ月先までの売上予測を行うPythonコードを書いてください。

技術要件：
- statsmodelsライブラリのARIMAを使用
- 自動パラメータ選択（AIC基準）
- モデルの診断（残差分析、ACF/PACF）
- 予測値と信頼区間の算出
- 予測精度の評価（MAE、RMSE、MAPE）

コードの実行結果として、予測グラフと精度指標を表示してください。
```

### 3. 機械学習予測プロンプト
```
売上予測にRandom Forestモデルを使用するPythonコードを作成してください。

特徴量設計：
1. ラグ変数（1ヶ月前、3ヶ月前、12ヶ月前の売上）
2. 移動平均（3ヶ月、6ヶ月、12ヶ月）
3. 季節性指標（月、四半期）
4. トレンド指標（線形、多項式）

評価方法：
- 時系列クロスバリデーション
- 特徴量重要度の分析
- 予測値の可視化
- ビジネス解釈の提供
```

### 4. 季節性考慮予測プロンプト
```
売上データの季節性を詳細に分析し、季節調整済み予測を行ってください。

分析内容：
1. 時系列分解（トレンド、季節性、残差の分離）
2. 季節性の統計的検定
3. 季節指数の計算と可視化
4. 季節調整後の予測モデル構築
5. 最終予測値の季節性復元

出力：
- 分解結果のグラフ
- 季節指数表
- 調整前後の予測比較
- 季節要因の解釈
```

### 5. 外部要因を考慮した予測プロンプト
```
売上予測に外部要因を組み込んだ高度な分析を行ってください。

考慮する外部要因：
1. 経済指標（GDP、消費者物価指数）
2. 競合他社の動向
3. プロモーション・キャンペーンの影響
4. 天候・季節イベント
5. マーケティング投資額

分析手法：
- 多重回帰分析
- 因果関係の検証
- 外部変数の影響度評価
- シナリオ分析（楽観・標準・悲観）

結果として、要因別の寄与度と実行可能な改善策を提示してください。
```

### 6. アンサンブル予測プロンプト
```
複数の予測モデルを組み合わせたアンサンブル予測システムを構築してください。

使用モデル：
1. ARIMA（時系列専用）
2. Random Forest（機械学習）
3. 指数平滑法（トレンド・季節性対応）
4. 線形回帰（シンプルベースライン）

アンサンブル手法：
- 単純平均
- 重み付き平均（過去精度ベース）
- スタッキング（メタラーナーによる組み合わせ）

評価・比較：
- 個別モデルと比較した精度向上の確認
- 予測の安定性評価
- 計算コストとのトレードオフ分析
```

### 7. リアルタイム予測更新プロンプト
```
新しい売上データが追加されるたびに予測を自動更新するシステムを作成してください。

機能要件：
1. 新データの検証（異常値、欠損値チェック）
2. モデルの再訓練判定（性能劣化の検出）
3. オンライン学習による予測更新
4. 予測精度の継続監視
5. アラート機能（予測大幅変更時）

技術実装：
- データパイプラインの自動化
- モデルのバージョン管理
- ログ・監視機能
- エラーハンドリング
- パフォーマンス最適化
```

### 8. 不確実性を考慮した予測プロンプト
```
予測の不確実性を定量化し、リスク管理に活用できる売上予測を作成してください。

不確実性の定量化：
1. 予測区間の算出（信頼区間とは異なる）
2. ベイズ手法による事後分布の推定
3. モンテカルロシミュレーション
4. シナリオ分析（確率重み付き）

リスク評価：
- VaR（Value at Risk）の計算
- 売上目標達成確率
- 最悪ケースの影響分析
- リスク要因の特定と対策

結果の活用：
- 在庫計画への反映
- 資金繰り計画の策定
- リスク管理戦略の立案
```

### 9. 予測精度向上プロンプト
```
既存の売上予測モデルの精度を向上させるための分析と改善提案を行ってください。

精度向上のアプローチ：
1. 特徴量エンジニアリング（新変数の創出）
2. データ品質の改善（外れ値処理、欠損値補完）
3. モデルハイパーパラメータの最適化
4. 予測期間の最適化（短期vs長期）
5. モデル選択基準の見直し

改善検証：
- A/Bテストによる改善効果の測定
- 統計的有意性の検定
- ビジネスインパクトの評価
- 改善コストと効果の比較

継続改善：
- 定期的な精度監視体制
- 自動再学習の仕組み
- フィードバックループの構築
```

### 10. 予測結果のビジネス活用プロンプト
```
売上予測結果を経営判断とビジネス戦略に直接活用するためのレポートを作成してください。

レポート構成：
1. エグゼクティブサマリー
   - 予測売上額と達成確率
   - 前年同期比、前月比
   - 重要な変化点と要因

2. 詳細予測結果
   - 商品別・地域別予測
   - 季節性・トレンド分析
   - 不確実性の範囲

3. ビジネス影響分析
   - 目標達成のシナリオ
   - リスク要因と対策
   - 機会の特定

4. アクション提案
   - 短期的改善策（1-3ヶ月）
   - 中長期戦略（6-12ヶ月）
   - 投資・リソース配分の推奨

5. 監視・評価指標
   - KPIダッシュボード
   - 早期警告システム
   - 継続改善計画

ビジネスリーダーが即座に意思決定できる形式で、データ根拠も明確に示してください。
```
