# 動画4：デモプロンプト集
## 作業ゼロで見える化：Looker Studioで自動更新ダッシュボード

### 1. ダッシュボード設計プロンプト
```
売上ダッシュボード用の重要KPIを選定し、Looker Studioでの配置設計を提案してください。

要件：
1. 経営判断に直結する指標の選定
2. 階層構造（全社→部門→商品）での分析可能
3. 時間軸（日次、月次、年次）での比較
4. アラート機能が必要な指標の特定
5. モバイル表示にも対応した配置

各KPIの計算式とビジネス上の意義も説明してください。
```

### 2. データ準備・接続プロンプト
```
Looker Studioに接続する売上データの準備をGoogle Apps Scriptで自動化してください。

機能要件：
1. 複数のCSVファイルを統合してスプレッドシートに集約
2. データの品質チェック（重複、欠損、異常値）
3. 日時自動化（毎日午前9時に実行）
4. エラー時のメール通知機能
5. ログ記録とエラーハンドリング

データ前処理：
- 日付形式の統一
- 数値データの検証
- カテゴリの正規化
- 計算フィールドの自動生成

実行可能なGoogle Apps Scriptコード全体を提供してください。
```

### 3. KPI自動計算プロンプト
```
Looker StudioとBigQueryを使用して、以下のKPIを自動計算する仕組みを構築してください：

主要KPI：
1. 総売上・前年同月比・成長率
2. 平均客単価・RFM分析
3. 商品別売上ランキング・ABC分析
4. 地域別売上・構成比
5. 新規・既存顧客の売上寄与度

技術要件：
- BigQueryでのSQL関数使用
- 計算フィールドの効率的な設計
- リアルタイム更新対応
- パフォーマンス最適化

各KPIのSQL文とLooker Studioでの表示設定を具体的に示してください。
```

### 4. 動的フィルター機能プロンプト
```
ユーザーが柔軟にデータを絞り込める動的フィルター機能をLooker Studioで実装してください。

フィルター項目：
1. 期間選択（日付範囲、相対期間）
2. 商品カテゴリ（複数選択可能）
3. 地域・店舗（階層構造対応）
4. 売上金額範囲（スライダー）
5. 顧客セグメント（新規/既存/VIP等）

高度な機能：
- フィルター間の連動機能
- 保存済みフィルタープリセット
- URLパラメータでの直接アクセス
- フィルター状態の共有機能

実装方法と設定手順を詳細に説明してください。
```

### 5. アラート・通知システムプロンプト
```
売上目標の達成状況や異常値を検出して自動通知するシステムをGoogle Apps Scriptで作成してください。

アラート条件：
1. 日次売上が目標を20%下回った場合
2. 前日比で30%以上の変動があった場合
3. 特定商品の在庫切れ兆候（売上急減）
4. 新規顧客獲得数の異常な減少
5. システムエラーや データ更新失敗

通知方式：
- メール通知（重要度別）
- Slack/Teams連携
- SMS通知（緊急時）
- ダッシュボード内のアラート表示

各アラートの判定ロジックと通知テンプレートも含めて実装してください。
```

### 6. レスポンシブデザインプロンプト
```
スマートフォンとタブレットでも見やすい売上ダッシュボードのレスポンシブデザインを設計してください。

3. 重要KPIの優先表示

具体的対応：
- KPIカードのサイズ調整

- フィルター操作のユーザビリティ
- 横スクロール回避
- 読み込み速度の向上

各デバイスでの表示例とLooker Studioでの設定方法を説明してください。
```

### 7. パフォーマンス最適化プロンプト
```
大量データ（100万レコード以上）でも高速に動作するLooker Studioダッシュボードを構築してください。

最適化手法：
1. データソースの事前集計
2. 効率的なSQL文の設計
3. 適切なデータ型の選択
4. インデックス設定の最適化
5. キャッシュ戦略の活用

BigQuery側の最適化：
- パーティション分割の設計
- クラスタリング設定
- マテリアライズドビューの活用
- 費用対効果の考慮

Looker Studio側の最適化：
- データの抽出タイミング
- ブレンド機能の適切な使用
- フィルタリング処理の最適化

実装手順とパフォーマンス測定方法も含めて説明してください。
```

### 8. セキュリティ・権限管理プロンプト
```
機密性の高い売上データを安全に共有できる権限管理システムをLooker Studioで設計してください。

権限レベル：
1. 経営層：全データアクセス可能
2. 部門管理者：担当部門のみ閲覧
3. 営業担当：個人実績のみ表示
4. 外部パートナー：限定データのみ
5. 閲覧専用ユーザー：編集権限なし

セキュリティ対策：
- 行レベルセキュリティの実装
- IPアドレス制限
- アクセスログの記録
- データの仮名化・マスキング
- 定期的なアクセス権見直し

Google Workspaceとの連携も含めて、具体的な設定手順を説明してください。
```

### 9. 自動レポート配信プロンプト
```
Looker Studioダッシュボードの内容を定期的にPDFレポートとして自動配信するシステムを構築してください。

配信スケジュール：
1. 日次レポート（営業日の朝9時）
2. 週次サマリー（月曜日朝）
3. 月次詳細レポート（月初3営業日以内）
4. 四半期レポート（四半期終了後1週間以内）

レポート内容：
- KPIサマリー
- 主要グラフの静的画像
- 前期比較・分析コメント
- アクションが必要な項目のハイライト

技術実装：
- Google Apps Scriptでの自動化
- PDFテンプレートの作成
- 配信リストの管理
- 配信失敗時の再送処理

完全に自動化されたシステムの実装コードを提供してください。
```

```
Looker Studioダッシュボードの継続的な運用とメンテナンス体制を構築するためのガイドラインを作成してください。

1. 定期的なデータ品質チェック

- 四半期：システム全体の評価と改善

品質管理：
- SLA（Service Level Agreement）の設定
- 監視指標とアラート
- インシデント対応手順
- 継続改善プロセス

体系的な運用マニュアルと、実際に使用できるチェックリストを作成してください。
```
