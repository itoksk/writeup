ChatGPTがうまく働かない

指示ミスを防ぐ5つの型

学習目標

ChatGPTを使った売上データ分析でよくある指示ミスのパターンを理解し、確実に期待した結果を得るための効果的なプロンプト設計技術を習得します。

よくある失敗パターン

曖昧な指示

「売上を分析して」など、具体性に欠ける指示で期待と異なる結果になるパターンです。

前提条件不足

データ形式や制約条件を明示せず、実行できないコードが生成されるパターンです。

出力形式未指定

どのような形で結果が欲しいか伝えず、使いにくい出力になるパターンです。

型1：明確な指示

明確な指示の要素

1分析対象：何のデータを分析するか

2分析期間：どの期間のデータか

3分析目的：何を明らかにしたいか

4期待する洞察：どのような発見を求めるか

悪い例

「売上データを分析してください。」

改善点：何をどう分析するか不明確

良い例

「過去12ヶ月の売上データから、月次売上トレンドを分析し、季節性の有無を判定してください。目的は来月の売上予測の精度向上です。」

改善点：期間、分析内容、目的が明確

# 明確な指示のテンプレート
【分析対象】: ○○のデータ
【期間】: ○年○月～○年○月
【分析目的】: ○○を判断するため
【期待する洞察】: ○○の傾向があるかどうか
【活用方法】: ○○の意思決定に使用

例：
【分析対象】: 商品別売上データ
【期間】: 2023年1月～2023年12月
【分析目的】: 収益性の高い商品を特定するため
【期待する洞察】: 利益率と売上量の関係性
【活用方法】: 来年度の商品戦略策定に使用

型2：具体例の提示

データ形式の例示

CSV構造、列名、データ型を具体的に示すことで、正確な処理が可能になります。

期待する出力例

どのような結果が欲しいか、具体的な例を示すことで期待値を明確化します。

具体例を含むプロンプト

以下の形式のCSVデータ（日付,商品名,売上金額,数量）を分析し、『商品A: 総売上○○円, 平均単価○○円』の形式で上位5商品を表示してください。
# データ構造の例示テンプレート
【データ形式】
- ファイル: CSV（UTF-8, カンマ区切り）
- 列構成: 
  * 日付 (YYYY-MM-DD)
  * 商品名 (文字列)
  * 売上金額 (数値)
  * 数量 (整数)

【出力例】
商品ランキング:
1. 商品A: 総売上1,250,000円, 平均単価2,500円
2. 商品B: 総売上980,000円, 平均単価1,960円
...

【処理要件】
- 欠損値は0で補完
- 異常値（単価10万円以上）は除外
- 売上金額の降順でソート

型3：制約条件の明示

明示すべき制約条件

  • データ制約：サイズ、品質、エンコード形式
  • 処理制約：使用可能ライブラリ、実行時間制限
  • 環境制約：メモリ制限、ファイルパス
  • 出力制約：ファイル形式、保存先、権限

    • 制約条件を含むプロンプト例

    売上CSVファイル（最大10MB、pandas使用）を読み込み、メモリ使用量8GB以下で処理し、結果をPNG形式（DPI300）で保存してください。処理時間は5分以内に完了させてください。
    # 制約条件の明示テンプレート
【データ制約】
- ファイルサイズ: 最大○○MB
- データ品質: 欠損値○○%以下
- 文字コード: UTF-8必須

【技術制約】
- 使用言語: Python 3.8+
- 必須ライブラリ: pandas, matplotlib
- 禁止ライブラリ: seaborn（利用不可）

【環境制約】
- メモリ制限: ○○GB以下
- 実行時間: ○○分以内
- 保存先: ./output/フォルダ

【品質制約】
- グラフ解像度: DPI300以上
- 統計精度: 小数点第2位まで
- エラー許容率: 1%以下

    型4：段階的な指示

    ステップ分割のメリット

    複雑な処理を段階的に分割することで、各ステップでの検証が可能になります。

    エラー対処の効率化

    問題が発生した箇所を特定しやすく、修正が容易になります。

    段階的指示の流れ

    1データ読み込み：ファイル存在確認と基本情報表示

    2データ検証：品質チェックと前処理

    3基本分析：統計量計算と可視化

    4詳細分析：高度な分析と解釈

    5結果出力：レポート作成と保存

    # 段階的指示のテンプレート

# ステップ1：データ読み込み確認
"""
売上CSVファイルを読み込み、以下を表示してください：
1. データの行数・列数
2. 各列のデータ型
3. 最初の5行のサンプル
4. 基本統計量（平均、中央値、標準偏差）
5. 欠損値の個数と比率

この結果を確認してから次のステップに進みます。
"""

# ステップ2：データ前処理
"""
前のステップで確認したデータに対して：
1. 欠損値を線形補間で補完
2. 異常値を3σ法で検出・除外
3. 処理前後の件数比較表を作成
4. データ品質スコアを算出

処理結果が妥当か確認してから分析に進みます。
"""

# このように段階的に進行

    型5：出力形式の指定

    グラフ仕様

    グラフの種類、サイズ、色、フォント、保存形式を詳細に指定します。

    データ出力

    ファイル形式、統計量、解釈コメントの有無を明確に指定します。

    レポート構成

    章立て、文章スタイル、図表の配置を具体的に指示します。

    詳細な出力指定例

    12×8インチの折れ線グラフ、タイトル『月別売上推移』、X軸『月』、Y軸『売上金額（万円）』、グリッド表示あり、青色の線、フォントサイズ12pt、PNG形式でsales_trend.pngに保存してください。
    # 出力形式の詳細指定テンプレート

【グラフ出力】
- 種類: ○○グラフ（折れ線、棒、円など）
- サイズ: 幅○○ × 高さ○○ インチ
- 色設定: メインカラー○○、アクセント○○
- フォント: ○○、サイズ○○pt
- タイトル: 「○○」（位置：中央上部）
- 軸設定: X軸「○○」、Y軸「○○」
- 保存: PNG、DPI300、ファイル名「○○.png」

【データ出力】
- 形式: CSV、UTF-8エンコード
- 統計量: 平均、中央値、標準偏差を含む
- 解釈: 各結果の意味を日本語で説明
- 推奨アクション: 具体的な次のステップ3つ

【レポート構成】
1. エグゼクティブサマリー（200文字以内）
2. 分析結果（図表付き）
3. 考察・解釈（ビジネス観点）
4. 推奨アクション（優先順位付き）

    5つの型チェックリスト

    • 目的と期待する結果が明確か？
    • 具体的な例を提示したか？
    • 制約条件を詳述したか？
    • 段階的に進行する設計にしたか？
    • 出力形式を具体的に指定したか？

    プロンプト品質指標

    • 明確性：曖昧な表現がない
    • 完全性：必要な情報が全て含まれる
    • 実行可能性：技術的に実現可能
    • 検証可能性：結果の妥当性を判断できる
    # 完璧なプロンプトのテンプレート
"""
【目的】○○を分析して○○を判断したい

【データ詳細】

- 構造：列名とデータ型を明記
- サイズ：約○○行、○○MB
- 品質：欠損値○○%、想定される異常値

【制約条件】
- 技術：使用可能ライブラリ、実行環境
- 品質：精度要件、処理時間制限
- 出力：ファイル形式、保存先、権限

【段階的処理】
1. ステップ○：○○を実行（確認ポイント：○○）
2. ステップ○：○○を実行（確認ポイント：○○）

【出力指定】
- グラフ：種類、サイズ、色、ファイル名
- データ：形式、統計量、解釈コメント
- レポート：構成、文章スタイル、図表配置

【確認ポイント】
各ステップで○○を確認してから次に進む
