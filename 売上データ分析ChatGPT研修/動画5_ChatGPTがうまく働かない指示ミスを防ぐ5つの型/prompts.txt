# 第5回：効果的なプロンプト作成 - 5つの型テンプレート集

## 第1の型：目的明示型

### 基本テンプレート
【目的】[何のために必要か]のため、[具体的な要求]をしてください。

### 実践例

#### 1. 経営報告用
【目的】月次役員会議で売上状況を報告するため、このデータから重要なポイントを3つ抽出してください。

#### 2. 問題解決用
【目的】売上低下の原因を特定するため、前年同期と比較して大きく変化した要因を分析してください。

#### 3. 意思決定支援
【目的】来期の予算策定の参考にするため、季節変動パターンと成長トレンドを明確にしてください。

#### 4. チーム共有用
【目的】営業チームに共有するため、商品カテゴリ別の売上実績を分かりやすくまとめてください。

#### 5. 顧客説明用
【目的】クライアントへの提案資料作成のため、投資効果を数値で示してください。

## 第2の型：形式指定型

### 基本テンプレート
[要求内容]を以下の形式で出力してください：
- 形式1
- 形式2
- 形式3

### 実践例

#### 6. 表形式出力
売上データの分析結果を以下の表形式で出力してください：
| 月 | 売上 | 前月比 | 前年同月比 |

#### 7. JSON形式
分析結果をJSON形式で構造化して出力してください：
```json
{
  "summary": "",
  "details": [],
  "recommendations": []
}
```

#### 8. Markdown階層
分析レポートをMarkdown形式で以下の構成で作成してください：
# タイトル
## 1. サマリー
## 2. 詳細分析
## 3. 提言

#### 9. 箇条書き形式
改善提案を以下の形式でまとめてください：
- 課題：[具体的な問題]
- 原因：[根本原因]
- 対策：[実行可能な解決策]

#### 10. レポート形式
エグゼクティブサマリーを以下の構成で作成してください：
1. 結論（1行）
2. 主要指標（3項目）
3. アクションアイテム（3項目）

## 第3の型：粒度指定型

### 基本テンプレート
[要求内容]を[詳細度レベル]で説明してください。

### 実践例

#### 11. 要約レベル
売上分析結果を経営層向けに3行以内で要約してください。

#### 12. 標準レベル
売上トレンドを管理職向けに、主要ポイント5つで説明してください。

#### 13. 詳細レベル
予測モデルの構築方法を、プログラミング初心者でも実装できるよう、ステップバイステップで詳細に説明してください。

#### 14. 段階的説明
この分析結果を：
- 役員向け：1行で
- 部長向け：1段落で
- 担当者向け：詳細な手順付きで
それぞれ説明してください。

#### 15. カスタム粒度
技術的な説明は省略し、ビジネスインパクトのみを重点的に、A4用紙1枚に収まる分量で説明してください。

## 第4の型：プロセス指定型

### 基本テンプレート
以下の手順で[要求内容]を実行してください：
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]

### 実践例

#### 16. 分析プロセス
売上データの異常値を以下の手順で検出してください：
1. まず基本統計量を確認
2. 次に標準偏差の2倍を超える値を特定
3. それらの発生月と要因を分析
4. 最後に対処法を提案

#### 17. 予測プロセス
来月の売上を以下の手順で予測してください：
1. 過去データのトレンドを分析
2. 季節性パターンを抽出
3. 外部要因を考慮
4. 複数モデルで予測
5. 最も信頼性の高い予測値を選定

#### 18. レポート作成プロセス
月次レポートを以下の流れで作成してください：
1. 前月の振り返り
2. 当月の実績分析
3. 目標との差異分析
4. 改善提案
5. 来月のアクションプラン

#### 19. 問題解決プロセス
売上低下の原因を以下の手順で特定してください：
1. 現状の把握（数値確認）
2. 前期との比較
3. 変化要因の列挙
4. 影響度の評価
5. 根本原因の特定

#### 20. 検証プロセス
予測精度を以下の手順で検証してください：
1. 過去6ヶ月の予測と実績を比較
2. 誤差率を計算
3. 誤差パターンを分析
4. モデルの改善点を提案

## 第5の型：エラー修正型

### 基本テンプレート
[エラー内容]が発生しました。[追加情報]を考慮して修正してください。

### 実践例

#### 21. コードエラー修正
以下のエラーが出ました：
```
NameError: name 'pd' is not defined
```
必要なインポート文を追加して、完全なコードを提供してください。

#### 22. 期待値との差異修正
先ほどの回答は求めていたものと違いました。
期待：月別の詳細な数値分析
実際：年間サマリーのみ
月別の詳細データを含めて再度分析してください。

#### 23. 形式エラー修正
グラフの日本語が文字化けしています。日本語フォントの設定を追加して、修正版のコードを提供してください。

#### 24. 計算エラー修正
前月比の計算が間違っているようです。正しい計算式を使って再計算してください。
現在の式：(当月-前月)/前月
正しい式：((当月-前月)/前月)*100

#### 25. 論理エラー修正
季節調整の方法が適切でないようです。乗法モデルではなく加法モデルを使用して再分析してください。

## 複合型プロンプト（5つの型の組み合わせ）

### 26. 完全指定型プロンプト
【目的】月次役員会議での売上報告のため
【形式】エグゼクティブサマリー形式で
【粒度】要点3つを各2行以内で
【プロセス】現状分析→課題抽出→改善提案の順で
【データ】添付の2023年売上データを使用して
上記条件で、1月の売上レビューを作成してください。

### 27. 段階的詳細化プロンプト
【第1段階】売上の概要を1行で
【第2段階】主要な変化を3点で
【第3段階】それぞれの原因を分析
【第4段階】具体的な対策を提案
【形式】番号付きリストで出力

### 28. 条件分岐型プロンプト
売上データを分析し：
- 成長している場合→さらなる成長戦略を3つ
- 横ばいの場合→ブレークスルー施策を3つ
- 下降している場合→回復策を3つ
提案してください。

### 29. 反復改善型プロンプト
1. まず簡易分析を実施
2. 問題点があれば詳細分析
3. 改善案を提示
4. 実現可能性を評価
5. 最終提案をまとめる
この流れで分析を進めてください。

### 30. カスタマイズ型プロンプト
【対象】[営業部門]
【目的】[売上向上施策の立案]
【制約】[予算100万円以内]
【期限】[3ヶ月以内に実施]
【成果指標】[売上10%向上]
上記条件での実行計画を作成してください。

## プロンプト作成のゴールデンルール

### 1. 明確性の原則
曖昧な表現を避け、具体的な数値や条件を明示する

### 2. 構造化の原則
要求事項を論理的に整理し、番号や箇条書きで構造化する

### 3. 文脈提供の原則
背景情報、制約条件、期待する成果を明確に伝える

### 4. 段階的詳細化の原則
複雑な要求は小さなステップに分解して依頼する

### 5. フィードバックループの原則
結果を確認し、必要に応じて修正依頼を明確に伝える

## よく使うプロンプトパーツ

### 開始フレーズ
- 「以下の条件で...」
- 「...を目的として...」
- 「...の観点から...」

### 指示フレーズ
- 「具体的に...」
- 「ステップバイステップで...」
- 「...の形式で出力して...」

### 制約フレーズ
- 「ただし...は除く」
- 「...の範囲内で」
- 「...を考慮して」

### 確認フレーズ
- 「理解できたら要約して」
- 「不明な点があれば質問して」
- 「実行前に確認事項を列挙して」
