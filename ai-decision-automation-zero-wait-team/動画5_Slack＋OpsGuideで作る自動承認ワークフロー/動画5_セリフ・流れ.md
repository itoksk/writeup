## Step 1: Slack App作成（3:00-4:30）
「最初にSlack側の設定をします」

### 実演：Slack App作成
```
1. api.slack.com にアクセス
2. "Create New App" → "From scratch"
3. App Name: "AI承認アシスタント"
4. ワークスペースを選択
```

### Bot権限の設定
```
OAuth & Permissions で以下を追加：
- app_mentions:read（メンション読み取り）
- chat:write（メッセージ送信）
- channels:history（履歴読み取り）
- channels:read（チャンネル情報）
```

「たった4つの権限だけ。セキュリティも安心です」

### Botトークン取得
```
1. "Install to Workspace"をクリック
2. Bot User OAuth Tokenをコピー
   （xoxb-で始まる文字列）
```

## Step 2: Make シナリオ作成（4:30-7:00）
「いよいよMakeで自動化フローを作ります」

### 新規シナリオ作成
```
1. Make.com にログイン
2. "Create a new scenario"
3. テンプレートは使わず空から開始
```

### 3つのモジュール設定

**1. Slack "Watch Events"（トリガー）**
```
設定内容：
- Connection: 新規作成
- Bot Token: 先ほどコピーしたもの
- Event Type: app_mention
```

**2. OpenAI "Create a Completion"（AI判断）**
```
設定内容：
- Connection: APIキーで接続
- Model: gpt-4
- Messages: 
  - Role: system
  - Content: [動画3で作成したFew-shot学習プロンプト]
  
  - Role: user  
  - Content: {{1.event.text}}
```

**3. Slack "Create a Message"（結果返信）**
```
設定内容：
- Connection: 同じBot Token
- Channel: {{1.event.channel}}
- Text: {{2.choices[0].message.content}}
- Thread TS: {{1.event.ts}}（スレッド返信）
```

「ドラッグ&ドロップで繋ぐだけ。コードは一切書きません」

以下の基準と過去事例に基づいて判断してください。

## 過去の判断事例（動画3で追加）

必ず以下の形式で回答してください：

【判断】承認/条件付き承認/却下/追加情報要求
【類似事例】過去の○○と類似
【理由】

## 動作テスト（8:30-9:30）
「では、実際に動かしてみましょう」

### テスト手順
1. **Makeでシナリオを有効化**
2. **Slackで専用チャンネル作成**（#ai-approval）
3. **Botをチャンネルに招待**
4. **テスト投稿**

### 実際のテスト
```
投稿例：
@AI承認アシスタント 
営業支援ツール「SalesBoost」月額4万円の導入承認をお願いします。
営業チーム15名で利用、活動時間30%削減見込みです。

↓（3秒後）

AI返信：
【判断】承認
【類似事例】過去のSFAツール導入30万円と類似
【理由】
- 理由1: 月額4万円は承認基準内で妥当
- 理由2: 営業活動時間30%削減は十分な投資効果
- 理由3: 過去の営業系ツール導入で成功実績あり
【信頼度】92%
【次のアクション】導入手続きを進めてください。3ヶ月後に効果測定を実施。
```

「完璧です！過去事例も参照して、的確な判断ができています」
