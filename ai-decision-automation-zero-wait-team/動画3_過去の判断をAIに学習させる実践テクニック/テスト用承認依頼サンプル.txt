# テスト用承認依頼サンプル - Few-shot学習の効果検証用

## 🧪 基本動作確認用（シンプルな案件）

### テスト1: 明確な承認案件
```
営業支援ツール「SalesBoost」の導入申請
月額4万円（年間48万円）
営業チーム15名で利用予定
見込み客管理の効率化により、営業活動時間を30%削減見込み
3ヶ月で黒字化予定
```

### テスト2: 明確な却下案件
```
社内パーティー用の高級ケータリング
予算：50万円
参加者：50名（1人1万円）
効果：チームビルディング
頻度：今回限り
```

### テスト3: 情報不足案件
```
新しいツールの導入
予算：だいたい20-30万円くらい
効果：業務が楽になる
いつ：なるべく早く
```

## 🎯 Few-shot学習の効果測定用（過去事例と類似）

### テスト4: SFA系ツール（成功事例と類似）
```
顧客管理システム「CustomerMax」導入申請
初期費用25万円、月額3万円
営業部20名で使用
商談履歴の一元管理で成約率向上見込み
導入企業の平均ROI: 250%
責任者：営業部長 実施時期：来月から
```
期待される回答：過去のSFAツール成功事例を参照して承認

### テスト5: 広告案件（失敗事例と類似）
```
Instagram広告キャンペーン
予算：60万円
期間：3ヶ月
ターゲット：20-30代女性
目的：新規顧客獲得
KPI：設定中
```
期待される回答：SNS広告の失敗事例を参照して慎重な判断

### テスト6: 重複ツール（却下事例と類似）
```
チャットツール「TeamTalk」導入
月額5万円
機能：チャット、ビデオ会議、ファイル共有
現在Slackを利用中だが、TeamTalkの方が新しい
```
期待される回答：重複機能の却下事例を参照して却下

## 🔄 条件付き承認のテストケース

期待される回答：段階的導入の事例を参照して条件付き承認

### テスト8: 新技術導入
```

```

## 🎯 精度測定用チェックリスト

各テストケースで確認すべき項目：
- [ ] 過去事例への言及があるか
- [ ] 判断の一貫性が保たれているか
- [ ] 信頼度スコアが適切か
- [ ] 出力フォーマットが正しいか
- [ ] 判断理由が具体的か

---

使用方法：
1. Few-shot学習なしでテスト1-13を実行
2. 結果を記録
3. Few-shot学習ありで同じテストを実行
4. 改善率を測定
5. 特に事例と類似したケース（テスト4-8）での精度向上を確認
