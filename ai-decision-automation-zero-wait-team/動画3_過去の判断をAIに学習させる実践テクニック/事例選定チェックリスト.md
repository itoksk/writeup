# 事例選定チェックリスト - Few-shot学習用

## 📋 事例選定の基本原則

### ✅ 良い事例の条件
- [ ] **1年以内の新しい事例**である
- [ ] **判断結果が明確**（成功/失敗が数値で分かる）
- [ ] **金額レンジが適切**（10万円〜100万円程度）
- [ ] **判断理由が詳細**に記録されている
- [ ] **典型的なパターン**を表している
- [ ] **再現性が高い**案件である

### ❌ 避けるべき事例
- [ ] 3年以上前の古い事例
- [ ] 特殊すぎる・例外的な案件
- [ ] 個人的事情が強く影響した判断
- [ ] 結果が曖昧・評価困難
- [ ] 政治的判断が絡む案件
- [ ] 一度きりの特別対応

## 🎯 カテゴリ別の選定目安

### 1. 承認・成功事例（5件）
必須カテゴリ：
- [ ] 営業系ツール（SFA、CRM等）: 1-2件
- [ ] マーケティング系（MA、広告等）: 1-2件
- [ ] 人材投資（研修、教育等）: 1件
- [ ] インフラ系（クラウド、セキュリティ等）: 1件
- [ ] 働き方改革系（リモート、効率化等）: 1件

### 2. 承認・失敗事例（3件）
学習効果の高い失敗：
- [ ] 期待効果が出なかった広告・マーケティング: 1件
- [ ] 活用されなかった高額ツール: 1件
- [ ] 連携・移行で問題が発生したシステム: 1件

### 3. 却下・正解事例（3件）
却下理由の代表例：
- [ ] 費用対効果が不明確: 1件
- [ ] 既存ツールと重複: 1件
- [ ] 実績・具体性がない: 1件

### 4. 条件付き承認事例（2件）
段階的導入の成功例：
- [ ] 大規模システムの段階導入: 1件
- [ ] 新技術の試用期間設定: 1件

## 📊 事例の質を高める工夫

### データの充実度チェック
各事例について以下を確認：

#### 必須項目
- [ ] 具体的な金額
- [ ] 明確な判断結果
- [ ] 判断理由（2つ以上）
- [ ] 実際の結果（数値含む）
- [ ] 日付（1年以内）

#### 推奨項目
- [ ] ROI（%表記）
- [ ] 導入期間
- [ ] 担当部署
- [ ] 競合他社比較
- [ ] 失敗要因分析（失敗事例の場合）

### 事例文の作成ポイント
1. **構造化された記述**
   ```
   案件名: [具体的なツール・サービス名] [金額]
   判断: [承認/却下/条件付き]
   結果: [具体的な数値を含む成果]
   学習: [今後の判断に活かすポイント]
   ```

2. **数値の明確化**
   - 曖昧: 「効果があった」
   - 明確: 「営業効率35%向上、5ヶ月でROI達成」

3. **学習ポイントの一般化**
   - 特殊: 「A社のツールは良かった」
   - 一般: 「効果測定が明確な営業系ツールは積極承認」

## 🔄 定期的な見直し

### 月次レビュー（毎月第1月曜日）
- [ ] 新規案件の追加（3-5件）
- [ ] 古い事例の削除（1年超過）
- [ ] 効果の再評価
- [ ] カテゴリバランスの確認

### 四半期レビュー（3ヶ月ごと）
- [ ] 全事例の棚卸し
- [ ] 判断精度の測定
- [ ] 事例数の最適化（10-15件を維持）
- [ ] 新カテゴリの追加検討

## 📝 記録フォーマット

### エクセル管理用テンプレート
```
A列: 依頼内容（ツール名と金額）
B列: 判断結果（承認/却下/条件付き）
C列: 判断理由（箇条書き）
D列: 実際の結果（数値含む）
E列: 優先度（高/中/低）
F列: 日付（YYYY-MM-DD）
G列: カテゴリ
H列: 金額（数値のみ）
I列: ROI（%）
J列: 導入期間
```

### プロンプト転記用フォーマット
```
【事例○】[ツール名][金額]
判断: [結果]
結果: [成果を数値で]
学習: [一般化されたポイント]
```
