🔑 なぜプロンプト設計が重要なのか？

💡 プロンプトの有無で天と地の差

プロンプトなし: AIは一般的な知識で適当に判断 → 使い物にならない

最適化プロンプト: あなたの判断基準で分析 → 信頼できる分身に！

🧩 プロンプト設計の3層構造

📋 Make + ChatGPTで実装する効果的なプロンプト

1️⃣ 役割設定（Role）- AIの人格を決める

ChatGPTに「誰として」振る舞わせるかを明確に定義

あなたは創業15年のIT企業で、10年以上の管理職経験を持つ 承認責任者です。コスト意識が高く、投資対効果を重視しますが、 チームの成長に必要な投資は積極的に承認する方針です。 部下からの承認依頼に対して、公平で一貫した判断を行ってください。

💡 ポイント: 具体的な経験年数や価値観を入れることで、AIの判断に一貫性が生まれます

2️⃣ 判断基準（Criteria）- 明確なルール

数値基準と評価軸を具体的に設定

【金額基準】 - 30万円以下＋ROI 200%以上 → 即承認 - 50万円以下＋必要性明確 → 条件付き承認 - 50万円超 or ROI不明 → 追加情報要求 - 法的リスクあり → 即却下 【重視ポイント】 1. 投資回収期間（12ヶ月以内が理想） 2. チーム生産性への影響（20%以上の改善期待） 3. 競合他社の動向（業界標準への対応） 4. 実行体制の確実性（責任者と期限が明確）

3️⃣ 出力形式（Format）- ブレない回答

Makeの自動化で処理しやすい構造化フォーマット

必ず以下の形式で回答してください： 【判断】承認/条件付き承認/却下/追加情報要求 【理由】 - 理由1: （具体的な根拠） - 理由2: （データに基づく説明） - 理由3: （リスクと効果の評価） 【信頼度】0-100%（判断の確実性） 【次のアクション】（申請者への具体的指示）

📊 Before/After 実例比較

❌ プロンプトなし・適当な設定

Slack入力:
「マーケティングツール月額8万円の承認お願いします」
ChatGPT回答:
「マーケティングツールの導入は良いアイデアだと思います。8万円は少し高いかもしれませんが、効果があれば良いでしょう。他の選択肢も検討してみてください。」

問題点:
• 判断が不明確（承認？却下？）
• 根拠が曖昧
• 次のアクション不明
• 自動化で使えない

✅ 最適化プロンプト使用

Slack入力:
「マーケティングツール月額8万円の承認お願いします」
ChatGPT回答:
【判断】追加情報要求
【理由】
- 理由1: 月額8万円は年間96万円の投資となり詳細検討が必要
- 理由2: ROIと具体的な活用計画が不明
- 理由3: 他社ツールとの比較検討結果が必要
【信頼度】95%
【次のアクション】以下の情報を提供してください：①期待ROI ②3社以上の比較表 ③活用計画書

改善点:
• 明確な判断結果
• 具体的な根拠
• 実行可能な指示
• Make自動化に最適

🛠️ Make実装での実践テクニック

⚙️ ChatGPTモジュールでの設定方法

1. System Messageの活用

Make内のChatGPTモジュール設定: 1. Role: System を選択 2. Message Content: プロンプト全文を貼り付け 3. Temperature: 0.3（一貫性重視） 4. Max Tokens: 500（簡潔な回答）

2. 変数を使った動的プロンプト

承認依頼の詳細: 申請者: {{1.user_name}} 案件: {{1.text}} 申請日時: {{1.ts}} この情報を基に判断してください。

3. エラーハンドリング

もし情報が不足している場合は、 【判断】追加情報要求 として、必要な情報を具体的に列挙してください。

🎯 プロンプト改善の実践テクニック

📖 具体例の威力
【成功事例】 Slack有料版(月3万円) → 効率40%向上で承認 【失敗事例】 高額CRM(月20万円) → 活用率10%で却下

過去の判断例を入れることで、AIの判断精度が劇的に向上します

🔀 条件分岐の明確化
IF 法務リスク → 即却下 ELSEIF 予算超過 → 上位承認 ELSEIF ROI不明 → 情報要求 ELSE → 通常判断

プログラムのような明確な条件分岐で、判断の一貫性を保証

  • 誤判断をプロンプトに反映
  • 新しいケースを追加
  • チームフィードバック反映

⚠️ 注意点とトラブルシューティング

🚨 よくある失敗と対策

失敗例 原因 対策
判断がブレる 基準が曖昧 数値基準を明確化
回答が長すぎる 形式指定なし 文字数制限を追加
重要項目見落とし チェックリスト不足 必須確認項目を明記
Make連携エラー フォーマット不一致 出力形式を厳密に指定
