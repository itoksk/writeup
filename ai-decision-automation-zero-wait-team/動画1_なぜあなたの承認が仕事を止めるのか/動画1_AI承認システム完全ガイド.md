## ⏱️ 10分セットアップ手順

### 準備（3分）
```bash
□ Slackワークスペースにログイン
□ クレジットカード準備（OpenAI用）
□ このガイドを開いておく
```

### Step 1: OpenAI APIキー取得（2分）
```bash
1. https://platform.openai.com/api-keys
2. 「+ Create new secret key」
3. 名前「AI承認Bot」→ キーをコピー
```

### Step 2: Slack Bot作成（3分）
```bash
1. https://api.slack.com/apps → 「Create New App」
2. 「From scratch」→ 名前「AI承認Bot」
3. OAuth & Permissions → Bot Token Scopesに追加:
   - channels:history
   - channels:read  
   - chat:write
   - chat:write.customize
4. 「Install to Workspace」→ Tokenコピー
5. Slackで #approval-requests 作成 → /invite @AI承認Bot
```

### Step 3: Makeで接続（2分）
```bash
1. https://www.make.com → 新規シナリオ
2. 3つのモジュールをドラッグ&ドロップ:
   Slack受信 → ChatGPT判断 → Slack返信
3. 各モジュールにToken/APIキーを貼り付け
```

**完成！🎉**

## 🤖 AI判断ロジック（プロンプト）

### シンプル版（デモ用）
```
あなたは経験豊富な管理職です。
判断基準：
- 30万円以下かつROI 200%以上 → 承認
- それ以外 → 詳細検討が必要

必ず以下の形式で回答：
【判断】承認 または 却下
【理由】50文字以内で説明
```

### 詳細版（実運用向け）
```
承認判断AI v1.0

【基本ルール】
1. 金額別判断
   - 10万円以下：原則承認
   - 10-50万円：ROI重視
   - 50万円超：戦略性重視

2. カテゴリ別重点
   - マーケティング：過去実績とROI
   - システム：効率化効果と保守性
   - 人材育成：スキルマップ適合性

3. リスク評価
   - 低：迅速承認
   - 中：条件付き承認
   - 高：段階的承認

【出力形式】
判断：承認/条件付き承認/却下
理由：（簡潔に）
条件：（該当する場合のみ）
次のアクション：（具体的に）
```

## 🚨 トラブルシューティング

### よくあるエラーと解決法

| エラー | 原因 | 解決法 |
|--------|------|--------|
| Slackメッセージが来ない | Botが未招待 | `/invite @AI承認Bot` |
| Invalid API key | コピーミス | 空白を削除して再入力 |
| Makeが動かない | 未保存 | 保存してScheduling ON |
| 返信が文字化け | 文字コード | UTF-8を確認 |

### 緊急時のバックアップ
1. **録画デモを使用**（最も安全）
2. **ローカルJSデモ**（デモ用_AI承認システム.js）
3. **手動でSlack返信**（最終手段）
