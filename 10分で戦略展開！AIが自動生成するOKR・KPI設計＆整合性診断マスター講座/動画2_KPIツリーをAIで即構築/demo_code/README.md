# デモコード・テンプレート集

## ファイル一覧

### 1. KPIツリー_サンプル.csv
ChatGPTで生成したKPIツリー（3階層構造）のサンプルデータです。
Excelまたは Googleスプレッドシートにインポートして使用できます。

#### 使用方法
1. ExcelまたはGoogleスプレッドシートを開く
2. 「ファイル」→「インポート」→「アップロード」
3. このCSVファイルを選択
4. インポート設定で「カンマ」を区切り文字に指定
5. 「データをインポート」をクリック

#### カスタマイズ
- KPI名を自社の目標に合わせて変更
- 目標値を現実的な数値に調整
- 担当者を実際の担当者名に変更
- 頻度・期限を自社の運用に合わせて調整

## Excel/スプレッドシートテンプレートの作成方法

### 基本構成

#### シート1：KPIツリー一覧
| 列 | 項目 | 説明 |
|----|------|------|
| A | 階層 | 第1階層/第2階層/第3階層 |
| B | KPI | KPI名称 |
| C | 目標値 | 達成すべき数値目標 |
| D | 担当 | 担当者・チーム名 |
| E | 頻度/期限 | 実施頻度または期限 |
| F | 進捗 | チェックボックス（☐/☑） |
| G | ステータス | 順調/注意/遅延 |

#### シート2：進捗ダッシュボード
各KPIの達成状況を可視化するダッシュボードシートです。

| 項目 | 内容 |
|------|------|
| 第1階層KPI | 大きく表示（進捗率とステータス） |
| 第2階層KPI | 中サイズで表示（達成状況を色分け） |
| 第3階層KPI | チェックリスト形式 |
| グラフ | 進捗推移を折れ線グラフで表示 |

#### シート3：週次レビュー記録
週次でのレビュー結果を記録します。

| 列 | 項目 |
|----|------|
| A | レビュー日 |
| B | KPI名 |
| C | 目標値 |
| D | 実績値 |
| E | 達成率（%） |
| F | 次のアクション |
| G | 担当者 |

## 書式設定とビジュアライゼーション

### 階層別の色分けルール
- **第1階層**: 濃い青背景 + 白文字（最重要KPI）
- **第2階層**: 中間の青背景 + 黒文字（中間指標）
- **第3階層**: 薄い青背景 + 黒文字（行動レベルKPI）

### インデント設定
階層構造を視覚的に分かりやすくするため、インデントを使用：
- 第1階層：インデントなし
- 第2階層：1段階インデント（左マージン +10px）
- 第3階層：2段階インデント（左マージン +20px）

### 条件付き書式
ステータス（G列）に応じて自動で色を変更：
- 順調: 緑
- 注意: 黄色
- 遅延: 赤

進捗（F列）のチェックボックス：
- ☐: 未完了（グレー）
- ☑: 完了（緑）

## 数式の例

### 達成率の自動計算
```
=IF(C2="", "", (実績値/目標値)*100)
```

### ステータスの自動判定
```
=IF(達成率>=90, "順調", IF(達成率>=70, "注意", "遅延"))
```

### 進捗バーの可視化
```
=REPT("■", INT(達成率/10)) & REPT("□", 10-INT(達成率/10))
```

### 第1階層KPIの集計進捗
第2階層・第3階層の達成状況から、第1階層KPIの総合進捗を計算：
```
=AVERAGE(第2階層KPIの達成率範囲)
```

## KPIツリーの階層構造の例

### 第1階層: Key Result
```
新規顧客獲得数を前年比30%増加
```

### 第2階層: 中間指標
```
├─ リード数を月間100件以上獲得
├─ 商談化率を25%以上に向上
└─ 成約率を20%以上に維持
```

### 第3階層: 行動レベルKPI
```
リード数を月間100件以上獲得
  ├─ 週1回の市場調査を実施し、新規見込み客リストを作成
  ├─ 月2回のウェビナーを開催し、参加者50名以上を目標
  └─ SNS広告経由のリード獲得数を週15件以上

商談化率を25%以上に向上
  ├─ 初回商談の事前準備を100%実施
  ├─ 週次で商談結果を分析し、改善点を抽出
  └─ 顧客ニーズヒアリングシートを全商談で使用

成約率を20%以上に維持
  ├─ 提案書のテンプレートを統一し、品質を均一化
  ├─ 四半期ごとに顧客満足度アンケートを実施
  └─ 失注理由を月次で分析し、改善策を共有
```

## ChatGPTとの連携フロー

1. **前回作成したOKRをChatGPTに入力**
   - プロンプト集のプロンプト1を使用
   - OKRをコピー&ペースト

2. **KPIツリーを生成**
   - ChatGPTの出力を確認
   - 必要に応じて再生成

3. **スプレッドシートに貼り付け**
   - 表形式のままコピー
   - スプレッドシートのA2セルから貼り付け

4. **階層構造を整理**
   - インデントを適用
   - 色分けルールを適用

5. **進捗管理を開始**
   - 週次でチェックボックスを更新
   - ダッシュボードで可視化

## タスク管理ツールとの連携

### Notionテンプレート
第3階層の行動レベルKPIを、Notionのデータベースに転記：

| プロパティ | 内容 |
|-----------|------|
| タスク名 | 行動レベルKPI |
| 担当者 | 担当者名 |
| 期限 | 頻度・期限 |
| ステータス | 未着手/進行中/完了 |
| 関連KPI | 第2階層KPI（リレーション） |

### Asanaプロジェクト
第3階層のKPIをAsanaのタスクとして登録：
- プロジェクト名：「営業部 OKR Q1」
- セクション：第2階層KPI
- タスク：第3階層KPI
- 担当者：担当者名
- 期限：頻度・期限

### Trelloボード
- リスト：第2階層KPI
- カード：第3階層KPI
- チェックリスト：具体的なアクション項目

## 進捗レビュー会議での活用

### 週次レビュー
- 第3階層KPIのチェックボックスを確認
- 未完了のKPIについて、阻害要因を議論
- 次週のアクションを決定

### 月次レビュー
- 第2階層KPIの達成率を確認
- 達成率が低いKPIについて、第3階層KPIを見直し
- 必要に応じてアクションを追加・修正

### 四半期レビュー
- 第1階層KPI（Key Results）の達成状況を評価
- 次四半期のKPIツリーを更新
- ChatGPTを使って新しいKPIツリーを生成

## トラブルシューティング

### Q: ChatGPTの出力が階層構造になっていない
A: プロンプトに「3階層構造で出力」と明記してください。また、「第1階層」「第2階層」「第3階層」と明確に指定すると確実です。

### Q: 第3階層のKPIが抽象的すぎる
A: プロンプトに「具体的なアクション」「担当者」「頻度・期限を含める」と追加してください。例を示すとより効果的です。

### Q: KPIツリーが長すぎて管理できない
A: 1つのKey Resultずつ生成し、段階的に拡大してください。また、第3階層は2〜3個に絞ることを推奨します。

### Q: スプレッドシートの階層構造が分かりにくい
A: インデントと色分けを適用してください。また、グループ化機能を使って第2階層・第3階層を折りたたみ可能にすると便利です。
