# デモコード・テンプレート集

## ファイル一覧

### 1. 整合性スコア表_サンプル.csv
ChatGPTで生成した整合性診断結果のサンプルデータです。
Googleスプレッドシートにインポートして可視化できます。

#### 使用方法
1. Googleスプレッドシートを開く
2. 「ファイル」→「インポート」→「アップロード」
3. このCSVファイルを選択
4. インポート設定で「カンマ」を区切り文字に指定
5. 「データをインポート」をクリック

#### カスタマイズ
- 診断観点を自社の評価基準に合わせて変更
- スコアを実際の診断結果に更新
- コメント欄に詳細な説明を追加

## スプレッドシートテンプレートの作成方法

### 基本構成

#### シート1：整合性スコア表
| 列 | 項目 | 説明 |
|----|------|------|
| A | 診断観点 | 診断の観点（上位目標との整合性、部門間の重複、方向性のズレ） |
| B | スコア | 5段階評価（5:完全一致、1:大きなズレ） |
| C | 評価 | スコアに応じた評価コメント |
| D | ステータス | 視覚的なステータス表示（🟢🟡🔴） |
| E | コメント | 詳細な説明・改善提案 |

#### シート2：指摘事項一覧
整合性診断で発見された具体的な指摘事項を記録します。

| 列 | 項目 |
|----|------|
| A | 指摘ID |
| B | カテゴリ（重複/矛盾/ギャップ） |
| C | 関係部門 |
| D | 指摘内容 |
| E | 改善提案 |
| F | 優先度（高/中/低） |
| G | 対応状況（未着手/対応中/完了） |

#### シート3：部門OKRマッピング
経営方針と部門OKRの対応関係を可視化します。

| 列 | 項目 |
|----|------|
| A | 経営方針 |
| B | 対応する部門OKR（営業部） |
| C | 対応する部門OKR（マーケティング部） |
| D | 対応する部門OKR（開発部） |
| E | 対応する部門OKR（人事部） |
| F | カバレッジ評価 |

## 書式設定とビジュアライゼーション

### スコア別の色分けルール
- **4〜5点（🟢緑）**: 整合性良好、現状維持
- **3点（🟡黄）**: 一部改善が必要、軽微な調整
- **1〜2点（🔴赤）**: 要改善、早急な対応が必要

### 条件付き書式
スコア（B列）に応じて自動で色を変更：
- 4以上: 緑背景
- 3: 黄背景
- 2以下: 赤背景

ステータス（D列）の絵文字表示：
```
=IF(B2>=4, "🟢", IF(B2>=3, "🟡", "🔴"))
```

## 数式の例

### 総合整合性スコアの計算
全ての診断観点のスコアの平均を算出：
```
=AVERAGE(スコア範囲)
```

### 整合性判定
総合スコアに基づいて整合性を判定：
```
=IF(総合スコア>=4, "整合性良好", IF(総合スコア>=3, "部分的改善必要", "要改善"))
```

### カバレッジ評価
経営方針の各項目に対応する部門OKRの数を集計：
```
=COUNTIF(部門OKR範囲, "<>")
```

## 整合性診断レポートのテンプレート

### エグゼクティブサマリー
```markdown
# OKR整合性診断レポート（2025年Q1）

## 総合評価
- **総合整合性スコア**: 3.0/5.0
- **ステータス**: 部分的改善必要

## 主要な発見事項
1. 営業部とマーケティング部で「リピート率向上」が重複
2. 開発部と人事部の連携が不明確
3. 経営方針「売上高15%増」が営業部OKRに明示されていない

## 推奨される次のステップ
1. 営業部とマーケティング部の役割分担を明確化
2. 開発部と人事部の連携会議を設定
3. 営業部のKey Resultsを具体化
```

### 詳細分析セクション
各診断観点ごとに、以下の情報を含める：
- スコアとその根拠
- 具体的な指摘事項
- 改善提案
- 期待される効果

## ChatGPTとの連携フロー

1. **各部門のOKRを一覧化**
   - スプレッドシートで各部門のOKRを整理
   - 表形式でコピー可能な状態にする

2. **整合性診断プロンプトを実行**
   - プロンプト集のプロンプト1を使用
   - 経営方針と部門OKRをコピー&ペースト

3. **診断結果を確認**
   - ChatGPTの出力を確認
   - 整合性スコアと指摘事項を抽出

4. **スプレッドシートに貼り付け**
   - 整合性スコア表に結果を貼り付け
   - 指摘事項一覧に詳細を記録

5. **ビジュアル化**
   - 条件付き書式を適用
   - レーダーチャートやダッシュボードを作成

6. **改善アクションの実行**
   - 指摘事項に基づいて部門OKRを調整
   - 四半期ごとに再診断

## ビジュアル化のオプション

### 1. レーダーチャート
整合性スコアの3観点をレーダーチャートで表示：
- 上位目標との整合性
- 部門間の重複
- 方向性のズレ

### 2. サンキーダイアグラム
経営方針から各部門OKRへの流れを可視化：
- 経営方針（左）→ 部門OKR（右）
- 対応関係を線の太さで表現

### 3. ヒートマップ
部門OKRと経営方針の対応関係をヒートマップで表示：
- 行：経営方針
- 列：部門
- セル：対応度（色の濃さで表現）

### 4. ネットワーク図
部門間の重複・矛盾をネットワーク図で可視化：
- ノード：部門OKR
- エッジ：重複・矛盾の関係

## トラブルシューティング

### Q: ChatGPTの診断結果が曖昧
A: プロンプトに「具体的な指摘」「改善提案を含める」と明記してください。また、「5段階評価」と数値評価を指定すると確実です。

### Q: 整合性スコアが全て低い
A: 経営方針と部門OKRの表現が抽象的すぎる可能性があります。できるだけ定量的な表現を使い、再度診断してください。

### Q: 部門OKRが多すぎて出力が長い
A: 1〜2部門ずつ診断を実施し、段階的に拡大してください。最後に全体の統合診断を実施すると効果的です。

### Q: 改善提案が実行不可能
A: ChatGPTの提案は参考にし、自社の業界特性や戦略的意図を踏まえて調整してください。AIはサポートツールであり、最終判断は人間が行います。

## 応用例

### シナリオプランニング
異なる市場環境シナリオでの整合性を事前診断：
- シナリオ1：市場縮小 → 売上目標を下方修正した場合の整合性
- シナリオ2：市場拡大 → 新規事業を加速した場合の整合性

### ベンチマーキング
他社や業界平均との整合性スコアを比較：
- 自社の総合スコア vs 業界平均
- 診断観点別のベンチマーク

### アラート機能
整合性スコアが一定値を下回った場合、自動でアラートを送信：
- Google Apps Scriptでスクリプトを作成
- スコアが2以下になったら、メール通知
